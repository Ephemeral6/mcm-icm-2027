# 06 获奖策略与评审标准

> 奖项等级体系为 **【官方·已验证】**；奖项比例、评审细节为 **【经验·未验证】**（社区/自称评委视角文章），两处社区来源的比例数字互有出入，均列出供参考。
> 这份文件讲"好论文长什么样"；[优秀论文的真实制作过程](优秀论文的真实制作过程.md) 讲"好论文实际是怎么被做出来的"（迭代过程、团队协作节奏），并附一份"网上流传但被证伪"的辟谣清单。

## 1. 奖项等级（已验证·官方）

由低到高共七级：

1. Disqualified（取消资格）
2. Unsuccessful Participant
3. **Successful Participant（S 奖，"成功参赛"）**
4. **Honorable Mention（H 奖，二等）**
5. **Meritorious（M 奖，一等）**
6. **Finalist（F 奖，特等提名）**
7. **Outstanding Winner（O 奖，特等）**

- 评审 3–4 月进行；论文提交后不重评、不受理申诉；队员署名顺序不影响评审。
- O 奖之上另有学会冠名奖（AMS/SIAM/INFORMS/MAA/ASA 等）及 C 题专设的 Veena Mendiratta 奖。

## 2. 各级比例（经验·未验证，两个社区来源）

| 等级 | 来源甲（知乎"评审流程"文） | 来源乙（mcmicm.org.cn） |
|---|---|---|
| O | ~1% | ~0.5% |
| F | ~1% | ~1% |
| M | ~10–15% | ~7% |
| H | ~30% | ~23% |
| S | ~50% | ~65% |

结论量级：**M 奖以上约 10–15%，H 奖以上约 35–45%**。COMAP 不固定公布比例，精确数字可从官方各题 Results PDF 反推（调研空白）。

## 3. 评审流程（经验·未验证，多来源一致）

- **盲评**：论文只以控制号标识；评审约在赛后三周的第一个周末启动。
- **两阶段**：
  - 第一阶段"鉴别/分诊（triage）"：每篇约 **10 分钟**，分三类——晋级（H 及以上）、止步 S 奖、不合格；
  - 第二阶段：每篇 **15–30 分钟**逐轮筛选；
  - O 奖由全体评委末轮共同讨论产生，须竞赛主席与副主席一致同意。
- 直接推论：**评委第一阶段基本只看摘要 + 图表 + 结构**。摘要平庸 = 10 分钟内出局，正文再好也没用。

## 4. 评委关注点（源自 2024 年 C 题官方评委文章的中文解读，经验·未验证）

- COMAP 每年赛后发布各题**评委评论文章（judge commentary）**——了解评委口味的最佳一手材料，务必读近三年 C 题的。
- 2024 年 C 题评委明确批评"只堆机器学习"的做法：相比黑箱 ML，评委更认可"查专业资料识别关键因素→构建可解释的动态表达式"的路径，**能解释模型每一项的来历和意义**才符合数学建模理念。
- 论文应展现**模型构建过程**（为什么选它、假设如何论证），而不是只给结果。
- 对你们的含义：AI/ML 可以用，但必须配上人工的解释层——特征重要性分析、机理讨论、敏感性分析，是"用了 ML 还能得奖"的关键。

## 5. O 奖论文的共性套路（经验·未验证，社区拆解共识）

- **摘要（第一页）**：问题重述一句话 → 每问用什么模型 → 关键数值结果 → 模型亮点；具体、有数字，不写空话。竞赛摘要是"详细摘要"，可接近一页（远长于期刊摘要）。官方原话佐证：获奖论文往往凭摘要与其他论文区分（已验证）。
- **模型总览图**：一张流程图讲清全文技术路线（几乎是标配）。
- **假设列表**：每条假设配 justification（为什么合理）。
- **图表**：风格统一、信息密度高、每图有结论性 caption；图表质量是 10 分钟初审的生死线。
- **敏感性分析 + 优缺点讨论**：标配章节，展示模型稳健性与自知之明。
- **符号表、规范引用**：细节整洁度影响评委印象。

## 6. 常见失误清单（经验·未验证，社区共识）

1. 摘要最后一小时才写、空洞无数字 —— 最致命
2. 选题反复横跳，浪费第一天
3. 论文出现姓名/校名（违反盲评，直接出事）
4. 严格分工导致写作最后一天才开始
5. 只堆黑箱模型、无解释、无敏感性分析
6. 图表粗糙、风格不统一
7. AI 使用不披露（新规下 = 抄袭风险，见 02）
8. 页数超 25、PDF 命名错误、卡点提交失败
9. 引用 AI 编造的不存在文献
10. 赛前在数模群里讨论思路导致查重连坐
