# 优秀论文的真实制作过程

> 专项调研（2026-07-19，95 代理）。这份文件和同目录的 [README](README.md)（评审标准/评委关注点/共性套路）互补：README 讲"好论文长什么样"，这份讲"好论文实际是怎么被做出来的"——迭代过程、时间分配、团队真实协作模式。
> **诚实的开场白**：这次调研最大的发现不是找到了多少"干货"，而是发现网上流传的"O 奖是怎么做出来的"经验帖，有相当一部分**经不起核实**——本文件把幸存的真实经验和被戳穿的"看起来很有道理但是假的"内容分开列，后者本身就是有用信息（知道哪些是假的，能少走弯路）。

## 0. 先说结论

1. **不存在系统性总结"O 奖论文共性"的可信资料**——本次调研专门去找"对多篇 O 奖论文做过文本共性分析"的文章，几乎全军覆没（唯一幸存的一条也只是对**一篇**评委评论文章的转述整理，不是独立的多论文研究）。这意味着"O 奖套路"这种东西，公开互联网上其实没有可靠的系统总结，你们能读到的都是零散的单一团队经验。
2. **真实的迭代过程比想象中混乱**：有队伍在比赛中途整体推翻已选题目重新开始，依然拿到美赛一等奖——好论文不是"一开始就想对了"，而是"能在错的时候及时发现并调头"。
3. **团队协作常常不是靠严格分工，而是自然形成的**：建模手因为持续输出想法，很自然地成为事实上的领导者，其他人围绕他的想法调整执行——这和 [04-三人分工](../04-三人分工/README.md) 里"人人都要懂建模"的建议方向一致，但也有队伍明确反对这种"一言堂"模式，主张集体讨论决策。两种做法都有人用，没有唯一正确答案。
4. **摘要的地位被反复强调**：不止一个来源独立提到摘要在评审"分诊"环节的决定性作用，也有队伍分享了具体的摘要写作流程（见第 3 节）。

## 1. 真实的迭代过程：不是"一次想对"，而是"能及时改"

- 【经验·单一团队，弱证据】有队伍在比赛进行到中途，**整个推翻已经选定的题目重新开始**，最终仍然拿到美赛一等奖。这条信息只有孤证，无法核实具体年份和队伍，但方向上和其他经验帖"选题要果断、发现不对要及时止损"的共识一致。
- 【经验·单一团队】另一支队伍的做法是：建模思路走偏是常事，发现的方式**不是靠个人独自判断，而是团队集体讨论**，通过查文献和及时看反馈结果来判断方向对不对。
- 调研没有找到能验证的"第一版模型和最终版模型具体差多少"的案例——网上流传的几个"模型迭代路径"故事（比如"先用方法 A 定位，再用方法 B 精修，最后自己改进出方法 C"这类叙事）在核实中站不住脚，来源要么对不上、要么细节被证明是编的，本文件不收录，避免以讹传讹。

**给你们的实操含义**：不要预设"O 奖队伍第一稿就是对的"，真实情况更可能是"中途发现不对、推倒重来"。11 月的模拟赛除了练"从头到尾走一遍流程"，也该刻意练"发现方向错了之后怎么快速调头"——这是被低估的能力。

## 2. 真实的时间分配与协作节奏

- 【经验·单一团队】拿到赛题后，一支队伍的实际节奏是"个人独立看题 + 集体开会"交替：早上开始每隔 1–2 小时开一次会，**6 小时内锁定最终选题**——比我们 [03-选题决策清单](../03-题型与选题策略/选题决策清单.md) 建议的"半天内定题"更快，但方向一致（选题要快）。
- 【经验·单一团队，混合证据】另一支队伍认为**论文写作应该贯穿整个赛程**，而不是等模型定了才动笔——论文手从选题结束就开始写，边做边记录建模过程。同一支队伍把美赛工作打趣总结成"数学建模 + 语文建模（中文行文）+ 英语建模（翻译润色）+ 美术建模（图表排版）"四部分，其中论文/翻译/图表这三块占了论文手约 3/4 的工作量——这支队伍最终拿的是 **M 奖（一等奖）而非 O 奖**，供参考但不代表 O 奖队伍也是这个比例。
- 同一支队伍还认为**美赛比国赛更看重建模思路本身、而不是精确的计算结果**，所以对编程手的要求相对没那么高，思路对、结果合理就有机会获奖——这是一个队伍的主观判断，没有独立信源佐证，仅供参考。

**给你们的实操含义**：写作不要留到最后一天，这条在两支不同队伍的经验里都被强调；选题要快（几小时内定，不要拖到第一天结束）。

## 3. 摘要的真实写作流程（评委权重最高的一页，值得单独抠细节）

- 摘要在评审"分诊（triage）"环节的决定性作用，在多个独立来源里被反复提及，是本轮证据强度最高的一条结论——直接呼应 [README 第 3 节](README.md) 已验证的"评委第一阶段基本只看摘要"。
- 【经验·单一团队】一支队伍的实际摘要写作流程：**先由一人写中文初稿 → 三人讨论修改 → 翻译成英文 → 多轮修改 → 指导老师在翻译前后各检查一次**，整个摘要环节大约花了 **5 小时**。
- 调研原本想找"摘要改了几十遍"这类具体次数细节，以及摘要标准结构的系统总结，检索到的几篇文章在核实后被发现内容对不上（比如摘要"五要素结构""只写优点不写缺点"这类具体建议，来源不可靠，未收录）。

**给你们的实操含义**：把摘要当成一个独立的、需要专门排时间的工序，不是"论文写完了顺手总结一下"——5 小时这个量级可以作为你们排 [05-赛中96小时作战手册](../05-备赛路线/赛中96小时作战手册.md) 时间表时的参考基准（该文件目前建议"D4 下午整块时间"，和这个经验量级吻合）。

## 4. 假设与模型选择：好论文和普通论文的可观察差异（中等强度证据）

这部分证据强度最高的来源是对 **2024 年美赛 C 题官方评委评论**的一篇中文解读整理（不是独立的多论文研究，是对评委原话的转述）：

- 好的假设是**具体、可数学化验证、并附有解释**的陈述，而不是"假设数据真实准确"这类走流程的套话——这条和 [12-模型与算法策略](../12-模型与算法策略/README.md) 已验证的"官方拒绝套路化"结论完全一致，是这次调研里少数能和已有内容交叉印证的部分。
- 评委明确不认可"黑箱"式机器学习使用——**即使事后补了交叉验证/敏感性分析来"证明"参数选择合理，仍然被认为缺乏对模型选择原因的真实讨论**，本质是"先射箭再画靶"式的事后合理化，不是真实的建模过程。这一条比我们之前掌握的"不要黑箱建模"结论更进一步：**补充验证不能挽救一个从一开始就没有理由的模型选择**。

## 5. 反面案例：为什么"很努力但没获奖"

这是原本最想深挖的部分，但**几乎没有找到能通过核实的具体案例**——网上有一些声称分析"投入很多但没获奖"失败模式的文章，在核实阶段被发现内容与来源对不上或证据不足，本文件不收录任何具体案例，避免误导。

唯一能确认的相关信息，是本仓库此前已验证的内容：国赛官方 2024 年会议纪要点名批评的"n 板斧"（见 [13-国赛CUMCM/官方评阅内幕](../13-国赛CUMCM/官方评阅内幕(2024年会议纪要).md)）——不管题目是什么都套用同一套固定模型清单，"一知半解、牵强附会"——这大概率就是"模型复杂、投入很多，但依然不获奖"最主要的方法论层面原因：复杂度不等于理解深度，评委看得出来区别。

## 7. 本轮调研的诚实评价

95 个代理跑下来，真正能站住脚的发现只有六条左右，且大多是**单一团队的个人经验**（证据强度标"低"或"中"），没有一条达到"多个独立案例交叉印证"的强度。这本身就是一个值得记住的结论：**"如何做出一份好论文"这件事，互联网上没有可靠的系统总结，能抄的"成功学"公式基本都是不可信的**。真正可靠的信息反而是本仓库已经验证过的官方一手材料——[13-国赛CUMCM/官方评阅内幕](../13-国赛CUMCM/官方评阅内幕(2024年会议纪要).md) 里评委的原话、[12-模型与算法策略](../12-模型与算法策略/README.md) 的"拒绝套路化"、本文件 README 的评审机制——这些比任何"经验帖"都更值得信任。好论文最终还是靠你们自己在 11 月的模拟赛里真刀真枪走一遍，摔了跟头再总结，而不是照抄网上的成功学公式。
