# 摘要（Summary Sheet）写作指南

> 为什么单独成文：官方原话"评委对摘要给予很大权重，获奖论文往往正是凭摘要质量与其他论文区分开"（已验证）；初审每篇约 10 分钟，摘要写砸=直接出局（社区评委视角文章）。

## 1. 定位

- 竞赛摘要是**详细摘要**，可接近一页（远长于期刊 200 词摘要），12 号字、一页内、PDF 第一页；
- 它不是"介绍"，是**全文的浓缩成果展示**：评委看完摘要应当已经知道你们做了什么、怎么做的、得到了什么数。

## 2. 结构模板（五段式）

```
段1 问题重述（2-3句）：用自己的话说清题目要解决什么，不抄题面。
段2 方法总览（3-4句）：针对每一问，我们建立了什么模型（点名模型），
     一句话说明为什么选它。
段3 关键结果（4-6句，最重要）：每一问的核心数值结论——
     "模型预测X为__，置信区间__；对比发现__方法使__%的淘汰结果改变"。
     必须有具体数字，空话是初审出局的头号原因。
段4 检验与稳健性（2句）：敏感性分析显示__；模型在__扰动下结论稳定。
段5 亮点句（1-2句）：本文最大的创新/贡献是__。
```

## 3. 自检清单

- [ ] 有具体数字吗？（没有数字的摘要 = 平庸摘要）
- [ ] 每一问都覆盖了吗？
- [ ] 点名了模型吗？（评委扫一眼就要看到方法关键词）
- [ ] 有"为什么可信"（验证/敏感性）吗？
- [ ] 一页以内、无姓名校名？
- [ ] 非本队的人读完能复述你们做了什么吗？（找不参赛的同学测试）

## 4. 写作流程（赛中 D4 下午执行）

1. 全文完稿后写（不是先写摘要）；
2. 建模手列"每问一句话结论+关键数字"清单 → 写作手成文 → AI 润色语言（留痕）→ 三人轮读各改一遍；
3. 朗读测试：读起来磕绊的句子重写；
4. 对照本清单逐项打勾。

## 5. 反面清单（评委视角文章总结的初审出局特征）

- 通篇"we built a model to analyze..."却不说是什么模型、得了什么数
- 把题目背景复述半页（评委比你熟题目）
- 数学符号堆砌（摘要里不应出现推导）
- 明显的 AI 生成腔（空洞的 "comprehensive framework" / "valuable insights" 连用）——摘要建议以人工写作为主
- 最后一小时赶出来的错别字和语法错误

## 6. 训练方法

备赛期每精读一篇 O 奖论文，就把它的摘要按"五段式"拆一遍：标出问题重述/方法/数字结果/检验/亮点各在哪句。拆 5 篇后自己写的摘要会有质的变化。模拟赛必须完整练一次摘要流程。
