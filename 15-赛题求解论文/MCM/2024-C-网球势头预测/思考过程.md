## 一、题目理解：这道题到底在问什么

先通读官方题面 `2024_MCM_Problem_C.md`。拆解后题目其实包含四个递进的子任务，而不是一个单一的建模题：

1. **刻画"势头"（momentum）**：给出一个模型，能对比赛中"谁打得更好、好多少"给出定量刻画，并可视化"flow of play"。题目特别提醒：发球方赢下该分的概率天然更高，这是一个必须显式处理的混杂因素（confounder），不能放任其污染"势头"的定义。
2. **检验教练的怀疑**：教练认为所谓势头swing只是随机噪声（run of success 纯属随机）。这是一个统计假设检验问题，不是建模题——用我的势头指标去反驳或支持教练的说法。
3. **势头拐点的预测**：找出能提前predict"flow of play即将变化"的先兆指标（predictive/前瞻性任务，区别于任务1的"事后刻画"）。并且要求"给即将对阵新对手的球员提建议"。
4. **泛化检验**：在其他比赛上测试模型表现，讨论模型对女子比赛、其他场地、其他运动（乒乓球）的可推广性。

这个拆解很重要，因为很多人会把任务1和任务2的模型混为一谈——实际上"如何定义势头指标"（描述性统计/信号处理问题）和"势头是否显著存在"（统计推断问题）是两件事，用同一套构造出的指标做两件事时，容易在检验环节自我循环论证（circular reasoning，比如"我定义的指标显示有波动，所以势头存在"这种同义反复）。这是我在设计检验方案时特别小心规避的一点，具体见下文"遇到的问题与调整"。

另外题目明确点出发球优势（"the player serving has a much higher probability of winning the point/game"），这是官方给的强提示：势头指标必须相对于"发球期望胜率"做残差化，否则会把"轮流发球带来的比分波动"误判成"势头"。这是本题最核心的建模陷阱，我在后面的随机性检验里就踩了一次坑又爬出来了（见下文）。

## 三、候选建模方案对比

对"任务1：势头刻画"，我列了三个候选方案：

**候选A：累计分差 / 局分差**（最朴素）
直接用 `p1_points_won - p2_points_won` 或者局分差的滚动斜率作为"势头"。
- 优点：零参数、零假设、极简单。
- 缺点：完全没有处理发球优势——一个人只要轮到自己发球的4分里赢3分，分差就会规律性抬升，这纯粹是发球结构造成的，跟"momentum"无关。**直接违反题目的明确提示**，第一时间被我否决。

**候选B：胜率增量法（Win Probability Added, WPA）**——体育分析里的标准做法
先建立一个"给定比分状态、发球人是谁"的比赛获胜概率模型（通常用马尔可夫链，从point级别一路滚动算到game/set/match级别的win probability），然后势头 = 每一分打完后 WP 的变化量，做平滑。这是很多正式发表的网球分析论文采用的方法，也更"教科书"。
- 优点：理论成熟，能自然把杠杆（leverage，比如破发点的分量远大于40-0时的普通分）内生到模型里，不需要我手工设计权重。
- 缺点：⚠️ **存在逻辑循环的风险**：要算"胜率"，通常要假设每一分独立同分布（i.i.d.，即无记忆的点胜率模型），这本身就是在**假设不存在势头**的前提下去计算胜率，然后再用这个"无势头假设下"算出来的WP去反过来检验"势头是否存在"——这是有循环论证嫌疑的。而且完整实现一个逐分转移的马尔可夫链WP模型工作量较大，31场比赛每场都要重新拟合game/set/match转移概率，对一个人一天内完成的demo级别工作而言性价比不高，且对教练/读者而言"黑箱感"更强（难以解释"63.2%"这个概率具体是怎么来的）。
- 决定：**记录为更严谨的备选方案，但本次不采用**，改用候选C，并在论文的Strengths/Weaknesses里明确承认候选B是更优的备选。

**候选C：发球期望基线 + 杠杆加权 + EWMA 平滑**（最终采用）
不去算完整的"获胜概率"，而是只对"这一分谁赢"这一件事定义一个"期望"（用一发/二发条件下的发球方经验胜率作为期望），把"实际结果 - 期望结果"定义为这一分的"意外程度"（surprise），再按该分的杠杆（是否破发点/是否平分/是否决定性的一分）加权，最后用指数加权滑动平均（EWMA）做时间平滑，得到势头指数。
- 优点：不需要假设点胜率的完整马尔可夫模型，避免了候选B的循环论证问题；计算透明、可解释性强（每一步都有清晰的业务含义）；参数少（只有一个平滑系数α和一组杠杆权重），便于做敏感性分析。
- 缺点：杠杆权重是我手工设定的规则（不是从数据中估计出来的），存在一定主观性——这个我在敏感性分析里专门检验了"去掉杠杆权重"是否会改变结论（结果：影响很小，见下文）。

**候选D（预测子任务专用）：LSTM/序列深度模型**
用于"任务3：预测拐点"时曾考虑过用RNN/LSTM直接对point序列建模做拐点预测。
- 决定放弃：31场比赛、单场只有120~340个点，训练样本量对深度模型而言太小，容易过拟合，而且不可解释（教练根本没法从LSTM权重里得到"应对建议"）。改用逻辑回归+梯度提升树（可解释、能给feature importance）。

最终选定：**势头刻画用候选C**；**随机性检验单独设计**（见下节，这是本题的一个关键调整）；**拐点预测用逻辑回归+GBT**。

## 四、关键假设及其论证

- **A1 分是最细粒度的时间单位**：数据本身以point为行，题目也是"as points occur"——顺理成章。
- **A2 发球方经验胜率（按一发/二发拆分）是"无势头假设下"每一分的合理期望**：这是题目明示的混杂因素，用全体31场比赛的整体经验频率做基线，而不是让每场比赛自己算基线，是为了保证基线的稳定性（单场三百个点，二发的样本更少，容易噪声很大）。
- **A3 杠杆权重（破发点+1，平分/AD+0.5，决胜分+0.5）**：网球评论员和教练本身就习惯说"关键分动能"，这是行业常识的形式化，而非我凭空捏造；具体数值敏感性见后。
- **A4 EWMA固定α做短期记忆衰减**：这是"势头"概念本身隐含的假设——势头应该是"最近表现"主导、而不是整场比赛平权，用EWMA是信号处理里最简单、最常用、且单参数可控的平滑方式；α on敏感性分析里从0.1扫到0.7。
- **A5 结果仅代表2023温网男单第3轮及以后**：数据本身的覆盖范围决定了基线概率、特征重要性等具体数值不能直接套用到女子比赛/其他场地/其他项目，这点在论文的"Strengths and Weaknesses"和最后一节里专门讨论，不外推编造数字。

## 五、计算过程（真实代码与真实输出）

### 5.1 环境准备
本机没有预装pandas等库，用venv装了pandas/numpy/scipy/matplotlib/scikit-learn（均为真实pip安装，见Bash记录）。

### 5.2 势头指数构造与可视化（针对题目点名的2023-wimbledon-1701 决赛）

核心公式（写入`momentum_model.py`）：

```
expected_p1(t) = p_serve[serve_no]      if server == 1
               = 1 - p_serve[serve_no]  if server == 2
surprise(t)    = 1{point_victor==1} - expected_p1(t)
leverage(t)    = 1 + 1{break point} + 0.5*1{deuce/AD} + 0.5*1{set-deciding point}
weighted(t)    = surprise(t) * leverage(t)
M(t)           = alpha * weighted(t) + (1-alpha) * M(t-1),   M(0)=0,  alpha=0.30
```

跑在Alcaraz(P1) vs Djokovic(P2)决赛上（334分），按盘统计平均势头：

```
Set 1 mean momentum P1: -0.114   (实际比分 P1 games=1 P2 games=6 → Djokovic 6-1 大胜)
Set 2 mean momentum P1:  0.017   (实际 6-6 抢七，Alcaraz 7-6 险胜)
Set 3 mean momentum P1:  0.216   (实际 6-1，Alcaraz 大胜)
Set 4 mean momentum P1: -0.082   (实际 3-6，Djokovic 大胜)
Set 5 mean momentum P1:  0.047   (实际 6-4，Alcaraz 险胜)
```

这个结果和题面文字描述的"戏剧性五盘"完全吻合（正负号、大小关系都对得上），说明这个指标至少通过了"表面合理性"检验（face validity）——这是我判断"这个模型没有明显做错"的第一道关卡。可视化图保存在 `figures/momentum_1701.png`，图中能清楚看到每一盘的红蓝区域切换与官方叙事一致。

### 5.3 教练"纯随机"假设检验——一次真实的返工

**第一次尝试（失败，被我自己发现并纠正）**：
最初我想："既然要控制发球优势，那就对 surprise(t) 取符号，再做游程检验（runs test），这样应该能剔除发球轮换的影响了吧。" 代码写完跑出来一看：

```
z_raw (对原始 point_victor 序列的游程检验) 与
z_adj (对 sign(surprise) 序列的游程检验)
——两组z值逐场比对，完全相同！
```

意识到问题：`surprise(t) = 实际 - 期望`，而实际只能是0或1，只要期望严格落在(0,1)区间内（它必然是，因为是个概率），那么 `sign(surprise)` 就**恒等于**"P1是否赢下这一分"的原始0/1指示——这是一个数学上必然的退化结果，跟我用什么期望值毫无关系。也就是说：**对残差取符号再做游程检验，根本没有做到"控制发球"，等价于直接对原始输赢序列做游程检验**。这是我在实现过程中一开始没想清楚就直接编码,跑出诡异的"两个不同方法结果完全一样"才意识到的逻辑漏洞。

**调整后的方案**：不对残差取符号，而是直接用**连续型**的 serve-adjusted surprise 做"滞后一分的持续性相关"检验：把 t-1 时刻发球方视角下的（滞后一分）势头 `prev_momentum_server` 与 t 时刻发球方视角下的即时"意外程度" `outcome_server` 做相关，并用**置换检验**（在每场比赛内部打乱outcome序列2000次，保持每场比赛自身的边际分布和发球结构不变）得到经验p值，这样才是真正"剔除发球结构"后的持续性检验。

同时也**保留了朴素游程检验作为对照**，用来在论文里演示"不做发球调整会得到什么样的（有偏）结论"，这本身也是一个有教学价值的对比。

真实运行结果（`randomness_test.py`）：

```
=== Naive point_victor runs test (未调整发球) ===
share of matches with p<0.05: 0.1613  (16.1%, 5/31场)
mean z (negative = 游程数少于随机期望 = 表面上"扎堆"): -0.986
```

朴素检验显示"表面上"有轻微聚集倾向（z均值为负），但绝大多数场次不显著，而且这种聚集本身可能就是发球轮换造成的假象——正因为一局约4分里同一人当发球方，容易连续拿分，这在游程检验里天然表现为"游程更少"，跟真正的momentum无关。

真实运行结果（`momentum_persistence_test.py`，控制发球后的持续性检验）：

```
=== 逐场 r(滞后势头, 当前意外程度) ===
mean r across 31 matches: 0.0379
share of matches with r>0: 0.742        (23/31场方向一致为正)
share of matches with permutation p<0.05: 0.065  (约2/31场单场显著)

=== 汇总全部31场、7284个点 ===
Pooled correlation r = 0.0356
Pooled permutation p-value = 0.0029985...
```

**这是本题最重要的结论**：单场比赛内部，势头持续性相关系数普遍很弱（|r|~0.04量级），只有2场（1406, 1503）在单场内部就统计显著；但74.2%的场次符号一致为正，**把全部7284个点汇总起来看，这个微弱但方向一致的效应变成统计显著（p=0.003）**。也就是说：教练"某一场比赛里能不能看出明显的势头模式"这个日常直觉基本是对的（单场噪声太大，几乎测不出来），但从统计学角度讲，"完全没有momentum、纯属随机"这个更强的说法是站不住脚的——数据里存在一个小但真实的、跨场一致的势头持续性效应。这是一个**"部分支持、部分反驳"**教练说法的、比较诚实的结论，而不是简单地说"教练错了"或"教练对了"。

### 5.4 拐点预测模型

定义"拐点"（swing）= 未来K个点内势头符号发生翻转（K=4为主设定）。特征包括：当前势头绝对值、势头方向、连胜长度、是否破发点、是否平分、拍数(rally_count)、体能差（累计跑动距离差）、发球方、盘/局进度、近8分一发成功率滑窗。

训练/测试切分按**轮次做时间外推**：第3、4轮（24场，5707个点）训练，第5-7轮及决赛（7场，1577个点）做测试——这样比随机切分更贴近题目"用其它比赛测试模型"的要求。

真实结果（`swing_prediction.py`）：

```

Test accuracy @0.5: 0.6563     (基线多数类准确率: 0.6760)

=== Gradient Boosted Trees ===
Test AUC: 0.6811
Test accuracy @0.5: 0.7108

Feature importances (GBT):
abs_momentum                0.599   ← 最主导
fatigue_diff_p1_minus_p2    0.167
server_is_p1                0.067
game_no                     0.055
roll_first_serve_in_rate    0.029
run_len                     0.024
...
break_pt                    0.000
```

逐场测试集AUC在0.55~0.761之间波动（1701决赛本身AUC=0.641），说明模型有一定预测力但远非完美，且**场次间差异较大**——不同对手组合下"势头可预测性"本身就不一样。

**自我批判发现**：`abs_momentum`（当前势头的绝对值）是最重要的特征，但仔细想想，这在很大程度上是一个**机械性（mechanical）而非因果性**的结果——EWMA是一个有界的滑动平均序列，当前值离0越远，短期内"回归零点/翻转符号"就越需要更大幅度的反向冲击，天然就更不容易翻转，这是滑动平均本身的数学性质（均值回归），而不是"物理击球"意义上的momentum成因。相比之下，`fatigue_diff`（体能差）是相对更有实质意义的解释变量，值得在论文里重点讨论并诚实标注前者的局限性。

### 5.5 敏感性分析

**α（EWMA平滑系数）敏感性**（对持续性相关系数r的影响）：

```
alpha=0.10  pooled r=0.0465
alpha=0.20  pooled r=0.0383
alpha=0.30  pooled r=0.0356
alpha=0.50  pooled r=0.0355
alpha=0.70  pooled r=0.0358
```
结论：r的量级在0.035~0.047之间稳定，α取值不改变"存在微弱但显著的持续性"这一定性结论。

**K（拐点定义窗口）敏感性**（对预测AUC的影响）：

```
K=2  logit_AUC=0.6884  gbt_AUC=0.6903
K=4  logit_AUC=0.6761  gbt_AUC=0.6811
K=6  logit_AUC=0.6696  gbt_AUC=0.6675
K=8  logit_AUC=0.6806  gbt_AUC=0.6669
```
AUC基本稳定在0.67~0.69区间，K的选择不是结果的主要驱动因素。

**杠杆加权 on/off 敏感性**：

```
Set 3: with-leverage mean=0.216   no-leverage mean=0.158
相关系数(带杠杆 vs 不带杠杆的势头序列) = 0.9538
```
去掉杠杆权重后势头序列整体高度相关（r=0.954），但对高杠杆盘（如第3盘）的信号幅度有明显放大作用（0.158→0.216），说明杠杆加权确实起到了"放大真实信号、不改变整体走势"的作用，是一个有价值但非决定性的改进。

**零穿越频率的额外检查**：顺手统计了每场比赛势头符号"翻转"的原始频率：

```
mean n_crossings per match: 65.97 (SD 20.05), 即约28次/百点
```
说明如果直接把"势头符号翻转"当作拐点，噪声极大（平均每3.5个点就翻一次），这也是我选择"未来K点内是否翻转"而不是"下一点是否翻转"作为拐点标签的原因——单点级别的翻转绝大多数是噪声。

## 六、遇到的问题与调整（汇总）

1. **游程检验的退化陷阱**（见5.3）：最初设计的"残差符号游程检验"在数学上等价于原始输赢序列检验，完全没起到控制发球的作用。发现后改用连续残差+置换检验的持续性相关分析，这是本次求解过程中最大的一次方案调整。
2. **WPA候选方案的循环论证风险**（见三、候选B）：意识到"先假设无势头去算胜率，再用这个胜率反过来验证有无势头"存在逻辑循环，因此放弃了看起来"更教科书"的WPA方案，改用更朴素但逻辑自洽的期望-残差法。
3. **深度学习方案的样本量问题**：31场比赛、单场几百个点，试图用LSTM会严重过拟合且不可解释，改用逻辑回归+梯度提升树。
4. **缺失值处理**：没有为了"用上所有列"而强行插补return_depth（缺18%）等列，只在rally_count（缺失很少）上做了中位数填充，且在论文里明确注明。

## 七、自我批判：局限与重来会怎么改

1. **杠杆权重是手工设定的规则而非数据驱动的估计**：虽然做了on/off敏感性分析证明不改变定性结论，但更严谨的做法是用真实的WPA/马尔可夫链模型（候选B）从数据中估计每个比分状态的真实杠杆，而不是我拍脑袋定的"破发点+1、平分+0.5"。时间充裕的话应该实现候选B并与候选C做正式对比。
2. **持续性检验的置换检验方式仍是一种简化**：置换检验打乱了时间顺序但没有对"发球局内在的时间结构"（比如同一局内前几分和后几分本身状态相关）做更精细的分块置换（block permutation），存在略微低估/高估显著性的可能，正式论文应该用block bootstrap之类更严谨的方法交叉验证。
3. **拐点预测模型最重要的特征存在"机械性"嫌疑**（见5.4），这削弱了"预测拐点"结论的因果解释力，只能说是"统计关联"，教练建议部分需要谨慎措辞，不能过度声称"体能差是造成拐点的原因"（只能说"是能提前观察到的一个信号"）。
4. **样本仅限2023温网男单第3轮起的31场比赛**，任何具体数字（发球基线概率、特征重要性、AUC）都不能不做重新估计直接套用到女子比赛/红土硬地/乒乓球，这一点论文里必须反复强调，不能因为篇幅或"看起来完整"而暗示泛化性。
5. 如果重来，我会：(a) 补充实现候选B（WPA/马尔可夫链）作为交叉验证；(b) 用block permutation取代简单shuffle；(c) 尝试对"势头susceptibility"做球员级别的固定效应分析（同一球员在不同比赛里是否表现出一致的"抗势头/易受势头影响"特质），这需要更多场次数据支撑，当前31场对每个球员而言样本太少，暂未展开。
