# 15 赛题求解论文

> ⚠️ **重要声明**：本目录全部 47 篇论文均由 AI 智能体（Claude）通过多智能体并行工作流生成，
是**数学建模训练与研究参考示例**，不是真实提交的参赛作品。根据 COMAP/CUMCM 官方 AI 使用政策，
直接把 AI 生成的论文当作参赛作品提交属于违规行为（详见 [02-AI使用政策](../02-AI使用政策/README.md)、
[13-国赛CUMCM](../13-国赛CUMCM/README.md)）。每篇论文开头都重复了这条声明。

## 这是什么

用 47 个并行 subagent，把仓库里收录的**全部历年真题**（[08-历年真题](../08-历年真题/README.md) 的 7 道美赛 C 题 + 2026 F 题、
[13-国赛CUMCM](../13-国赛CUMCM/README.md) 真题库的 39 道国赛题）各做了一遍完整建模，每题产出两个文件：

| 文件 | 内容 |
|---|---|
| `思考过程.md` | **留痕**：题目理解、候选建模方案的取舍论证、实际执行的代码与真实输出、中途遇到的问题与调整、自我批判 |
| `论文.md` | 正式论文格式（美赛英文 Summary Sheet 结构 / 国赛中文摘要结构），含免责声明 |

**数据处理原则**：有官方数据附件的题目，一律用 Python 实际读取计算，结果是真实跑出来的；
没有数据或数据体量过大无法加载的题目，一律用明确标注为"示例/说明性"的数值，绝不伪装成真实计算结果——
这是延续 [14-AI在数学建模中的应用与局限](../14-AI在数学建模中的应用与局限/README.md) 里反复强调的纪律。

1. **学建模思路，不要抄论文**：重点看 `思考过程.md`——里面记录了真实的方案取舍过程（为什么选这个模型不选那个）、遇到 bug 怎么调整、结果不合理时怎么发现的，这些是备赛真正有用的部分。
2. **不要直接拿 `论文.md` 去交作业**：会被查重系统当场揪出来（本仓库 [官方评阅内幕](../13-国赛CUMCM/官方评阅内幕(2024年会议纪要).md) 已经证实"网络模板/AI生成"是雷同率高企的头号原因），且违反 AI 使用披露规则。
3. **标注⚠️示例数值的题目**要格外小心，论文里的具体数字是为了验证模型逻辑自洽而生成的，不是真实结论。
4. 部分题目的 `思考过程.md` 里记录了真实发现的代码 bug 和方法论纠错过程（如 2016-C 的预算未花完、2025-C 田径项目份额模型的失败重来）——这类"如实记录失败"的部分尤其值得读，比看一份"完美"的论文更能学到东西。

## 生成方式

47 个 subagent 并行执行（Workflow 编排，concurrency cap ~16），单次运行约 2.2 小时、消耗约 640 万 token、2198 次工具调用（Read/Bash/Write）。每个 agent 独立读题、读数据、用 Bash 跑 Python 做真实计算、写两份文件，过程互不干扰。
