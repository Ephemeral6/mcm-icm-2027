## 一、题目理解：这道题到底在问什么

先读题（`/Users/ephemeral/美赛/13-国赛CUMCM/真题2020/2020_D题.md`）。拆解后，题目的核心是一个**逆向工程 / 参数化重建**问题：

- 轮廓仪测得的是离散点列 `(x, z)`（探针沿 x 匀速滑行，z 是探针高度）；
- 已知先验：工件真实轮廓**只由直线段和圆弧段拼接而成**（题目原话）；
- 要做的事情是：给一堆"粗糙不平"（探针沾污、缺陷、位置误差导致的噪声）的点云，**自动**把它分解成一段段直线/圆弧图元，并报出槽宽、圆弧半径、圆心距、弧长、直线段长度、斜线长度与夹角、"人"字形高度这些具体尺寸；
- 问题2 在此基础上叠加了一个"位姿变化"：同一工件倾斜放置后再测一次，要求先估计倾斜角、做水平校正，再重复问题1 的标注流程，并比较两次结果的差异——这本质是在检验算法对**刚体变换**的鲁棒性；
- 问题3/4 是把"单次测量"扩展到"多次测量拼接与局部精测融合"，需要附件2～4 的数据。

**本次任务的数据边界**：任务说明中明确只提供了附件1（工件1的水平/倾斜测量数据），附件2～4没有提供。所以我把主要精力和真实计算都放在问题1、2上，问题3、4只给出方法论延展，不编造数字。

## 二、候选建模方案对比

拿到 143043 个点（level）之后，第一反应是"怎么把点云自动切成直线段和圆弧段"。想了三种路子：

### 方案A：全局多项式/样条拟合 + 求导定位特征点
思路：对整条 z(x) 做高阶多项式或样条拟合，然后用一阶、二阶导数的过零点定位转折点。
**放弃理由**：轮廓在不同区间是本质不同的几何类型（直线 vs 圆弧），一条全局多项式或样条无法在段与段之间保证曲率的正确阶跃/恒定，容易震荡（Runge 现象），而且"多项式阶数"这个超参数没有物理意义，纯粹是凑合。

### 方案B：机器学习分类器（如 BP 神经网络）逐点打"直线/圆弧"标签
思路：把每个点的局部窗口特征喂给一个分类器。
**放弃理由**：这是本题最容易掉进去的"熟练工陷阱"——遇到"自动标注/自动分类"问题就想上神经网络。但仔细想，这个问题的机理是**清晰的解析几何问题**：直线的曲率恒为0，圆弧的曲率恒为1/R，二者可以用曲率的符号和幅值直接、确定性地区分，不存在"学习"的必要，也没有足够标注数据去训练一个可靠的分类器。用黑箱模型解决一个有解析解的问题，是不必要的复杂度，也不符合"先想清楚机理再选模型"的要求。

### 方案C（采纳）：曲率驱动的分段识别 + 分段解析拟合 + 图元交点重构
思路分三层：
1. **分类**：对点云做 x 方向分箱去噪，用 Savitzky-Golay 滤波估计一阶导（斜率）和二阶导（曲率），按 `|曲率|` 和 `|斜率|` 的阈值把数据点分类为 `flat`（平线，基准面）、`line`（斜线段）、`arc+`/`arc-`（凹/凸圆弧段），合并短片段去噪；
2. **精拟合**：对每个足够长的"核心段"（trim 掉端点附近的过渡区，理由见下）分别用**总体最小二乘（TLS/主成分方向）拟合直线**、**几何最小二乘拟合圆**（Kåsa 代数解做初值 + `scipy.optimize.least_squares` 精修）；
3. **重构断点**：图元与图元之间的边界点，不用原始数据的边界，而是用**相邻两个图元的解析交点**（线-线交点解线性方程组；线-圆交点解二次方程）反推出来。

**为什么选C**：
- 直接对应轮廓仪的物理成像过程（直线曲率0、圆弧曲率恒定，是精确的解析判据，不是近似）；
- 参数意义明确，每一步都可解释，可以逐段核对拟合残差；
- 用解析交点代替原始端点，天然能够**吸收接触式探针有限半径造成的圆角效应**——这一点是我在探索数据过程中才发现其必要性的（见下文"遇到的问题"）。

也考虑过更"自动化"的 RANSAC split-and-merge（不预设窗口边界，让算法自己找断点），这在工程上会更鲁棒、更适合噪声更大或没有先验分段信息的数据。但本题数据信噪比很高、图元结构清晰，用曲率阈值分类已经足够干净地找到所有分段边界；出于时间预算考虑没有实现完整的 RANSAC 版本，这是本方案的一个可以改进之处（见"自我批判"）。

## 三、关键假设及其论证

**假设A1：探针沿 x 匀速滑行，测得的是单值函数 z(x)。**
论证：实测 `level` 表中 dx 的统计——

```
dx stats: count 143042, mean 0.000500, std 0.000011, min 0.000321, max 0.001001
```

即便在最陡的凹槽侧壁处 dx 也保持在 ~0.0005mm 量级、没有出现"回头"（x 减小）的采样点，说明可以放心把数据当函数图像处理，不存在多值悬空问题。

**假设A2：工件真实轮廓仅由直线和圆弧构成**——这是题目给定的先验，直接采用。

**假设A3：探针存在一个近似恒定的接触半径，导致所有原本的"尖角"连接处在原始数据中呈现出同一量级的小圆角。**
这是我在分析凹槽底部数据时意外发现、进而修正建模思路的关键假设，论证过程见下节"遇到的问题"。支持证据：三个凹槽的槽壁-槽底转角处、"人"字尖峰顶点处、以及每个圆弧与基准面衔接处，都独立地表现出量级 0.1～0.6mm 的曲率突变区（不是设计尺寸，是测量伪影）。

**假设A4："level"状态下工件基准面近似水平。**
验证：把所有被识别为 `flat` 的长直段合并后做总体最小二乘拟合，得到

```
[baseline] slope_angle = -0.0154 deg
```

残余倾角可以忽略，说明"水平"标签基本可信，可以把这条线当作零角度参考。

**假设A5："down"状态是一次刚体位姿变化（绕某轴转动 + 平移），而不是局部形变。**
验证：在 `down` 数据里，被识别为基准面的 8 段"flat/line"（分布在 x=48.9~119.3 的不同位置）斜率高度一致：

```
-0.1303, -0.1307, -0.1318, -0.1309, -0.1313, -0.1348, -0.1310, -0.1311, -0.1317
```

标准差不到 0.002，远小于均值 0.131，强烈支持"整体一次性刚体旋转"这一假设，而不是逐段独立的位置误差。

### 4.1 读数据、看形状

```python
import pandas as pd
f = 'extract2020/CUMCM2020Problems/D/附件1_工件1的测量数据.xlsx'
xl = pd.ExcelFile(f)
print(xl.sheet_names)   # ['level', 'down']
```

`level` 143043 行，`down` 141584 行，列均为 `x, z`（单位 mm）。画出全貌图后，肉眼可辨认出：3个凹槽（V形深槽）、一个"人"字形凸棱、一对小圆弧（小凹坑+小凸包）、一对大圆弧（大凹坑+大凸包），全部通过一条基准平线相连——与题目图4描述的"工件1轮廓线"完全对应。

### 4.2 第一次尝试曲率分段——失败，噪声爆炸

第一版直接在原始高分辨率数据上用 `np.gradient` 两次求导估计曲率：

```python
zs = pd.Series(z).rolling(win, center=True, min_periods=1).median().values
dz = np.gradient(zs, x)
d2z = np.gradient(dz, x)
curv = d2z / (1+dz**2)**1.5
```

结果**在平坦区的曲率标准差高达 0.44～0.82**，而真实大圆弧的曲率幅值只有 ~0.25 —— 信噪比倒挂，完全无法分类。原因是：原始点间距只有 0.5 微米，而基线本身残余噪声标准差约 0.0016mm；对这种密采样噪声数据直接做两次数值微分，噪声会被放大 `1/dx²` 量级，任何"先平滑再微分"的顺序如果平滑窗口不够大，都压不住。

### 4.3 调整：先分箱去噪，再用 Savitzky-Golay 求导

把点云先按 x 方向分箱平均（`step=0.02mm`，即约40个原始点合并为1个，靠中心极限定理把噪声打下来），再对分箱后的粗序列用 Savitzky-Golay 滤波器直接估计一阶、二阶导数（比"先差分再平滑"数值上稳定得多）：

```python
xb, zb = bin_average(x, z, step=0.02)
dz  = savgol_filter(zb, win, 3, deriv=1, delta=dxb)
d2z = savgol_filter(zb, win, 3, deriv=2, delta=dxb)
curv = d2z/(1+dz**2)**1.5
```

用 win=21（对应约 0.42mm 平滑窗口）实测：

```
flat region: curv std 0.0898816
bigbump apex curv mean/std -0.2467 / 0.0643   (R≈4mm 对应 curv≈0.25，吻合)
bigdip  apex curv mean/std  0.2506 / 0.1101
```

信噪比一下从"倒挂"变成"平坦区噪声只有真实弧段曲率的 1/3～1/2"，可以分类了。进一步测试不同平滑窗口发现 **win=61（约1.2mm）时平坦区噪声最小**（curv std ≈0.009），而大圆弧曲率均值几乎不变（仍是0.25），这是分类用窗口的最终选择。

### 4.4 第二个坑：同号合并把相邻凹凸弧粘成一坨

只按 `|curvature|>阈值` 判定"是不是圆弧"、不看符号时，小凹坑和紧邻的小凸包（曲率符号相反）会被误判成"同一段圆弧"合并在一起：

```
arc x=[84.250,90.011] nbins=289   # 把小凹坑+短平段+小凸包全糊成一个 run
```

修正：分类时把 `arc` 拆成 `arc+`（凹, curv≥0）和 `arc-`（凸, curv<0）两个标签，只合并同号的相邻段。之后小凹坑/小凸包各自独立成段，问题解决。

### 4.5 第三个坑（也是最重要的发现）：所有"尖角"连接处都有一个共同量级的小圆角

用更细的窗口（win=11）重新检查"人"字尖峰顶点附近，发现曲率在 76.7~76.9mm 之间**从接近0陡升到 -1.65**（对应半径约0.6mm），然后迅速回落——说明这个理论上应该是"直线-直线尖角相交"的地方，实测数据里有一个小圆角。

进一步检查三个凹槽底部的圆弧拟合半径：

```
notch1 bottom arc: R=0.5026 mm
notch2 bottom arc: R=0.2870 mm
notch3 bottom arc: R=0.3202 mm
```

以及大/小圆弧与基准面衔接处也都有类似量级（约0.4~0.6mm）的"过渡小圆弧"（这从粗分类的输出里能看到，比如 `bigdip` 前后各有一段 0.46mm 长的 `arc-` 夹在 `flat` 和主体 `arc+` 之间）。

**调整**：这些"过渡小圆弧"很可能不是工件的设计尺寸，而是接触式探针有限球头半径造成的系统性钝化效应（凡是原本应该是尖角或非相切连接的地方，实测数据里都会因为探针无法测出"绝对尖锐"的角点而带上一个小圆角，这是接触式轮廓仪的已知物理限制）。于是把算法调整为：**对每个图元只用远离转折点的"核心点"做拟合**（每侧留出0.05~0.5mm 安全边距，具体见 `fit_level.py` 中的窗口定义），刻意避开这些过渡区，用拟合出的直线/圆弧**解析外推**得到理想的角点/切点，而不是直接用原始数据的端点。这个调整之后，同一工件三个凹槽的槽深计算值高度一致（见下），验证了处理方式是对的。

### 4.6 深度计算方式的选择：交点法 vs 圆弧最低点法（真实数据教会我的教训）

一开始我用两条槽壁直线的**交点**（假设V形槽底是理想尖点）计算槽深：

```
notch1 vertex(line-line) depth = 3.9816 mm
notch2 vertex(line-line) depth = 3.6413 mm
notch3 vertex(line-line) depth = 3.6994 mm
```

三个数字彼此不一致（3.64~3.98mm），而且都**比实测到的最深点还要深**（notch1 原始数据最小 z 只到 -4.7566mm，换算深度约2.98mm，但线-线交点算出的深度要到3.98mm——这是一个物理上"探针从未到达过"的虚拟点）。这提醒我：如果三个凹槽真的是同一设计深度，交点法给出不一致结果，说明这不是正确的建模方式。

于是换成"圆弧最低点法"：把槽底当作一段真实设计的小圆弧（半径就是拟合出来的 R），槽深 = 基准面高度 − (圆心z − R)：

```
notch1: depth(arc-bottom) = 2.9845 mm
notch2: depth(arc-bottom) = 2.9723 mm
notch3: depth(arc-bottom) = 2.9908 mm
```

三者高度一致（都在 2.97~2.99mm，标准差<0.01mm），强烈提示真实设计槽深就是 **3.00mm 左右**，三个槽只是槽宽和槽底圆角半径不同。这是本次探索中一次"模型选择被数据反驳、进而修正"的真实过程，我把它完整记录下来。

```
  shrink=0.0 R=0.9935   shrink=0.3 R=0.9829   (跨度约1.1%)
--- small_bump base=(87.9,89.12) ---

  shrink=0.0 R=4.0014   shrink=0.3 R=4.0082   (跨度<0.2%)
```

结论：**圆弧半径的拟合稳定性和"弧长跨越的点数/角度"正相关**——大圆弧（半径4mm，跨越约7~9mm弧长、上万个点）对窗口选择几乎不敏感；小圆弧（半径~1mm，跨越约1.5~2mm弧长、约2000~2600点）就明显敏感一些。这与最小二乘圆拟合的经典结论一致（弧段越短/曲率半径越大，圆心和半径的病态程度越高），也解释了4.8节里小凹槽底部圆角（R更小，约0.3mm）相对误差更大的现象。

## 五、遇到的问题与调整（汇总）

1. **直接数值微分噪声爆炸** → 改为"分箱去噪 + Savitzky-Golay 直接估计导数"。
2. **曲率符号未区分导致相邻异号圆弧被错误合并** → 引入 `arc+`/`arc-` 双标签。
3. **误以为凹槽底部、人字尖峰是理想尖角，用直线-直线交点算深度/高度，三个"应该相同"的凹槽算出的深度却互不一致，且比实测最深点还深** → 通过检查发现所有转折点处普遍存在约0.1~0.6mm量级的小圆角，判断这是探针接触半径造成的测量伪影，进而改用"圆弧拟合+核心点trim+解析外推"的方式处理所有转折点，槽深结果立刻收敛到高度一致的~2.98mm。这是本次探索中最重要的一次"模型被数据反驳后修正"的经历。
4. **小半径圆弧拟合对窗口选择敏感** → 没有强行"修正"这个问题（它是真实的统计性质，不是bug），而是在敏感性分析里如实量化了这个不确定度，并在论文里提示读者小圆角类参数的可信度低于大圆弧类参数。

## 六、自我批判：这个解答的局限，如果重来会怎么改

1. **分段窗口是半自动的**：最终每个特征的"核心拟合窗口"边界是我通过反复画图、看粗分类输出、手工圈定+微调得到的，不是一个端到端全自动、不需要人工介入的算法。如果重来，应该把 `classify_runs` 的输出直接程序化地转换成拟合窗口（自动trim固定比例、自动过滤过短的run），做成一个不需要人看图就能跑通任意工件的通用函数。当前版本更像是"半自动化的验证性实现"而非"生产级自动标注系统"，这与题目"自动标注"的本意还有差距。
2. **没有做探针半径的显式反卷积**：只是"绕开"了转折点附近被钝化的数据，没有真正建立探针几何（球头半径 r_p）与测得轮廓的正演关系再做反演。如果能进一步估计出一个全局共用的 r_p（三个凹槽的圆角半径其实并不完全相同：0.50/0.29/0.32mm，如果是纯粹的探针半径效应，理论上应该趋于同一个值，说明凹槽本身也有设计圆角，探针半径效应和设计圆角叠加在一起，没有彻底分离），标注的物理意义会更严谨。
3. **未使用附件2~4**：问题3、4要求的"多次测量拼接"和"局部精测融合"完全没有真实数据支撑，论文里只给出方法论，没有真实数字，这是相对于完整国赛提交的最大缺口。
4. **圆拟合用的是几何最小二乘（等权重）**，没有考虑数据点在弧上分布不均匀、噪声方差可能不是常数等情况，更严谨的做法可以引入加权最小二乘或M-估计等鲁棒方法。
5. **没有做正式的假设检验**（比如用F检验判断某段数据究竟更像直线还是极大半径圆弧），分类阈值（曲率0.12、斜率0.05）是通过试验几组数值挑出来的，没有从统计意义上论证其最优性。
