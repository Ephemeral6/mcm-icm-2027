## 一、题目理解：这道题到底在问什么

先把题面拆解成一句话："超市每天凌晨定好当天要进多少菜、卖多少钱，用的是历史流水去猜今天的需求，而不是提前知道确切的到货价格和到货量。"

四问的关系其实是一个"由粗到细"的递进结构：

- **问题1**：先看数据本身长什么样——6个品类、251个单品的销量分布规律和相互关系。这是纯描述性/探索性分析，是后面建模的地基。
- **问题2**：以品类为粒度做补货+定价决策，目标是收益最大化。这里"成本加成定价"和"销售总量"的关系，本质上是要建立一条需求曲线 Q(P)，再结合成本做定价优化。
- **问题3**：从品类下沉到单品，多了两个硬约束——可售单品数27-33个、单品最小陈列量2.5千克——这是一个带约束的资源分配问题（品类总量→单品分摊），而不是重新做一次独立的需求建模。
- **问题4**：跳出模型本身，反思"还缺什么数据"。这一问最好的答案来源，就是老老实实把前三问建模过程中"卡壳"或"只能用代理变量凑合"的地方如实列出来——而不是泛泛地说"要采集天气数据"这种正确的废话。

拿到附件后第一件事是读数据字典，而不是急着套模型：附件1是单品-品类映射表（251个单品，6个品类），附件2是2020-07-01至2023-06-30共878503条销售流水（单品级，含"退货"负值），附件3是同期251个单品的批发价格（55982条），附件4只给了**品类级**（不是单品级）的平均损耗率。这个"附件4只到品类粒度"的细节很重要，后面问题3做单品定价时会因此受限，也是问题4回答的重要素材之一。

## 二、候选方案对比

### 问题1：分布规律与关联关系怎么分析

- **方案A（本次采用）**：分层描述——(1) 偏度/对数偏度刻画分布形状；(2) 星期效应、月度季节性指数刻画时间规律；(3) 品类间/单品间 Pearson 相关系数刻画关联关系，并做"去趋势"处理避免被共同的季节趋势"虚增"相关性。
- **方案B（考虑后放弃）**：对每个品类逐一做分布拟合检验（如 K-S 检验拟合对数正态/Gamma分布），追求"报出精确分布族"。放弃原因：题目问的是"分布规律及相互关系"，落点在业务解释力（旺季淡季、周末效应、品类联动）而非分布族的精确统计检验；且日销量还叠加了促销、断货等非随机扰动，用单一参数分布硬拟合反而会掩盖真正有业务价值的规律（比如周末效应）。**结论：保留偏度/对数变换作为形状描述，把精力放在更有解释力的时间规律和相关关系上。**
- **方案C（考虑后放弃）**：用聚类算法（如层次聚类）对246个单品做"品类再发现"，看聚类结果是否还原附件1给的6分类。放弃原因：这属于"为了炫技而做"，附件1已经给定分类且业务含义明确，重新聚类对回答问题1没有增量价值，时间上不划算。

### 问题2：品类补货+定价怎么建模

这是本题的核心难点，也是本次求解中**唯一发生了实质性方案调整**的地方，过程记录如下。

**最初方案（对数-对数需求曲线 + 经典垄断定价公式）**：
经济学教科书里最常见的做法是假设需求为常弹性形式 Q = A·P^(-ε)，通过 ln(Q) ~ ln(P) 的线性回归估计弹性 ε，再用"边际成本加成"公式求最优价 P* = ε·c'/(ε-1)（c' 为经损耗调整后的有效成本）。这个公式简洁、在教材和往年赛题解答中都很常见，是"标准动作"。

我先按这个思路把回归跑了一遍（控制星期、月份、时间趋势后）：

```
=== 分品类 对数-对数 需求弹性回归: lnQ ~ lnP + 星期 + 月份 + 时间趋势 ===
水生根茎类: 弹性ε(需求对价格) = 0.356 (se=0.086, p=0.0000), 模型R2=0.600, n=1085
花叶类: 弹性ε(需求对价格) = 0.886 (se=0.045, p=0.0000), 模型R2=0.469, n=1085
花菜类: 弹性ε(需求对价格) = 0.901 (se=0.068, p=0.0000), 模型R2=0.318, n=1084
茄类: 弹性ε(需求对价格) = 0.073 (se=0.073, p=0.3197), 模型R2=0.499, n=1050
辣椒类: 弹性ε(需求对价格) = 0.525 (se=0.042, p=0.0000), 模型R2=0.459, n=1085
食用菌: 弹性ε(需求对价格) = 0.744 (se=0.069, p=0.0000), 模型R2=0.420, n=1085
```

**问题出现了**：全部6个品类的弹性都 < 1（需求缺乏弹性），而经典垄断定价公式 P* = ε·c'/(ε-1) 只有在 ε > 1 时才有有限正解——ε < 1 意味着按常弹性模型的数学结构，利润会随价格单调上升、无穷推高价格才是"最优"，这在业务上完全不现实（蔬菜不可能卖到脱离市场的价格）。这其实是符合直觉的：蔬菜是生活必需品，短期价格弹性本来就偏低，这与"生鲜蔬菜=刚需品"的常识一致，不是数据错误。

**方案调整**：放弃常弹性（对数-对数）模型，改用**局部线性需求模型** Q = a - bP（同样控制星期/月份/趋势），线性需求下利润函数 π=(P-c')(a-bP) 恒有内部极值 P*=(a+bc')/(2b)，不依赖弹性是否大于1。重新跑：

```
=== 线性需求模型: qty ~ avg_price + 星期 + 月份 + 趋势 ===
水生根茎类: dQ/dP = -0.604 (b=0.604), p=0.0678, R2=0.403
花叶类: dQ/dP = -26.612 (b=26.612), p=0.0000, R2=0.395
花菜类: dQ/dP = -3.257 (b=3.257), p=0.0000, R2=0.330
茄类: dQ/dP = 0.056 (b=-0.056), p=0.7535, R2=0.392
辣椒类: dQ/dP = -2.050 (b=2.050), p=0.0000, R2=0.320
食用菌: dQ/dP = -4.233 (b=4.233), p=0.0000, R2=0.330
```

**第二个问题又出现了**：直接套公式 P*=(a+bc')/(2b) 算出来的"最优价"离谱地高（比如水生根茎类算出 P*≈58元/kg，而该品类历史实际均价从未超过33元/kg）。排查后发现根源是——公式里的截距 a 是"价格外推到 P=0 时的理论需求量"，而历史数据里价格从未接近过0，这是在**样本支持范围之外做了大幅外推**，b 本身数值又偏小（说明控制了星期/月份/趋势后，价格对销量的边际解释力较弱），"a/(2b)"这一项对 b 的微小误差极度敏感，一放大就失真。

**第二次调整（最终方案）**：不再用解析公式外推到全局最优，而是把定价决策限制在**历史真实出现过的加成率区间**内做网格搜索（每个品类的加成率5%-95%分位数、再各放宽15%/30%作缓冲），在这个"有数据支持"的区间里用线性需求模型预测各价格点的销量，选利润最大的格点。这本质上是把"成本加成定价"这一题目给定的业务范式，转化成了"在合理加成率范围内搜索最优加成率"的约束优化，而不是自由定价——这个转向反而更贴合题目"成本加成定价"的原始表述，是一次因数据问题倒逼出的、方向更对的调整。

（详见下文"计算过程"一节的完整代码与结果，以及"遇到的问题与调整"一节对边界解的进一步讨论。）

### 问题3：单品配置怎么做

- **方案A（本次采用）**：两层分解——先用问题2得到的品类总补货量作为"预算"，再在满足 27-33 个单品、每品类候选池（仅限2023-06-24~06-30有销售记录的单品）、单品最小陈列量2.5kg 的约束下，(1) 按品类补货量占比分配各品类的单品名额（不低于2个、不超过该品类候选单品数），(2) 品类内按"预期日毛利贡献"（历史日均销量×单位毛利）排序选品，(3) 品类总量按各单品历史销量份额分摊到单品、并对低于2.5kg的做最小陈列量修正。
- **方案B（考虑后放弃）：整数规划/背包问题精确求解**。把"选27-33个单品、每个单品要不要选是0/1变量、每个选中单品补货量是连续变量"写成一个混合整数非线性规划（目标函数带有各单品自己的价格-销量关系，且价格本身还要优化），理论上更"精确"。放弃原因：(1) 我们并没有对每个单品单独估计出统计显著的需求曲线（单品级数据量比品类级稀疏得多，很多单品在这一周只有几条记录），强行建 251 个单品级需求函数会引入大量过拟合和噪声；(2) 时间和数据支持度不允许；(3) 题目明确说"以品类为单位做补货计划"是问题2的设定，问题3是在问题2基础上做单品落地，用启发式分摊而不是从零重新优化，逻辑上更连贯，也更符合"公司实际操作"的直觉（先定品类预算，再谈单品分货）。**这是本次求解中"有更精确的方法但主动选择更朴素方法"的一个例子，原因是数据支持度不足以撑起更复杂的模型，硬上复杂模型反而是过度自信。**

## 三、关键假设的论证

1. **"收益最大化"按利润(P-c')·Q理解，而非营业额 P·Q**。理由：营业额最大化在数学上等价于把价格压到接近0（此时销量趋于饱和），这不是任何真实商超的目标，也不是"成本加成定价"这一业务前提想表达的意思；国赛官方参考解读普遍也是按"利润/收益"理解。
2. **用附件3的批发价格代表"进货成本"**。理由：题目明确说凌晨3-4点交易的批发价格就是商超的进货渠道价，这是题目给定的唯一成本信息源，没有更细的（如物流费、损耗以外的其他费用）数据，只能以此为准，且题目本身也是这样设定的。
3. **有效边际成本 c' = c/(1-η)，η为品类损耗率**。理由：损耗的菜大部分当日/隔日无法再售（题面原文），意味着为了卖出1kg最终商品，实际要多进 1/(1-η) kg，多进的这部分损耗掉的成本要摊到卖出的每一单位商品上，这是生鲜行业"有效成本"的标准算法，也是把损耗率数据用起来的自然方式（否则附件4就白给了）。
4. **需求 Q 是价格 P 的线性函数（局部近似），而非常弹性形式**。理由见上文"候选方案对比"，经过对数-对数模型的实证检验（全部品类弹性<1，模型无内部最优解）后，改用线性需求局部近似，这是数据驱动的选择而非主观偏好。
5. **未来一周(7月1-7日)的批发成本用最近14天(6月17日-6月30日)历史均价做持久性预测(persistence forecast)**，而非"去年同期+趋势调整"的季节朴素法。理由见"计算过程"部分的实测对比：季节朴素法在不同品类间给出的调整因子波动极大(0.72~1.57)，说明用一个月窗口估计的"年度趋势因子"噪声过大、不稳定；而近7/14/30天三种窗口给出的均值彼此高度一致，说明短期持久性预测在这个1天到1周的预测窗口上更稳健，这也符合生鲜蔬菜批发价"短期强自相关"的常识。
6. **定价搜索区间限定在历史加成率的经验分布范围内（5%-95%分位数附近，适度放宽）**，不做无约束的全局最优外推。理由见上文"问题2候选方案"部分对外推失真问题的实测发现，这是这次建模中最重要的一次"发现问题→调整"。
7. **问题3单品选择以问题2的品类总补货量为预算约束**，理由见"候选方案对比"。
8. **问题3的候选单品池限定为2023-06-24~06-30实际有销售记录的单品**，这是题目原文明确规定的（"根据2023年6月24-30日的可售品种"）。

## 五、遇到的问题与调整（汇总）

1. **对数-对数需求曲线弹性普遍<1，垄断定价公式无内部解** → 改用线性需求局部近似（见上文"候选方案对比"，这是本次最核心的一次方向调整）。
2. **线性需求模型直接外推到P=0求解全局最优价，结果严重偏离历史价格区间（如58元/kg vs 历史最高33元/kg）** → 改为在历史加成率经验分布范围内做网格搜索，不做无支持区间的外推。
3. **约束网格搜索后仍有37/42（约88%）的品类-日期组合停在搜索区间的边界上**，说明控制季节/星期因素后，价格对销量的边际影响本身就偏弱（这与前面对数模型算出的低弹性是同一个现象的两种体现）。这不是代码bug，而是数据本身反映出的"蔬菜类刚需商品短期价格弹性低"的现实，我如实保留了这个结果，并在论文的敏感性分析和模型评价部分把它作为局限明确写出，而不是悄悄调整搜索区间让结果"看起来更好看"。
4. **茄类的价格系数在统计上不显著（线性模型p=0.75，对数模型p=0.32）** → 对该品类采用"历史平均加成率+同期历史平均销量"的兜底策略，而不是强行使用一个不可信的价格系数去做优化，这是诚实反映数据支持度不足的处理方式。
5. **季节朴素成本预测法（去年同期×趋势因子）在辣椒类上给出与近期实际严重不符的结果（7.11 vs 3.40）** → 弃用，改为近14天持久性预测，并用近7/30天窗口做交叉验证确认稳健。
6. **附件4的损耗率只精确到品类，不到单品** → 问题3单品级计算中，同一品类下所有单品被迫共享同一个损耗率，这是数据本身的限制，如实写入问题4的回答和模型局限部分，没有编造单品级损耗率数据去"填坑"。
7. **问题3中花菜类和茄类的候选单品分别只有2个和5个**，即便按占比公式应该分配更多名额，也受限于候选池大小，只能选到全部候选（2个/2个）。这反映了2023年6月24-30日这一周该商超在这两个品类上的单品丰富度本身就低，属于真实业务约束，不是模型bug。

## 六、自我批判：局限与如果重来会怎么改进

1. **需求曲线估计存在价格-销量联立内生性**：本模型用历史观测数据回归 Q~P，但现实中价格本身是商超根据预期需求(季节、库存)主动设定的，二者互为因果，OLS估计的价格系数可能有偏。更严谨的做法需要工具变量(如用批发成本的外生波动作为价格的工具变量)或找自然实验(如某天因供应链偶然因素导致价格偏离常规加成率)，本次求解时间有限未做此项，是最大的方法论局限。
2. **约束网格搜索的边界解问题没有从根本上解决，只是被"框"住了**：37/42个品类-日期组合停在搜索区间上界，说明我们给出的"最优加成率建议"很大程度上等于"历史观察到的加成率上限"，模型的增量信息其实有限。如果重来，应该在需求建模阶段就想办法获取真正的价格实验数据（哪怕是同品类不同门店的定价差异这样的准自然实验），而不是纯粹依赖观测性历史数据。
3. **问题3的单品分摊是启发式(比例分摊+最小陈列量修正)，不是全局最优**：没有对每个单品建立独立需求曲线（因数据稀疏而放弃），意味着单品间的替代效应（如问题1发现的"按斤/按份"负相关）在问题3的补货量分配中并未被显式建模，这是一个已知但未处理的局限。
4. **成本预测只做了到7月1-7日、基于近14天持久性的简单外推**，没有引入任何外部信息(如极端天气预警、节假日效应)，对突发性供给冲击(暴雨导致批发价骤涨)完全没有预警能力。
5. **敏感性分析的样本较小**（只对3个代表性品类、±10%成本/±20%损耗率做了单变量扰动），没有做多变量联合的蒙特卡洛敏感性分析，如果重来会补上这一块以给出置信区间而非点估计。
6. **附件2/3/4之间的口径潜在错位未做更细致核实**（如"退货"记录是否应该反向冲抵对应的进货成本、"是否打折销售"的记录是否应该从"正常价格需求"样本中剔除以避免污染需求曲线估计）。本次为了控制工作量，打折记录未被特殊剔除，直接混在了品类日度加权均价里，这会让估计出的"价格-销量关系"里混入了"打折促销拉动销量"这一额外机制，与"正常价格变化引起的需求响应"存在概念上的差异，是尚未处理的一个已知偏差来源。
