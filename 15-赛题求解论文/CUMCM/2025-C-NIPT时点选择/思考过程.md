## 一、题目理解：这道题到底在问什么

先通读题目原文（`13-国赛CUMCM/真题2025/2025_C题.md`），拆解出四个子问题的真实诉求：

- **问题1**：Y染色体浓度与孕周、BMI 等指标的"关系模型"——这是一个带显著性检验的回归/相关分析问题，本质是探索性统计建模，不是机器学习竞赛。
- **问题2**：给 BMI 分组，每组给出"最佳 NIPT 时点"，目标是让"孕妇可能的潜在风险最小"，并分析检测误差的影响。关键词是"风险最小"——这意味着不能只是简单地对 BMI 做聚类，还要显式定义一个风险函数，把"检测越晚、治疗窗口期风险越高"和"测太早、Y浓度未达标、结果不准"两种风险都装进去。
- **问题3**：在问题2基础上加入身高、体重、年龄等更多因素，同时明确要求考虑"检测误差"和"达标比例"，本质是问题2模型的稳健性/推广版本。
- **问题4**：女胎没有 Y 染色体，题目改问如何用 21/18/13 号染色体的非整倍体标签（AB列）作为判定目标，结合 X 染色体、Z 值、GC 含量、读段数比例、BMI 等做**分类问题**。这是本题里唯一真正意义上的"分类/判别模型"子任务。

拿到题目后第一个判断：**这不是一道可以直接套 AHP/熵权法/BP神经网络的题**。问题1明显是统计推断（相关+回归+显著性检验）；问题2/3 是"回归模型 + 风险函数 + 最优分组（组合优化）"；问题4 是"有金标签的二分类问题"。所以我从一开始就没有考虑用主观赋权法（AHP/熵权），这类方法适用于"指标重要性未知、需要专家或数据驱动赋权"的多准则决策场景，而本题所有问题都有明确的因变量（Y浓度数值、达标与否、非整倍体与否），属于典型的有监督统计/机器学习问题，用回归、分类模型比"赋权打分"更贴合题意、更可检验。

## 二、数据探查（第一步永远是看真实数据长什么样）

用 pandas 读取附件，得到：

```
男胎检测数据: 1082 条记录, 267 名孕妇 (孕妇代码去重)
女胎检测数据: 605 条记录, 147 名孕妇
```

关键发现（真实输出）：

- 男胎 BMI 范围 20.70~46.88，均值 32.29——印证题目"该地区大多为高BMI孕妇"的说法。
- 男胎 Y染色体浓度均值 7.72%，达标（≥4%）记录占比 86.6%。
- "检测孕周"列是字符串格式如 `11w+6`，需要自己写正则解析成十进制周数（`11 + 6/7`）。
- 女胎数据里"染色体的非整倍体"（AB列）取值有 `T18`、`T13T18`、`T13`、`T21`、`T18T21`、`T13T21` 等组合，NaN 代表未检出异常。记录层面异常占比 11.1%，但按孕妇去重后，147 人中有 44 人（29.9%）至少一次检出异常——这说明**这批数据本身是病例富集样本，不是人群自然患病率样本**，后面建模、解释预测值时要特别注意，不能把模型输出的绝对概率直接类比到一般人群。
- 男胎数据里有 238/267 人做过 4 次抽血，说明这是纵向重复测量数据，**不是独立同分布的横截面数据**——这个观察直接影响了问题1和问题2的模型选择（下面会详细说）。

## 三、候选方案对比

### 问题1：关系模型怎么选

**候选方案 A：普通最小二乘 OLS（Y ~ 孕周 + BMI）**
最直接，但忽略了同一孕妇多次抽血形成的组内相关（226/267人有≥3次记录）。跑出来一看：

```
OLS: Y = 0.1194 + 0.0013*GA - 0.0020*BMI, R²=0.046
Durbin-Watson = 0.818
```

Durbin-Watson 远低于2，说明残差存在强烈正自相关——这正是重复测量数据被当作独立样本处理的典型症状（同一孕妇的多次残差会系统性地同向偏离）。**这是我在计算过程中真实发现的问题，不是提前设计好的"教学情节"**。

**候选方案 B：混合效应模型（随机截距，按孕妇代码分组）**
为了修正候选方案A暴露的问题，改用 `statsmodels.MixedLM`，把"孕妇个体差异"作为随机截距吸收掉，只让孕周、BMI 作为固定效应。这样估计的固定效应系数标准误更可信，也更符合"同一人的多次检测应该被视为一组相关观测"这个数据生成机制。跑出来：

```
MixedLM: Y = 0.070 + 0.003*GA - 0.001*BMI
GA_weeks: p<0.001 (z=19.4)   孕妇BMI: p=0.008 (z=-2.65)
```

两个系数都显著，方向符合生理直觉（孕周越大、胎儿游离DNA积累越多，Y浓度上升；BMI越高、母体血浆总DNA被稀释，Y浓度下降）。**最终选用候选方案B作为问题1的主模型**，因为它正确处理了数据的层次结构（个体内重复测量），OLS 只作为对照、用来引出"为什么要用混合模型"这个论证过程。

同时测试了 logit(Y) 变换 + 孕周×BMI 交互项，交互项在混合模型下 p=0.097，OLS下 p=0.271，均不显著，所以最终模型采用**不含交互项的可加形式**，避免过度参数化。

### 问题2：风险最小化怎么定义、怎么分组

这是全题最容易"想当然套模板"的地方，我实际试了两版方案，第一版明显不合理，被我推翻重做：

**候选方案 A（第一版，已废弃）：显式重复检测流程建模**
假设检测失败就按周重新检测，一直到25周，构造一个"期望风险"函数，对起始检测时点 t0 求期望：

```python
E[Risk(t0,BMI)] = Σ_k P(前k次都失败)·P(第k+1次成功)·风险(t)  + P(全部失败)·失败惩罚
```

跑出来的结果是**所有BMI组的最优 t0 都是 10 周（窗口最早时点）**——因为在这个模型里"提前尝试"没有任何代价，越早开始尝试，越有更多机会在低风险区（≤12周）成功，模型必然导向"越早越好"这个退化解，完全不能反映"BMI越高应该等得越久"的临床直觉。**这个方案在计算后被我判定为不可用，问题出在模型没有对"检测本身"（抽血、等待、多次跑单的成本）设置代价**，于是放弃。

**候选方案 B（第二版，同样被放弃）：单次测试风险 R(t)=p(t)·风险等级(t)+(1-p(t))·失败惩罚**
去掉了重复检测循环，改成单次测试的期望风险。跑出来发现**最优时点又退化到固定的12周**（风险等级函数在12→13周处有一个台阶式跳变，从1跳到3，这个跳变的惩罚幅度总是大于晚一周带来的达标概率提升带来的收益，所以模型无论BMI多高都不愿意跨过12周这道坎）。这个方案的问题是：风险函数的具体权重（惩罚值1/3/5、失败惩罚5）是我拍的，对结果起决定性作用，而且**阶梯函数的跳变点会产生人为的最优解锁死现象**，不是一个稳健的建模方式。

**最终方案 C：基于"达标可靠性分位数"的分组模型**
放弃直接构造风险函数打分，改用更贴近临床决策逻辑、也更容易解释和检验的思路：

1. 用问题1/GEE 拟合的达标概率模型 `P(Y≥4% | 孕周t, BMI)`（logistic回归，用 GEE 处理组内相关，见下文）。
2. 对每个 BMI 值，定义 `t90(BMI)` = 使得预测达标概率首次达到 90% 的最早孕周——这是一个"以90%把握测准"为目标的可靠性分位数，比直接编风险权重更贴近临床决策原则（检测的首要价值在于准确，而不是早），也更容易做敏感性分析（把 90% 换成 80%/95% 看结果怎么变）。
3. 对 267 名孕妇的 BMI 值做**动态规划最优分段**（1维分段回归/K-segments问题），每一段的推荐检测时点取该段内"最大 t90"（保证段内所有孕妇都能以≥90%把握测准），目标函数是最小化"组内孕妇相对自己理想 t90 的超额等待时间之和"。

这一版没有出现退化解，BMI 越高分组、推荐时点越晚，且分组数 K 的边际收益递减（K=5→6 只再省 0.6 周/人的超额等待），可以用"肘部法则"给出选 K=5 的理由，与题目给出的示例分组数（5组）吻合，可以直接对照验证。**最终采用方案C。**

### 问题4：分类模型选型

**候选方案 A：临床经验规则，|Z|>3 判定非整倍体**
这是产前诊断领域最经典的判定规则（非整倍体导致目标染色体读段计数偏离正常两拷贝的期望值，Z值理论上会显著偏离0）。作为基线跑了一下：

```
准确率 82.8%，但对真阳性的召回率(敏感度)只有 6.0%
```

这个结果出乎意料地差——远低于文献里"NIPT Z值法灵敏度通常>99%"的报道。我又专门做了分组核验（按标签 T13/T18/T21 分别看对应染色体 Z 值分布），发现**即便被标记为 T21 的记录，21号染色体 Z 值范围也只是 -2.39~1.33，没有一条超过3**。这说明这份题目提供的数据集里，Z值与AB标签之间的关系明显比教科书上的"简单阈值判定"复杂得多（可能掺入了噪声，或者AB标签综合了其他信息、GC含量校正、测序质量等因素）。**这个"意外发现"本身就成为选择候选方案B的关键论据**：既然单一Z值阈值在这批真实数据上明显失效，就必须用能综合多指标的统计模型来处理。

**候选方案 B（最终采用）：多特征逻辑回归（GC含量、Z值、读段质量指标、BMI 联合建模）**
把题目提示的所有相关列（13/18/21号Z值、X染色体Z值、总GC含量及分染色体GC含量、比对比例、重复读段比例、被过滤读段比例、原始/唯一读段数的对数、BMI）都纳入，标准化后用带类别权重的逻辑回归，用**按孕妇分组的5折交叉验证**（避免同一孕妇的多次记录同时出现在训练集和测试集造成信息泄漏）评估：

```
LogReg  分组CV  AUC=0.778±0.073  PR-AUC=0.445±0.069（基线阳性率仅0.111）
RandomForest 分组CV  AUC=0.656±0.080（小样本下过拟合更明显，弃用）
```

逻辑回归明显优于随机森林（这份数据只有67个阳性样本，随机森林这种高方差模型在小样本下容易过拟合），最终选逻辑回归为主模型，随机森林结果作为对照写入正文，用于说明"模型选择也是要基于实际交叉验证结果而不是默认高级模型更好"。

## 四、关键假设及其论证

1. **假设 Y 染色体浓度可用高斯误差的线性/logistic 回归近似**：浓度是 (0,1) 内的比例数据，理论上更适合 Beta 回归或 logit 变换；但本题浓度整体集中在 0.01~0.23，远离 0/1 边界，线性近似引入的偏差有限，且线性系数更易解释、便于后续风险函数求逆运算（brentq 求根）。做了 logit 变换的对照回归作为稳健性检查（见文件 `p1_corr.py` 输出），结论方向一致，故认为该假设合理。

2. **假设组内相关可用随机截距 / GEE 可交换相关结构描述**：即认为"同一孕妇不同孕周的检测结果"之间的相关性近似不随孕周间隔变化。这是简化假设，真实情况可能是间隔越远相关性越弱（AR(1)结构更精确），但本题多数孕妇只有4次检测、时间跨度不大（11~29周），可交换结构是常见且稳健的默认选择，且我们的目的是修正标准误、估计群体平均效应，而不是精确预测个体轨迹，简化是合理的。

3. **假设检测误差可用同一次抽血多次检测的组内标准差来估计**：数据里真实存在"一次采血多次检测"的记录（题目原文也提到了这种情况），我提取了 40 组这样的重复测量（101条记录），计算出真实的组内标准差 SD≈0.00655（绝对浓度），相对变异系数中位数 8.4%。**这是本题里少有的可以直接从数据里"抠出"测量误差、而不需要凭空假设一个误差幅度的机会**，所以没有采用文献里"NIPT测序误差CV约5-15%"这类间接引用的数字，而是优先使用了本数据集的真实经验值。

4. **假设 90% 是一个合理的"达标可靠性"目标**：这是一个可调参数而非硬约束，论文里用 80%/85%/90%/95% 四档做了敏感性分析（见下），说明选择90%只是一个折中的默认展示值，实际决策时机构可以按自己的风险偏好调整。

5. **假设风险等级函数（早期低、中期高、晚期极高）只作为定性论证依据，不再用于构造精确加权的风险打分函数**：这是从"候选方案A/B失败"中吸取的教训——题目描述的三档风险定性合理，但把它数值化为1/3/5这类权重后极易主导优化结果、掩盖了BMI真实的分组信号。最终选择用达标可靠性分位数(t90)代替显式打分，风险等级只用来做结果的**事后**合理性解释（比如指出高BMI组的推荐时点已经逼近甚至超过25周检测窗口上限，提示这批孕妇的潜在风险客观上更难完全消除）。

### 5.4 测量误差敏感性分析（真实 bootstrap，节选自 `p2_error_sensitivity.py`）

先从数据里量出真实测量误差（节选自 `meas_error.py`）：

```
重复测量记录数(同一次抽血多次检测): 101，涉及 40 组 孕妇×抽血次数
pooled within-group std = 0.00655
相对CV中位数 = 0.0837 (8.37%)
```

然后用这个真实误差幅度对原始 Y 浓度加噪声、重新二值化、重新拟合 GEE，重复200次，得到 t90(BMI) 的抽样分布：

```

BMI=30: t90 均值=17.60周 标准差=0.62周 95%区间=[16.47,18.76]
BMI=34: t90 均值=20.37周 标准差=0.55周 95%区间=[19.47,21.52]
BMI=38: t90 均值=23.14周 标准差=1.06周 95%区间=[21.14,25.34]
```

结论：检测误差对推荐时点的扰动大约在 ±0.5~1.2 周量级，中等BMI区间最稳健，两端（BMI较低样本少、BMI较高已接近窗口上限）不确定性更大。

## 六、遇到的问题与调整（如实记录）

1. **系统Python缺 sklearn/statsmodels，pip被PEP668拦截** → 建虚拟环境解决，不是绕过系统保护，是标准做法。

4. **问题4朴素Z值规则准确率高但召回率极低（82.8% vs 6.0%）** → 没有直接采信"准确率高"这个表面数字，而是继续看混淆矩阵和分组下的Z值分布，发现类别极度不平衡下准确率具有误导性，进一步确认了需要多特征模型、且要用PR-AUC/灵敏度而不是单纯accuracy来评估。

## 七、自我批判（局限与如果重来会怎么改）

- **问题1**：用线性/logistic近似浓度这一比例型变量，理论上不如 Beta 回归或广义可加模型（GAM）严谨；若时间允许，应加做 Beta 回归对照，并对孕周做非线性（样条）项，因为生理上Y浓度随孕周的上升速率很可能不是恒定的。
- **问题2/3的t90分位数模型**：90%可靠性阈值本身是人为选定的折中值，虽然做了敏感性分析，但没有从"漏检成本 vs 复检成本"的真实医疗经济学角度反推最优阈值，这需要额外的成本参数（本题未给出），只能作为方法论上的一个可扩展点写入"模型评价与推广"。
- **DP最优分段**用的目标函数（超额等待周数之和）是一种代理指标，没有直接对接"降低胎儿治疗窗口期风险"这个题目最终关心的结局；如果重来，可以尝试把风险等级的期望值也纳入目标函数做多目标优化，但要吸取"手工设计风险权重容易退化"的教训，权重需要更谨慎地标定或做成可调参数并给出多组结果而不是单一"最优解"。
- **问题3的年龄/身高/体重信息使用得比较保守**（直接因VIF剔除了身高体重），更完整的做法是构造一个与BMI正交的"体成分残差"指标（例如身高对BMI的回归残差）纳入模型，但受限于时间没有深入做。
- **问题4的正负样本比例失衡（67:538）且总阳性样本量不大**，5折交叉验证的AUC标准差达到0.073~0.08，说明模型评估本身也有相当的不确定性，247条记录、147人的数据规模不足以支撑非常精细的模型比较结论，随机森林的表现差也可能只是这批数据量下的偶然结果，不能排除在更大样本上表现会反超逻辑回归。
- **本文所有模型都没有做外部数据集验证**（因为题目只给了这一份数据），所有"最佳时点""判定阈值"都只是对这批数据的样本内拟合结果加交叉验证，实际部署到别的医院/别的检测平台前需要重新校准。
