## 一、题目理解：这道题到底在问什么

先通读原文（`13-国赛CUMCM/真题2020/2020_C题.md`），发现 md 提取版把"贷款额度为**__**万元"
中间的数字丢了（应是排版/上标导致提取异常）。为避免编造，我直接解压 `.docx` 的
`word/document.xml` 用正则去标签读原文核实：

```
优惠。银行首先根据中小微企业的实力、信誉对其信贷风险做出评估，然后依据信贷风险等因素来
确定是否放贷及贷款额度、利率和期限等信贷策略。某银行对确定要放贷企业的贷款额度为10~100万元；
年利率为4%~15%；贷款期限为1年。
```

确认关键约束：**单户贷款额度 10~100 万元、年利率 4%~15%、期限 1 年**。这是后面所有优化模型
的硬约束，必须以 docx 原文为准，而不是 md 提取版（这提醒我：任何"看似缺失的数字"都要去
原始文件核实，不能靠上下文脑补）。

题目三问的本质：
1. **问题1**：123 家*有*信贷记录企业（有信誉评级 A/B/C/D，有历史"是否违约"标签）——本质是一个
   **有监督**的信用风险量化问题：能不能从发票流水里学出一个和真实违约结果、和银行人工评级
   都自洽的风险打分，并据此在"总额固定"时做放贷金额与利率的组合决策。
2. **问题2**：302 家*无*信贷记录企业（没有信誉评级，没有违约标签）——本质是**模型迁移/域外推**
   问题：能不能把问题1里学到的"发票特征 → 风险"的映射，迁移到一批完全没有标签的新企业上，
   并在总额固定为 1 亿元时给出策略。
3. **问题3**：在问题2的基础上叠加"突发因素对不同行业不同类别企业影响不同"——但题目给的数据里
   **没有任何行业字段**（企业名称已脱敏为"个体经营E124"这种代号），这是这道题最容易"作弊"的
   地方：很多往届解法直接编一个行业列或者编一套冲击系数表。我决定诚实处理：用发票行为特征做
   聚类构造"经营模式代理分组"，并在论文里明确标注这是"没有真实行业标签时的替代方案"，冲击强度
   本身是情景假设而非从数据估计出来的。

## 二、候选建模思路的比较

### 风险量化环节：候选 A vs 候选 B vs 候选 C

- **候选 A：AHP/熵权法打分卡**（往届最常见的"三板斧"做法）。优点是不需要违约标签、解释性强；
  缺点是这道题**恰恰给了 123 家企业的真实违约结果**，如果不用这个监督信号去验证/训练模型，
  等于把最宝贵的信息扔掉了，评委明确批评过这种"有标签不用、硬套主观赋权"的做法。**放弃**。
- **候选 B：纯规则打分**（比如直接用银行给的信誉评级 A/B/C/D 当风险等级，不再深挖发票）。
  问题是问题2的302家企业**根本没有信誉评级**，这条路在问题2直接走不通，必须放弃或只能作为
  问题1的下界基准。
- **候选 C（采纳）：从发票流水提取特征 → 有监督模型（logistic regression）预测违约概率，
  同时训练一个"信誉评级分类器"用于问题2的迁移**。理由：
  - 123家企业的"是否违约"是真实历史结果，天然适合做验证：可以用 AUC/交叉验证这种客观指标
    检验模型有没有学到东西，而不是自说自话。
  - 302家企业和123家企业共享同一套发票字段（进项/销项发票金额、税额、状态、交易对手代号），
    所以特征工程完全可迁移。
  - 保留可解释性（用逻辑回归而不是黑箱模型），方便把模型系数写成"什么样的经营特征意味着高
    风险"这种能讲清楚机理的结论，符合"不要黑箱三板斧"的要求。

我也跑了随机森林做对比（见下文代码），交叉验证 AUC 逊于逻辑回归且样本量小（123个样本，27个
违约）容易过拟合，所以最终选逻辑回归为主模型，随机森林只作为稳健性对照。

### 定价/额度分配环节：候选 A vs 候选 B

- **候选 A：整数规划/0-1背包**，把"选哪些企业+具体给多少钱"一起当整数规划求解。
  对 123/302 家企业做 0-1 背包在理论上更"标准"，但由于本题的每笔贷款利率是可以针对企业
  连续调整的决策变量，问题实际上是一个"先对每个企业独立求最优利率（不受总额约束），
  再对企业排序做背包"的**可分离结构**：给定企业 i 的最优利率后，单位资金的期望收益
  `e_i` 与该企业分到多少钱无关（收益对贷款金额是线性的）。这种"线性背包"问题的 LP 松弛
  用贪心（按 e_i 降序，从高到低顺序把每户额度打满到上限，直到总额用完）就是**精确最优解**，
  不需要上整数规划求解器。选**候选 B（贪心/分数背包）**，更简洁、可解释性更强，还能直接
  从贪心过程读出"边际收益递减"这种有意义的经济学结论，用于问题2/3的敏感性分析。

### 违约概率与利率的联动：候选 A vs 候选 B

- **候选 A：先固定所有企业统一利率**（比如都按信誉评级给一个固定利率表），不去利用附件3的
  流失率数据做优化。这样附件3就白给了，也无法体现"高风险高定价、但定价太高会吓跑客户"的
  真实银行决策逻辑。**放弃**。
- **候选 B（采纳）：把利率作为决策变量，对每个企业在 [4%,15%] 网格搜索，最大化
  "客户接受概率(1-流失率) × 单位资金期望收益"**，用附件3的经验流失率曲线做客户接受概率的
  代理。这个设计能同时用上附件1的违约标签和附件3的流失率数据，是本题三个附件真正被"放在一起
  用"的关键环节。

## 三、关键假设及论证

1. **贷款损失假设（LGD，Loss Given Default）**：中小微企业普遍缺少抵押物（题目原文点明这一点），
   基准情形假设违约后本金 100% 损失（LGD=1），并在敏感性分析中放松到 LGD=0.4/0.6/0.8 做对照。
   这是保守但符合题目背景（"缺少抵押资产"）的假设，不是随手拍的数字，且做了敏感性检验其影响
   方向是否符合直觉（LGD 越低，可放贷利润越高、可接受的定价越低）——检验结果符合直觉（见下文）。
2. **"违约=完全损失"发生在贷款期末，利息与本金同时计入**：期限固定 1 年（题目给定），
   不做期限结构建模，简化为单期决策，这是题目本身的设定，不是我加的假设。
3. **流失率曲线只区分 A/B/C 三档**（附件3本身如此，D 档银行原则上不放贷、故无需流失率）：
   对问题2的302家无评级企业，需要先用信誉评级分类器把它们映射到 A/B/C/D 四档之一，才能查到
   对应的流失率曲线，这是模型 A（评级分类器）存在的第二个必要性（不仅仅是风险打分，还要给
   查表用的"档位"）。
4. **D档企业原则上不放贷**：这是题目原文明确写的银行政策（"银行对信誉评级为D的企业原则上不予
   放贷"），不是我的假设，是硬约束；同时数据也验证了这条政策的合理性——见下文"D档100%历史
   违约率"的发现。
5. **问题3行业冲击的代理分组是情景假设，不是从数据反推的真实行业标签**：因为原始数据里企业名称
   已脱敏（302家里没有行业字段），我用经营特征（收入波动率、近6月/早6月增长动量、客户集中度、
   收入规模）做 K-means 聚类，构造"经营模式相近"的分组，并对最脆弱分组赋予更高的冲击系数。
   冲击系数的具体数值（1.05~2.5倍违约概率放大）是**情景演示性设定**，论文里会反复标注这一点，
   避免给读者"这是从数据算出来的真实行业影响"的错觉。

### 4.3 特征工程

从进项/销项发票明细为每家企业构造以下特征（代码见 `work2020c/features.py`，核心逻辑）：

```python
total_cost = j_valid['价税合计'].sum()          # 有效进项发票合计（近似成本）
total_revenue = x_valid['价税合计'].sum()        # 有效销项发票合计（近似收入）
gross_profit = total_revenue - total_cost
profit_margin = gross_profit/total_revenue        # 进销差沉淀率，不是真实会计利润率
neg_rate_j / neg_rate_x   # 负数发票(退货)占比：进/销两侧
void_rate_j / void_rate_x # 作废发票占比：进/销两侧
n_suppliers / n_customers # 上游/下游交易对手数
cust_hhi / supp_hhi        # 按交易额计算的客户/供应商集中度(HHI指数)
growth_trend                # 月度收入的线性趋势斜率(标准化)
cv_revenue                  # 月度收入的变异系数(波动率)
momentum                    # 近6月均值 vs 最早6月均值 的相对变化(仅当月度数>=12才算)

A: 82家, B: 88家, C: 79家, D: 53家   (合计302)
```

### 4.7 违约概率的"收缩估计"(shrinkage)——为什么不直接用逻辑回归原始输出

如果直接把 logistic 模型对 A 档企业输出的违约概率当真，会出现一个问题：A档27家历史违约0起，
如果模型对某个A档企业硬预测出接近0的违约概率，会显得"过度自信"（0起违约不代表真实违约率是0，
只是样本量小、暂时没观察到）。所以违约概率最终用**分档基准率（Laplace平滑）和逐户模型分数的
加权**：

```python
p_base = ((违约数+1)/(样本数+2))   # Laplace平滑，避免0/0的极端值
```
真实输出：
```
p_base by tier: {'A': 0.0345, 'B': 0.05, 'C': 0.0833, 'D': 0.9615}
```
最终 `p_final = w * p_model + (1-w) * p_base(评级)`，基准情形取 w=0.5，并在敏感性分析里
扫描 w∈{0, 0.25, 0.5, 0.75, 1.0} 看结果稳健性（见下文）。

### 4.8 附件3流失率曲线与利率优化

```python
def churn_func(rate, tier):
    return np.interp(rate, f3['rate'], f3[col])   # 30个离散点做线性插值

def best_rate_and_return(p_default, tier, LGD=1.0):
    rate_grid = np.linspace(0.04, 0.15, 111)
    accept_prob = 1 - churn_func(rate_grid, tier)
    unit_return = (1-p_default)*rate_grid - p_default*LGD
    expected_unit = accept_prob * unit_return
    i = argmax(expected_unit)
    return rate_grid[i], expected_unit[i], accept_prob[i]
```
演示性输出（不同违约概率下三档的最优定价）：
```
A  p=0.01  best_rate=5.80%  E[单位收益]=0.0312  接受概率=0.658
A  p=0.05  best_rate=10.10% E[单位收益]=0.0127  接受概率=0.277
A  p=0.10  best_rate=14.30% E[单位收益]=0.0035  接受概率=0.122
A  p=0.30  best_rate=15.00% E[单位收益]=-0.0152 接受概率=0.078  (亏损，不该放)
B/C 档同样模式，数值接近A档（流失率曲线本身在A/B/C之间差异不大）
```
可以看到模型能正确反映"风险越高、最优定价越高，但定价推高到一定程度流失率飙升，期望收益反而
见顶后下降甚至转负"这个银行定价的核心权衡，这正是我选择"网格搜索利率"而非"固定利率表"的原因。

### 4.10 敏感性分析（真实跑出的四组扫描）

**(1) LGD（违约损失率）敏感性**：
```
LGD=1.0: 资助100家, 期望利润145.99万, 平均利率10.40%
LGD=0.8: 资助100家, 期望利润175.29万, 平均利率9.46%
LGD=0.6: 资助100家, 期望利润210.71万, 平均利率8.42%
LGD=0.4: 资助100家, 期望利润254.02万, 平均利率7.49%
```
方向完全符合直觉：损失率假设越保守（LGD越低即可回收比例越高），银行不需要用高利率去覆盖
风险，平均最优利率随之下降、期望利润上升。**这个单调关系本身就是模型内部逻辑自洽的一个检验**。

**(2) 收缩权重 w 敏感性**：
```
w=0.00: 资助100家, 期望利润176.63万, 平均p_final=0.0373  (完全信任分档基准率)
w=0.25: 资助100家, 期望利润154.94万, 平均p_final=0.0447
w=0.50: 资助100家, 期望利润145.99万, 平均p_final=0.0500  (基准情形)
w=0.75: 资助100家, 期望利润141.50万, 平均p_final=0.0540
w=1.00: 资助99家,  期望利润139.42万, 平均p_final=0.0573  (完全信任逐户模型)
```
w越大越"个体化"、越保守（因为逐户模型能捕捉到分档基准率掩盖的个体高风险异常值），结果单调
变化、幅度不算剧烈（145.99万 vs 176.63万区间内波动约±20%），说明结论对这个超参数不算特别
敏感，可以接受w=0.5作为折中的默认设置。

**(3) 总信贷额度 C_total 敏感性**（体现边际收益递减规律）：
```
C=3000万:  资助30家,  期望利润64.38万,  边际回报率2.15%
C=6000万:  资助60家,  期望利润109.35万, 边际回报率1.82%
C=10000万: 资助100家, 期望利润145.99万, 边际回报率1.46%
C=15000万: 资助142家, 期望利润158.59万, 边际回报率1.06%
C=20000万: 资助142家, 期望利润158.59万, 边际回报率0.79%  (与15000万完全一样!)
```
在C=15000万和20000万时资助家数、期望利润完全相同——这不是bug，是因为302家企业里
`e_unit>0`（值得放贷）的总共只有142家，一旦这142家全部按100万上限放满，再多的额度也没有
更多"划算"的客户可投，边际利润为0。这正好给出一个有意义的结论：**在当前风险定价体系下，
这批302家客户能够消化的"划算"信贷总量上限约为14200万元**，超过这个规模银行应该考虑放松风控
标准或开拓新客户，而不是把钱硬塞给已经识别为期望亏损的企业。

**(4) 单户额度上下限 cap/floor 敏感性**：
```
cap=100,floor=10(基准): 资助100家, 期望利润145.99万
cap=80, floor=10:       资助125家, 期望利润124.96万  (分散但总利润降低——集中打给最优质客户更划算)
cap=100,floor=20:       资助100家, 期望利润145.99万  (与基准相同，floor没有真正起作用)
cap=50, floor=10:       资助142家, 期望利润79.29万   (进一步分散，总利润明显下降)
```
说明"把钱集中给风险调整后收益最高的客户直到上限"确实是（在纯粹追求期望利润最大化意义下）
更优的策略，这也是为什么现实中银行还需要额外的分散化/普惠金融政策约束（本模型没有内生这类
约束，属于局限性，会在论文里写明）。

### 4.11 问题3：突发因素冲击情景（诚实处理"无行业数据"这个难点）

先确认302家企业信息里确实没有行业字段（只有企业代号和企业名称，且名称已脱敏为
"个体经营Exxx"这种格式，不含行业信息）：
```python
info2['企业名称'].head()
# 0    个体经营E124
# 1    个体经营E125 ...
info2['企业名称'].str.contains('个体经营').mean()   # 0.1854 —— 只有18.5%是个体户，不是全部
```
（这里我一开始猜测"是不是302家企业名称全都是个体经营户"，实际算出来只有18.5%，及时纠正了
这个想当然的猜测，避免论文里写错。）

既然没有行业标签，用经营模式特征做 K-means 聚类（4类）构造"经营模式相近分组"作为行业冲击的
**代理变量**（明确标注为情景假设，不是数据反推的真实行业分类）：

```python
cluster_cols = ['cv_revenue_w','momentum_w','cust_hhi','log_revenue']
km = KMeans(n_clusters=4, n_init=20, random_state=42)
feat2['cluster'] = km.fit_predict(标准化后的特征)
```
真实输出（各簇特征均值）：
```
簇0(n=39):  cv=0.83  momentum=-0.01  客户集中度=0.70  ln收入=16.47  → 客户集中、无增长动能
簇1(n=77):  cv=1.07  momentum=+2.68  客户集中度=0.23  ln收入=17.10  → 高增长动能、客户分散
簇2(n=75):  cv=1.01  momentum=+0.06  客户集中度=0.21  ln收入=13.88  → 规模最小
簇3(n=111): cv=0.71  momentum=+0.30  客户集中度=0.12  ln收入=17.75  → 规模最大、最分散、最稳定
```
按"脆弱度"= z(波动率) - z(动量) + z(客户集中度) - z(规模) 排序：簇0 > 簇2 > 簇3 > 簇1，
符合直觉（客户集中+无增长的簇0最脆弱，高增长+分散客户的簇1最抗冲击）。

赋予**情景性**冲击倍数（违约概率的"胜率"放大倍数，越脆弱倍数越高）：
```
簇0: ×2.5   簇2: ×1.8   簇3: ×1.3   簇1: ×1.05
```
用 `p_shock = 1-(1-p_final)^倍数` 重新计算受冲击后的违约概率（保证仍落在[0,1]区间），
重新做定价优化和贪心分配（同样1亿元总额）：
```
冲击情景: 资助100家(与基准相同家数，但换人), 剩余未分配=0.0万
冲击情景期望利润=116.88万 (相较基准145.99万下降19.9%)
被冲击后"退出资助名单"的企业数=5, 新进入资助名单的企业数=5
调整涉及金额=500万元 (5家×100万，一进一出)
```
这个结果的经济含义是合理的：总额度不变时，银行的最优应对不是简单地对所有企业统一降低额度，
而是**在同一笔预算内做结构再平衡**——把原本处于"资助边缘"的高脆弱度企业替换为冲击后相对
更稳健的企业，同时全局期望利润因整体风险上升而下降（这也符合直觉：冲击是负面事件，无论怎么
调整都不可能让利润变得比冲击前更高，这构成了对模型是否荒谬的一个基本合理性检验）。

## 六、自我批判：这版解答的局限，如果重来会怎么改进

1. **样本量小、违约事件更少（27例，且24例集中在D档）**，A/B/C档内部真正的违约样本只有3例，
   任何在这3个样本上训练/验证的模型统计功效都很弱，模型系数、AUC数字的置信区间事实上很宽，
   论文里给出的0.8611这类AUC点估计，如果重来应该补充bootstrap置信区间，而不是只报点值。
2. **`profit_margin`只是"销项价税合计-进项价税合计"占销项的比例**，是一个现金流/发票口径的
   代理指标，不是真实的会计口径毛利率（可能包含跨期错配、非主营业务发票等噪音），论文里已经
   把它称为"进销差沉淀率"而不是"利润率"以避免误导，但如果有时间应该做月度/年度对齐的更严谨
   核算。
3. **问题3的"行业"是聚类代理而不是真实标签**，冲击倍数是人为设定的情景数值，这是本题在没有
   行业字段情况下能做的最诚实处理，但终究不能替代真实行业分类——如果能找到工商注册行业代码
   之类的外部数据做匹配，会是更可靠的做法。
4. **贪心分配没有考虑单一大客户风险敞口以外的宏观分散化约束**（比如不能把钱都投向同一预测评级

5. **利率-流失率关系用的是2019年一年的横截面统计数据**，且只分A/B/C三档，没有企业异质性（同
