## 1. 题目理解：这道题到底在问什么

先把叙述性描述"翻译"成可计算的对象：

- **假目标**在原点 O=(0,0,0)；**真目标**是一个底面圆心 (0,200,0)、半径 7、高 10 的
  圆柱体（轴线沿 z）。
- **导弹**（M1/M2/M3）以 300 m/s 恒速沿一条**直线**飞向假目标（原点），方向和速度全程不变——
  这是题目"导弹飞行方向直指假目标"给出的强约束，意味着导弹轨迹是完全确定的、
  与无人机策略无关的已知函数 M(t)。
- **无人机**在受领任务后可以"瞬时调整飞行方向"，之后以 70~140 m/s 的速度**等高度**
  （高度恒等于初始高度）匀速直线飞行，航向/速度一旦确定就不再变化。这是每架无人机
  的两个自由决策变量（对每一枚弹还有投放时刻、起爆延时两个自由变量）。
- **烟幕弹**脱离无人机后做**平抛运动**（水平速度=无人机速度，竖直方向自由落体）；
  起爆瞬间形成半径可视为 10 m 的球形烟幕云团，此后云团中心以 3 m/s **匀速下沉**，
  该云团在起爆后 20 s 内"可为目标提供有效遮蔽"。
- **有效遮蔽**：题目没有给出精确的数学判据，只说"云团中心 10 m 范围内的烟幕浓度……
  可为目标提供有效遮蔽"。这是本题最关键、也最需要建模者自己"定义清楚"的地方——
  下面第 3 节会详细论证我采用的判据。
- 目标量：**多枚烟幕弹对真目标的有效遮蔽时间尽可能长**，"不同烟幕弹的遮蔽可不连续"
  提示我们要计算的是**多个区间的并集时长**，不是简单求和（重叠部分不能重复计），
  这一点在问题3/4/5的代码实现中必须显式处理。

题目共 5 问，难度递进：
1. 参数全部给定，纯计算一次仿真；
2. 单弹但方向/速度/投放时刻/起爆延时全部自由，是一个 4 维连续优化问题；
3. 同一架无人机投 3 弹（共享航向与速度，只有投放/起爆时刻可分别选择，且相邻投放间隔≥1s）；
4. 3 架无人机各投 1 弹，对同一枚导弹协同；
5. 5 架无人机（每架至多3弹）对 3 枚导弹协同——组合规模最大，需要分配+优化两层决策。

## 2. 候选建模方案的比较

### 2.1 "有效遮蔽"判据怎么定义？——至少想了两种方案

**方案A（点判据）**：只检查导弹到真目标**几何中心**（如 (0,200,5)，圆柱中轴中点）
连线是否被烟幕球遮住。
- 优点：计算量最小，一个时间步只需算一次点到线段距离。
- 缺点：真目标是一个有一定尺寸（半径7、高10）的圆柱体，只看中心点会**系统性高估**
  遮蔽效果——因为哪怕中心被挡住，圆柱体边缘的一部分仍可能从烟幕球侧边"露出来"被
  导弹的光学/雷达导引头看到。对于"尽量避免来袭导弹发现真目标"这个防御性任务而言，
  只要目标有一部分暴露，遮蔽就是不完全的、不能算作"有效"。这个方案在物理上偏乐观，
  不够严谨。

**方案B（轮廓包络判据，最终采用）**：用圆柱体上下底面圆周的采样点集合，近似该
凸体从任意外部视点看到的**轮廓边界**（silhouette）。只有当烟幕球同时挡住"导弹位置→
每一个采样点"的所有连线时，才判定该时刻为有效遮蔽。
- 论证：圆柱体是凸体。从任何一个外部视点看，凸体的可见轮廓边界完全由其表面若干
  "极值方向"上的点决定，物体内部所有点的视线方向都落在轮廓边界视线所张成的锥体内部
  （凸包性质）。因此，只要烟幕球（其"遮挡方向集合"本身也是一个以视点为顶点的凸的
  角锥/球冠区域）能挡住轮廓边界上足够密集的采样点，就能保证挡住轮廓内部的所有点，
  从而保证整个目标都被遮蔽。上下底面圆周正是本题圆柱目标轮廓边界的"极值点"来源
  （侧面是直纹面，侧棱上的极值点必然落在上下圆周之间的连线上，用足够密的圆周采样
  可以很好地逼近）。
- 我在 `core.py` 里对上下圆周各采样 60 个点（细校验）/16 个点（优化搜索时用更少
  点提速），实测两者结果几乎一致（问题1用60点/16点算出的时长都是1.392s，完全相同），
  说明采样密度已经收敛，不是精度不足的问题。

**取舍结论**：采用方案B。虽然计算量比方案A大（每个时间步要做 O(采样点数) 次
点到线段距离计算），但物理上更严谨，符合"有效遮蔽=导弹完全看不到目标"的直觉，
而且用 numpy 向量化后速度完全够用（单次仿真约 5~10 ms）。

### 2.2 数值积分 vs. 解析求区间边界

给定所有参数后，"有效遮蔽时长"理论上可以通过求解遮蔽条件的边界方程得到精确区间，
但由于遮蔽条件是"多个采样点的判据同时成立"的**逻辑与**，边界方程是分段光滑、
无法写出闭式解的。因此采用**细粒度数值积分**（时间步长最细到 0.0002~0.0005 s）
直接对指示函数求和，是更稳健、更容易验证正确性的方案。为了检验数值积分本身的
收敛性，我在问题1中把步长从 0.0005 s 加密到 0.0002 s，结果时长从 1.3920 s
变为 1.3916 s，仅相差 0.0004 s（0.03%），说明积分已经收敛，可以信赖。

### 2.3 优化方法：差分进化 (DE) vs. 几何反解种子 + 多起点局部搜索

这是本题**最大的一次方案调整**，详见第5节"遇到的问题与调整"。先说结论：
标准的 `scipy.optimize.differential_evolution` 在本题的目标函数上几乎完全失效
（原因是可行域极窄），最终改用"利用问题的物理结构反解出合理初始解 + 多起点随机
扰动局部搜索"的两阶段方法。这不是"n板斧"式的无脑套用优化器，而是先分析了目标函数
的病态性质（几乎处处为0、极窄的非零走廊），再针对性设计的搜索策略。

## 4. 关键的物理简化（对求解效率至关重要）

在写代码验证问题1的过程中，我发现一个非常重要的、可以极大简化后续优化问题的
几何事实，把它写在这里因为它直接决定了问题2~5的求解策略：

> **烟幕弹在任意时刻 t≥0 的水平位置 (x(t), y(t))，无论这一时刻它是否已经脱离无人机，
> 都恒等于 `FY0_xy + v·(cosH, sinH)·t`——与具体的投放时刻 t_release 完全无关。**

原因：投放前，烟幕弹随无人机一起水平匀速运动；投放后，水平方向不受力（假设水平
速度不变），所以水平方向的运动方程从 t=0 开始就是同一条直线运动，只是"这枚弹在
t_release 之前还挂在机翼上"而已，不影响水平位置公式。只有竖直方向，从 t_release
时刻才开始自由落体：`z(t) = z0 - 0.5*g*(t-t_release)²`（t≥t_release）。

**这个发现把"投放点+起爆点"这个看似复杂的4维反解问题，转化成了一个可以直接解析
求解的问题**：给定期望的起爆点 (xd,yd,zd) 和起爆时刻 t_det，可以直接反解出唯一的
(航向, 速度, 投放时刻, 起爆延时)：
```
tau = sqrt(2*(z0-zd)/g)
t_release = t_det - tau
speed = |[(xd,yd)-FY0_xy]| / t_det
heading = atan2(yd-FY0_y, xd-FY0_x)
```
（代码见 `geo_search.py: solve_params`）。这让我可以把"在导弹-目标视线上取一个点作为
起爆点"这种有物理意义的直觉，直接转换成无人机的飞行参数，大幅提高了初始种子的质量，
是后续所有优化都依赖的核心技巧。

## 5. 遇到的问题与调整

### 5.1 差分进化直接失效（最大的一次弯路）

问题2最初我直接调用 `scipy.optimize.differential_evolution`对 (heading, speed,
t_release, tau) 4维暴力优化。第一次运行（真实输出）：

```
DE best -> heading=2.6279 speed=96.414 t_release=9.942 tau=20.375  dur=0.0000
精细网格校验 dur= 0.0
```

结果是 0——DE 收敛到了一个完全无效的点。为了搞清楚是不是评价函数写错了，我先做了
一次纯随机搜索验证目标函数的"地形"：

```python
cands = random_search_single(M0, FY0, bounds, n=15000, dt=0.03, target_pts=TARGET_PTS_COARSE, seed=1)
```
真实输出：
```
搜索用时 8.981151103973389 找到非零候选数 1
(2.4, (6.266299447108902, 71.05428084964015, 0.7427702502613143, 0.7052465905950156))
```
**15000 次随机采样，只有 1 次命中非零遮蔽时长**——这说明可行走廊极窄（烟幕球半径
10m，目标半径只有7m，导弹速度又高达300m/s，稍微偏一点整个遮蔽判据就归零），
DE 的默认变异/交叉策略在这种"几乎处处平坦为0、只有窄峰"的地形里基本是在做纯随机
游走，几乎不可能稳定收敛到窄峰内部。

**调整**：放弃直接暴力优化，改用第4节的几何反解技巧生成高质量种子，再用"随机扰动
局部搜索"（坐标下降式、每步接受更优解、步长逐渐衰减）做局部细化。第一次这样做，
效果立刻显现：
```
fine scan: (2.44, (0.1077, 139.35, 0.2131, 0.9077), (1.1208, 0.0870))
refine1(coarse pts) dur= 3.813 params= [1.78e+00 1.40e+02 0.00e+00 7.18e-02] time= 16.9s
refine2(fine pts) dur= 3.84 params= [1.661e+00 1.398e+02 0.00e+00 7.03e-02]
```
从 2.44s 提升到 3.84s。

### 5.2 单起点局部搜索容易陷入局部最优——改成多起点

3.84s 是不是全局最优？不放心，于是写了 `p2_multistart.py`，用5个不同的
LOS(视线)锚点（目标中心、目标上/下边缘等）分别生成种子，加上纯随机搜索的
top-5 候选，一共6个起点分别做两阶段局部细化（粗网格→细网格）。真实输出：
```
候选起点数 6 用时 5.485713720321655
起点0 -> 粗refine=3.612 细refine=3.625
起点1 -> 粗refine=3.708 细refine=3.709
起点2 -> 粗refine=3.772 细refine=3.789
起点3 -> 粗refine=4.588 细refine=4.584
起点4 -> 粗refine=3.688 细refine=3.686
起点5 -> 粗refine=4.584 细refine=4.581
====全局最佳====
dur=4.5840 params(heading,speed,t_release,tau)= [0.08217 102.6955 1.0610 0.1919]
最终极细校验 dur= 4.5838
```
从 3.84s 又提升到 4.58s——说明单起点确实会陷入局部最优（起点0~2、4 都停在
3.6~3.8s 附近的一个次优簇），必须多起点才有把握。这也是我在问题3/4/5 中坚持用
多起点、而不是只跑一次局部搜索就报告结果的原因。**这里我如实说明：由于计算资源和
时间预算有限，我没有做成百上千起点的穷尽式验证，因此不能证明 4.58s 是全局最优，
只能说这是"多起点局部搜索找到的最好解"，论文里会明确写成启发式结果。**

### 5.3 问题3："3弹共享航线"的第3枚弹经常"打酱油"——一个意外但重要的发现

问题3要求同一架无人机3枚弹**共享**航向和速度，只有投放/起爆时刻可以分别选择，
且相邻投放间隔≥1s。我最初以为3枚弹能大致均分总遮蔽时间的3倍，实际跑出来完全
不是这样。

先复用问题2的最优航线（heading=4.7°, speed=102.7），扫描这条线上不同 t_det 的
最优单弹时长：
```
t_det=0.20 tau=0.13 dur=3.060
t_det=0.45 tau=0.41 dur=3.780
t_det=0.70 tau=0.59 dur=4.500
t_det=0.95 tau=0.17 dur=4.590
t_det=1.20 tau=0.05 dur=4.590
t_det=1.45 tau=0.05 dur=3.900
t_det=1.70 tau=0.05 dur=2.940
t_det=1.95 tau=0.05 dur=1.950
t_det=2.20 tau=0.05 dur=0.960
```
**这条线上"有效"的 t_det 窗口只有大约 0~2.2s 宽**——超过这个范围直接归零。
而"投放间隔≥1s"这个约束，天然要求第3枚弹的投放时刻至少比第1枚晚2s，往往已经
落在这条线的"死区"之外。

我又反过来试了"朝假目标方向飞"这条线（heading=180°，即问题1指定的方向），
它的有效窗口宽很多（可以延伸到 t_det≈13s），但**单点遮蔽时长普遍只有2.2~3.0s**，
而且实测3个分散的投放点算出来的**并集**时长反而更低：
```python
bombs = [(t_release=0.65,tau=4.06), (t_release=2.14,tau=4.82), (t_release=3.28,tau=5.43)]
union dur(coarse)= 4.49
```
逐一检查了每枚弹的具体遮蔽区间：
```
t_release=0.65 tau=4.06 t_det=4.71 -> segs= [(6.69, 9.59)]
t_release=2.14 tau=4.82 t_det=6.96 -> segs= [(7.79, 10.65)]
t_release=3.28 tau=5.43 t_det=8.71 -> segs= [(8.8, 11.18)]
```
三段区间大量重叠（都挤在6.7~11.2s之间），并集只有4.49s，比"窄峰高值"策略下
2枚弹撑起的6.38s还差。

**调整**：对整个8维参数（heading, speed, 3组 t_release/tau）做多起点局部搜索
（`p3_refine.py`，10个不同种子），真实收敛结果：
```
种子0: 粗=4.590 细=4.580
种子1: 粗=5.790 细=5.770
种子2: 粗=6.390 细=6.375
种子3: 粗=6.060 细=6.190
种子4: 粗=3.570 细=3.645
种子5: 粗=6.390 细=6.375
种子6: 粗=6.300 细=6.290
种子7: 粗=3.930 细=3.950
种子8: 粗=6.390 细=6.380
种子9: 粗=6.390 细=6.380
=== P3 最佳 ===  dur=6.3800
```
多个不同种子稳定收敛到 6.38s 附近（说明这是一个比较稳健的局部最优，不是偶然），
但**每次收敛出来的方案里，第3枚弹对并集的边际贡献都是0**（例如最终解里
`弹3: t_release=2.000 tau=1.119 t_det=3.119 单弹=0.000`）。原因和上面分析的一致：
高值窄峰航线的有效窗口只有约0~1.7s宽，第1、2枚弹（间隔正好等于最小允许的1s）
已经把这个窗口"吃满"了，第3枚弹被约束推到2s之后，必然落在死区。

**这不是代码bug，是这道题在这个无人机-导弹几何关系下的真实结构性结论**：受
"3弹共享同一条航线"这个约束限制，FY1 的3发弹里，实际上只有2发能发挥作用，
第3发在满足最小投放间隔的前提下已经没有额外收益空间了。我把这个发现原样写进了
论文的结果分析里，而不是硬凑一个"3发弹均衡贡献"的假象。

### 5.4 问题5：分配 + 优化的两层结构，以及为什么放弃"每机3弹"的暴力枚举

5机、每机至多3弹、3枚导弹，理论组合空间是天文数字，不可能穷举，真实的国赛选手
一般也是用遗传算法/粒子群等元启发式在有限时间内找一个"足够好"的方案，而不是
证明全局最优。我采用了更轻量但同样诚实的两步启发式：

1. **快速评估**：对5机×3弹=15个"无人机-导弹"组合，各用较粗的LOS锚点扫描
   （不做局部细化）得到一个快速估计的单弹遮蔽时长上限，真实输出：
   ```
   FY1 M1 quick_best_dur= 2.4    FY1 M2 quick_best_dur= 0.0   FY1 M3 quick_best_dur= 0.0
   FY2 M1 quick_best_dur= 2.35   FY2 M2 quick_best_dur= 2.35  FY2 M3 quick_best_dur= 2.15
   FY3 M1 quick_best_dur= 2.2    FY3 M2 quick_best_dur= 2.15  FY3 M3 quick_best_dur= 2.1
   FY4 M1 quick_best_dur= 2.3    FY4 M2 quick_best_dur= 2.3   FY4 M3 quick_best_dur= 2.05
   FY5 M1 quick_best_dur= 2.25   FY5 M2 quick_best_dur= 2.0   FY5 M3 quick_best_dur= 2.1
   ```
   **FY1 对 M2、M3 完全打不到**（数值都是0——FY1 位于 (17800,0,1800)，几何上只
   跟 y=0 的 M1 航线对齐得上），这是一条硬约束，FY1 必须分给 M1。其余4架无人机
   对3枚导弹的可及性大致相当（quick_best_dur 都在2.0~2.35之间），所以分配主要
   看如何把5机尽量均衡地分给3枚弹（2/2/1），而不是靠这个粗糙的quick_best做精细决策。
2. **分配**：FY1+FY2 → M1，FY3+FY4 → M2，FY5 → M3（单机），使3枚导弹都至少有
   1架无人机负责，且分配到2机的组合优先给自然更难拦截（quick_best偏低）的目标。
3. 对每个"无人机→导弹"配对做完整的多起点局部搜索优化（同问题2/4方法），
   然后按导弹分组计算并集：M1并集=8.477s，M2并集=6.633s，M3单弹=3.728s。
4. 尝试给唯一负责M3的FY5加派第2、3枚弹（复用问题3的"共享航线扫描"方法）时，
   真实扫描 `t_det∈[0.2,40]` 全程未找到任何非零点：
   ```
   (对FY5→M3的最优航线做t_det/tau网格扫描，输出为空——全部dur=0)
   ```
   进一步定点复查发现：这个方向上的有效窗口窄到只有约0.2~0.3s宽（`t_det=12.8`
   到`t_det=13.0`之间），比我扫描网格的分辨率还窄，普通网格根本捕捉不到，
   而这个宽度也远小于1s的最小投放间隔，所以**即使不计算，也能从问题3已经得到的
   结构性结论推断**：FY5 增派的第2、3枚弹几乎不可能有边际收益。
   **调整**：不再徒劳地为每架无人机凑满3弹，而是采用"1机1弹"的5弹方案
   （用满了"每机至多3弹"里的"至多"二字，多出的弹药配额在这道题给定的
   几何关系下确实用不上），并在论文里诚实说明这是时间预算内的启发式结论，
   更大算力/更精细搜索或许能找到极窄的额外窗口，但预期收益很小。

## 6. 自我批判：这个解答的局限性

1. **优化不保证全局最优**。所有"自由优化"问题（2/3/4/5）用的都是"几何反解种子+
   多起点随机扰动局部搜索"的启发式方法，不是凸优化，无法给出全局最优的数学证明。
   多起点、多次独立运行结果收敛到相近数值（如问题2的4.58s、问题3的6.38s）增强了
   对"这是一个较好的局部最优"的信心，但不能100%排除更好解的存在。
2. **有效遮蔽判据本身是一种建模选择**。用圆柱体上下圆周采样点近似轮廓，理论上有
   凸体几何论证支持，但离散采样终究是近似（虽然16点/60点结果一致，说明采样密度
   足够，但如果实际考核标准是别的判据——比如只看目标中心、或者要求遮蔽比例达到
   某阈值而非100%完全遮蔽——数值会不同）。
3. **没有考虑无人机之间、烟幕弹之间的物理碰撞/干扰**，也没有考虑起爆点是否会
   落地（只检查了 z_det≥0 这一个基本可行性条件），更没有考虑无人机航路是否会
   相互穿越导致现实中不可行。
4. **问题5的"1机1弹"结论是在有限算力/时间预算下得出的，不是穷举验证的**。虽然
   我用问题3的结构性发现（共享航线的有效窗口极窄）作为支撑证据，但没有对全部
   15个"无人机-导弹-第2/3发弹"组合都做过精细扫描，存在遗漏更优增派方案的可能性。
5. **本题没有官方参考答案或数据文件可供比对**，所有数值都来自我自己搭建的仿真与
   优化代码，虽然做了收敛性检验（细化时间步长/采样点数结果一致）和多种物理量的
   合理性检查（如"检验结果合理性"一节），但缺乏第三方基准，无法排除模型本身
   存在系统性偏差的可能。
6. **如果重来会怎么改进**：(a) 用更专业的元启发式（如CMA-ES、遗传算法+精英保留）
   替代手写的坐标扰动局部搜索，理论收敛性更好；(b) 对"共享航线"约束下的多弹问题，
   可以把"选择哪几个时间窗口"显式建模成一个区间调度/覆盖问题，用动态规划或整数
   规划精确求解，而不是连续优化里顺带解决；(c) 问题5的无人机-导弹分配可以用
   匈牙利算法或更系统的组合优化方法替代当前的贪心启发式；(d) 增加对无人机航路
   物理可行性（转弯半径、爬升限制等，虽然本题假设瞬时转向）和多弹碰撞规避的约束。
