## 一、题目理解：这道题到底在问什么

先把题目原文过一遍（`2019_E题.md`）。四问的结构是层层递进的：

1. 算每天的**营业额**和**利润率**——这是最基础的描述性统计，但题目特意提示"非打折商品的成本价缺失，零售商利润率一般在20%-40%"，说明这一问的真正难点不是算术，而是**如何在成本信息不完整的情况下合理地把成本"补"出来**。这是一个数据插补/建模问题，不是简单求和。
2. 建立一个指标衡量**打折力度**——题目没有给出打折力度的定义，要求自己设计。这是本题的核心建模自由度所在。
3. 分析打折力度与营业额、利润率的关系——这是"薄利多销"这个说法本身要不要被数据支持的实证检验。
4. 在问题3的基础上按商品大类细分——检验"薄利多销"是否对所有品类都成立，还是因品类而异（生活必需品 vs. 可选消费品，直觉上弹性应该不同）。

一句话概括：**这是一道"先做合理的缺失值插补，再自建指标，再做时间序列/面板的相关与回归分析"的应用统计题，而不是一道套模板的题**（比如硬套灰色关联、AHP 之类）。真正的难点全在于"打折力度指数怎么定义"和"成本怎么补全"这两个建模决策上，这两步选错了，后面算得再精细也没有意义。

### 2.5 关键的"发现和调整"：sku_cost_prc 字段不能直接当真实成本用

这是本题解题过程中最重要的一次方向调整。我最初的候选方案A是：

> **候选方案A（成本插补）**：直接采用有 `sku_cost_prc>0` 记录的行作为"真实成本"，对没有促销记录的 SKU，用同一 SKU 在其他日期的促销成本、或同二级类目其他商品的已知成本做均值插补，把缺失成本补齐。

这是"看起来最贴合字段名字面意思"的方案，也是很多人拿到这题第一反应会做的事。但在写插补代码之前，我先做了一步验证性检验（这一步的产出在下面），结果发现这个方案的前提是错的：

```python
sub = df[df.sku_cost_prc > 0]
diff = sub.sku_sale_prc - sub.sku_cost_prc
diff.describe()
```
真实输出：
```
count    438331.000000
mean         -0.089759
std           0.655995
min         -22.000000
25%           0.000000
50%           0.000000
75%           0.000000
max           0.000000

fraction where sale==cost exactly: 0.9770926537251529   （97.7%）
fraction where sale>cost:          0.0                   （0%，从未出现过）
fraction where sale<cost:          0.02290734627484709   （2.3%）
```

也就是说：在所有标了"成本价"的43.8万条记录里，**97.7%的记录 sku_cost_prc 与 sku_sale_prc 完全相等，且没有一条记录 sale_prc 大于 cost_prc**。这和数据说明的文字完全对得上——`sku_cost_prc` 记录的其实就是"单品直降"这种促销类型下的那个直降后价格本身，只是在系统里被复用/误命名为"成本价"字段，**它不是任何意义上独立观测到的进货成本**。

如果按候选方案A硬着头皮用它当真实成本，会导致一个系统性荒谬结论：**所有"单品直降"商品当天的利润恒为0（或因四舍五入误差略为负）**，这和题目原文明确写的"零售商利润率一般在20%-40%"直接矛盾——不是数据本身有问题，是把这个字段当"成本"来用这个建模假设错了。

**调整决策**：放弃候选方案A，改用候选方案B——不再试图从数据里"抠"出真实成本，而是显式采用题目给的先验区间（20%-40%），假设成本与门店标价成固定比例：

$$\text{成本价}_i = (1-m) \times \text{门店标价}_i,\quad m \in [0.20, 0.40]$$

这个假设更诚实：它不假装自己知道真实成本，而是明确承认"成本率是一个待检验的外生参数"，并通过后续的敏感性分析（对 m 取 20%/25%/30%/35%/40% 五档）来展示结论对这个假设的依赖程度。这是"不编造数字"原则在这道题里最直接的体现——附件数据本身并不支持任何精确的成本推断，与其编造一个虚假精确的插补结果，不如坦承假设、做敏感性分析。

### 2.6 打折力度指数的候选方案比较

**候选方案A**：基于附件3（促销信息表）构建打折力度。附件3记录了促销类型（单品直降/秒杀/买赠/新人专享/第二件N折）、促销价、起止时间，还给出了促销类型之间复杂的优先级覆盖规则（新人专享 > 秒杀 > 直降 > 第二件N折，同类型看开始时间和力度）。理论上这样可以做出一个"按促销手段分类"的更精细指数。但实际做起来问题很大：

- 促销记录是"计划"数据（哪个时间段哪个SKU打几折），不代表当天真的有人买、买了多少，要把它和流水表拼起来还要重新实现附件3里那套复杂的重叠优先级判定逻辑，工程量大且容易引入新的假设误差；
- 我快速看了一眼附件3（`head`），发现很多字段如 `cost_price` 本质上也等于 `promotion_price`（和附件1/2里同样的陷阱），继续深挖价值有限；
- 促销表与题目要问的"打折力度对营业额/利润率的关系"这个目标并不是最短路径——题目关心的是消费者实际拿到的优惠程度，而不是商场用了哪种促销手段。

**候选方案B（采用）**：直接从销售流水表出发，用**实际成交价相对门店标价的下降幅度**定义打折力度，不关心背后是哪种促销机制：

$$D_t = 1 - \frac{\sum_i s_i q_i}{\sum_i p_i q_i}$$

其中 $p_i$=门店标价（sku_prc），$s_i$=实际成交价（sku_sale_prc），$q_i$=销量（sku_cnt），求和对第 $t$ 天所有成交记录。这是一个**以营业额为权重的平均折扣深度**：如果当天所有商品都是全价（$s_i=p_i$），$D_t=0$；折扣越深、参与折扣的商品占比越大，$D_t$ 越大。

选择B而不是A的理由：(1) 与营业额、利润率的计算同源（都基于 sku_prc/sku_sale_prc/sku_cnt 三个字段），逻辑自洽，不需要额外假设匹配规则；(2) 直接反映消费者实际获得的价格优惠这个"薄利多销"定义里最核心的量；(3) 工程上更稳健，不依赖对附件3那套优先级规则的正确复现。

（作为对比和补充，我还构造了第二个辅助指标——打折覆盖率 $B_t$=打折商品销量占比，用于交叉验证结论的稳健性，见下文。）

### 2.7 一个意外发现："利润率对打折力度"的关系在我的成本假设下是**恒等式**，不是统计相关关系

这是解题过程中第二个重要的自我纠错。按照 2.5 节的成本假设，利润率可以解析地写成：

$$\rho_t = 1 - \frac{C_t}{R_t} = 1 - \frac{(1-m)R_t^0}{R_t^0(1-D_t)} = 1 - \frac{1-m}{1-D_t}$$

第一次跑完 Pearson/Spearman 相关时看到这个结果：

```
discount_intensity: profit_rate Pearson r=-0.996 (p≈0)  Spearman rho=-1.000 (p≈0)
```

Spearman 相关系数恰好是 **-1.000**，我一开始差点直接把这个当"强负相关"的实证发现写进论文——但相关系数精确到 -1.000（完全单调）这件事本身就很反常，正常的经济数据不会有这么"干净"的相关性。停下来推了一下公式，才意识到：**在我自己设定的固定成本率假设下，利润率本来就是打折力度的确定性函数，两者相关系数为-1是数学恒等式的必然结果，不是数据告诉我们的新信息**。

验证：
```python
pred = 1-(1-m)/(1-daily.discount_intensity)
np.max(np.abs(pred - daily.profit_rate))  # 输出 2.220446049250313e-16
```
误差在浮点精度范围内，确认是恒等式。

**调整**：把这一步从"发现打折力度与利润率负相关"改写为"**在固定成本率假设下，解析推导出利润率随打折力度线性衰减的公式，并求出盈亏平衡点 $D=m$**"——这是模型的一个解析结论，不是从数据拟合出来的经验相关关系，两者在论文里必须明确区分表述，否则是一种"伪装成实证发现的同义反复"。

同时，为了得到真正有信息量的实证检验，我把重点转向**销量/营业额对打折力度的响应**（这部分不是恒等式，因为销量是真实观测到的行为数据，与打折力度指数的构造方式没有代数耦合），这是第3问真正应该做实证检验的地方。

### 2.8 时间趋势混杂问题：原始相关系数被放大了约4倍

看月度平均营业额：
```
2016-11    2833.70
2016-12    8205.78
...
2018-08   52567.51
2018-12   66420.27
```
营业额从开店初期到两年后增长了约20倍，明显是门店规模/客流自然成长的趋势，如果不控制这个趋势，营业额和打折力度的相关系数会被这个共同趋势严重污染。

先看不控制趋势的朴素回归：
```
log_revenue ~ D :  beta=12.57, p=4.3e-32, R²=0.167
```
再看控制时间趋势（t, t²）和星期几虚拟变量后：
```
log_revenue ~ D + t + t² + dow : beta_D=3.020, se=0.525, p=1.24e-08, R²=0.816
```
系数从12.57骤降到3.02——说明朴素相关里有相当大一部分是"门店越开越大、同时打折也越来越常态化"这个共同趋势造成的虚假放大，控制趋势后效应显著缩小了约4倍，但仍然显著为正。这提醒我：**报告朴素相关系数是有误导性的，必须把控制趋势后的结果作为主要结论**，朴素相关只能作为对比参照。

## 四、遇到的问题与调整（汇总）

1. **venv 环境问题**：第一次建的 venv 因 Homebrew python 链接混乱导致 python3.12/3.14 目录并存、import 失败，重新用绝对路径的 python3 建 venv 解决。
2. **CSV 编码问题**：utf-8 读取报错，改用 `gb18030`。
3. **成本插补方案推翻重来**：候选方案A（直接用 sku_cost_prc 当真实成本）经验证后发现该字段等价于促销价本身、不是独立成本观测，遂改用候选方案B（固定成本率假设+敏感性分析）。
4. **"利润率-打折力度负相关"被识别为恒等式而非实证发现**：及时把这一结果从"回归/相关分析结论"改写为"模型内生的解析推导结论"，避免论文里出现伪装成实证发现的重言式。
5. **朴素相关系数被时间趋势严重放大**：加入时间趋势与星期控制变量后重新估计，主结论以受控回归为准。
6. **附件3最终没有被用于主模型**：评估后认为其促销优先级规则复现成本高、且价值有限（也踩到了同样的"cost_price=promotion_price"陷阱），转而完全基于流水表构建打折力度指数，仅在文中作为候选方案对比出现。

## 五、自我批判：这份解答的局限，如果重来会怎么改

1. **成本假设仍是模型中最弱的一环**。固定成本率 $m$（相对门店标价的加价率）是全局假设，没有考虑不同品类、不同商品生命周期阶段成本结构的差异；虽然做了敏感性分析，但没有能力从数据本身"验证"m的真实取值——这是数据本身的硬约束（附件里确实没有可靠的真实成本观测），不是分析方法能弥补的缺口。
2. **打折力度与销量的关系是相关性而非因果性**，尤其在酒水饮料、日化用品这类品类上很可能存在"促销力度是滞销结果"的反向因果，本文没有做工具变量或双重差分等因果识别，只能在结论中如实标注为相关关系。
3. **附件3（促销类型表）完全没有使用**，如果重做，可以尝试把促销类型（秒杀/直降/买赠/满减）作为打折力度的补充维度，区分不同促销手段的效果差异，这需要先解决附件3里复杂的时间重叠优先级规则，工作量较大，本次时间预算内未展开。
4. **没有做节假日/大促日历（双11、周年庆等）的显式虚拟变量控制**，只用了通用的时间趋势项(t, t²)和星期几哑变量，如果某些大促日同时是深度打折日和自然客流高峰日，仍可能有残余混杂。
5. **品类分析只做了8个营收最高的一级类目**，二级/三级类目更细的异质性未展开，样本量和篇幅所限。
