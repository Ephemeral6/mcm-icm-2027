## 一、题目理解与拆解

读取 `/Users/ephemeral/美赛/13-国赛CUMCM/真题2025/2025_E题.md` 后，题目结构非常清晰，四问层层递进：

- **问题1**：给定运动者1、2的33个人体关键点在301帧视频中的像素坐标（附件1）以及他们的真实跳远成绩，
  要求确定"起跳时刻"和"落地时刻"，并描述滞空阶段的运动过程。——这是一个**时间序列事件检测**问题。
- **问题2**：给定8名运动者（3~10号）纠正前、纠正后的多次试跳数据（附件3）+ 体质报告（附件4），
  要求分析"影响跳远成绩的主要因素"。——这是**特征提取 + 相关性/回归分析**问题，且明确要求"分析主要因素"而不是"预测成绩"，
  所以重点应放在可解释性上，而不是堆砌一个黑箱模型刷精度。
- **问题3**：用问题1、2建立的模型，结合运动者11的视频、位置信息与体质信息（无真实成绩），预测其实际成绩。
  ——这是**模型的迁移应用**，需要问题1的"起跳-落地"检测算法 + 问题2挖掘出的因素关系，两者缺一不可。
- **问题4**：给出运动者11的姿势训练建议，并预测其经过短期训练后可能达到的理想成绩。
  ——需要用到问题2里"纠正前 vs 纠正后"的对比结果，做反事实（what-if）式的估计。

四问是一条完整链路：**检测算法（Q1）→ 因素挖掘（Q2）→ 迁移预测（Q3）→ 训练建议与反事实推演（Q4）**。
我在正式建模前先花时间探查了全部数据文件的目录结构，发现除了任务说明里点名的附件1三个文件外，
本地还有附件3（训练前后对比，8人×多次试跳）、附件4（9人体质报告，含11号）、附件5（11号的视频位置数据），
这些正是问题2/3/4所需的原始数据，而且都是真实数据文件（不是缺失），所以我判断**应该把问题2~4也建立在真实数据上**，
而不是像"没有数据"那样只做示例性说明——这是这次任务里一个重要的自主判断。

## 二、候选建模思路对比

### 关于"起跳/落地时刻检测"（问题1核心）

**思路A：基于关键点绝对高度阈值判断触地/离地**（最先尝试，后来放弃）
用双脚（踝+跟+趾）像素纵坐标 Y 的直方图找"地面基线"众数，设一个像素阈值，Y 大于阈值视为触地。
——**问题**：落地后运动员常有前倾、整理动作、镜头透视的整体漂移，导致触地信号在落地后仍长时间低于阈值，
被误判为"又出现了一次更长的滞空"。实测中运动者2的这次尝试把落地后 150 帧的缓慢漂移误判成了一次 22 帧的"滞空"，
是完全错误的结果（见下文"三、计算过程"第1轮）。

**思路B：基于速度"平静段"回溯/前进搜索**（第二轮，仍有缺陷）
用双脚 Y 的全局最小值（离地最高点）作为腾空中点（apex），从 apex 向前/向后找速度趋于 0 的"平静段"分别定为起跳/落地。
——**问题①**：抛体运动在顶点处垂直速度天然趋近于 0，会被误判"已经落地"，导致落地帧定得过早（滞空时间被严重低估）。
——**问题②**：若视频里除真实跳远外还有更大幅度的噪声起伏（姿态估计抖动、遮挡），全局最小值可能压根不是真实的跳跃事件。

**思路C（最终采用）：候选事件分割 + 局部阈值穿越法**
1. 用 `scipy.signal.find_peaks` 在"负的脚部Y信号"上找所有显著的局部极小值（候选腾空事件），
   而不是只取全局最小值，避免被落地后的整理动作或其他噪声"劫持"。
2. 相邻候选事件之间用中点切割局部窗口，避免互相污染。
3. **起跳前基线和落地后基线分开计算**（各自取窗口内 85 分位数），避免"落地后整体前倾"抬高或压低阈值。
4. 用"基线到谷值 15% 处"作为离地/触地的判定阈值，从 apex 分别向两侧扫描找首次穿越点——
   这一步不依赖速度是否平静，从根本上避开了思路B在顶点处的缺陷。
5. 物理先验过滤：滞空时长必须落在 [0.15s, 0.9s]（立定跳远合理范围），在满足条件的候选里选水平位移最大的一个
   （真实起跳的水平位移显著大于准备阶段的晃动或落地后的小幅调整）。

选择思路C的理由：它是所有思路里**唯一在全部35条试跳记录上都给出物理合理滞空时长（0.30~0.63秒）**的方法，
且不依赖任何"手工挑選"某个视频专用的参数。放弃 A、B 不是因为它们"理论不对"，而是**实测直接暴露出了系统性错误**，
详见下文。

### 关于"影响跳远成绩的主要因素"（问题2核心）

**思路A（放弃）：无脑上 AHP/熵权法或 BP 神经网络**
——这是题目和任务说明里明确提醒要避免的"n板斧"。本题数据量小（8名运动员、~26条试跳、9人体质表），
用神经网络严重不匹配样本量，且熵权法/AHP这类主观赋权法在这里根本没有"专家打分"的场景可用，纯属误用。

**思路B（采用）：先做机理分析，再用两条独立证据链相互印证**
- **证据链①：截面回归**（不同运动员之间比较）——用体质指标（身高、体重、BMI、体脂率、肌肉率、年龄）
  与视频提取的运动学特征（起跳水平速度、垂直速度、滞空时间、屈膝深度、摆臂幅度、躯干前倾角、重心腾空上升高度）
  分别与真实成绩算 Pearson 相关系数，看哪些变量解释力强。
- **证据链②：纵向配对差分**（同一运动员训练前后比较）——因为同一人训练前后的身高体重年龄几乎不变，
  这组对比能天然剔除"体型优势"这个混杂因素，单独看"技术动作改善"对成绩的净效应。

两条证据链一个侧重"个体间为什么甲比乙跳得远"（更多是体型/发育因素），一个侧重"同一个人怎么进步"（更多是技术因素），
二者结合才能回答"主要因素"这个问题——这也是为什么放弃直接把所有变量塞进一个黑箱模型：那样会把两种性质完全不同的
"个体间差异"和"个体内变化"混在一起，解释力反而下降。

### 关于"预测运动者11成绩"（问题3核心）

**思路A（放弃）：直接用截面回归（体质指标→成绩）外插**
运动者11只有6岁、身高120cm、体重21kg，而训练集里除了8岁的运动者9外，其余都是19~34岁的成年人，
身高体重范围是138.7~173.5cm / 31.6~67.0kg——11号完全落在训练集范围之外，线性回归外插给出的数值
（按身高单变量回归得0.86m，按体重单变量回归得0.90m）本质上不可信，我在计算后也把这两个数字明确标注为
"外插、仅作参考"，没有采信为主结果。

**思路B（采用）：视频测距法（摄影测量），用截面数据校准系统偏差**
核心逻辑：运动者11的身高（120cm）是已知的真实值，可以直接把它作为"标尺"，从视频关键点里
量出像素→米的比例尺，再算出起跳-落地的像素位移换算成的"视频测距成绩"。
这不依赖"运动员像谁"，是直接的物理测量，理论上比任何回归外插都更可靠。
但第一轮直接测出来的距离系统性地比真实成绩大了约27%（下文有详细验证），
所以补一步：用8名有真实成绩的运动员（附件3+4）把"视频测距值"与"真实成绩"做一次线性标定回归，
再把这个标定关系套用到运动者11身上。这样既用上了摄影测量的物理直接性，又用真实数据修正了系统偏差，
是我认为本题下最站得住脚的方案。

### 3.2 起跳/落地检测算法的三轮迭代（问题1）

**第一轮（思路A，朴素阈值法）在运动者2身上的真实输出：**
```
运动者2的跳远位置信息.xlsx n_frames= 302
  起跳帧=279 落地帧=301  滞空帧数=22  滞空时间=0.733s
  起跳-落地水平像素距离= 68.4
  若用真实成绩标定比例尺: px->m = 0.01681286549707602
```
运动者1标定出的比例尺是 0.00367 m/px，运动者2却是 0.0168 m/px，相差4.5倍——两条视频分辨率、拍摄距离接近，
比例尺不该差这么多，说明检测到的"滞空区间"根本是错的。把脚部Y信号逐帧打印出来才看清真相：
```
150 545.9 ... 165 534.5 166 439.4 167 414.6 168 386.3 169 389.3 170 446.0 ... 176 546.6 177 557.5 ...
```
真实的跳跃发生在约160~177帧（Y值有一次干净的大幅下探），而算法却把279~301帧判成了"滞空"——
逐帧检查发现落地(约177帧)之后，脚部Y值缓慢从546一路漂移到511（frame 200~300），这段缓慢前倾/晃动
被朴素阈值判成了持续"未触地"，酿成了假阳性。这是**第一次实质性的思路调整**：放弃全局阈值法。

**第二轮（思路B，速度平静段回溯法）** 部分修好了运动者1、2（滞空时间变为0.467s/0.400s，比例尺基本一致：
0.00355 vs 0.00368 m/px），但在批量跑全部35条记录时，在运动者3调整前第5次、运动者7调整前第2次、
运动者10调整前第1次等样本上出现新的错误，例如：
```
ath10_before1 BEST takeoff= 142 landing= 147 flight_s= 0.167 dist_px= 76.75
```
而实际逐帧数据显示真实滞空跨越 142~156 帧左右（约0.47秒），落地帧被错误地定在了147帧——
原因正是"抛体顶点附近速度天然趋近于0"，被误判成了"已经落地"。这是**第二次思路调整**。

**过程中意外发现的数据质量问题：** 在排查运动者7调整后第2次为什么被识别出一个"腾空高度超过半个身长"的
明显不合理候选时，逐帧打印发现：
```
总零值帧-关键点对数: 66 / 9933
含零值的帧号: [177 178]
```
即该样本第177、178帧有关键点被记为坐标 (0,0)——这是姿态估计在运动员高速腾空、动作形变剧烈的瞬间
（恰好是我们最关心的时刻！）漏检后的占位值，如果直接拿来做差分/找极值会引入巨大伪影。
对全部35条记录扫描后发现，这个问题相当普遍（如运动者5调整前第2次多达160帧缺失、运动者8调整前第2次48帧缺失），
于是给 `load_traj()` 加上了"缺失值线性插值修复"这一步（把(0,0)记为NaN，按帧号插值），
这是整个流程里**最关键的一次数据清洗调整**，修复前后很多样本的极值检测结果发生了显著变化。

**第三轮（最终版，候选窗口+局部阈值穿越法）**：清洗数据 + 改进算法后，重新验证前述失败样本：
```
ath3_before5 BEST takeoff= 110 landing= 124 flight_s= 0.467 dist_px= 398.2
ath7_before2 BEST takeoff= 161 landing= 173 flight_s= 0.4 dist_px= 513.5
ath10_before1 BEST takeoff= 142 landing= 156 flight_s= 0.467 dist_px= 395.8
```
滞空时长全部落回合理区间。对全部35条记录跑一遍统计检验：
```
滞空时间分布: mean 0.438s, std 0.067s, min 0.300s, max 0.633s
是否有 flight_time 超出 0.15-0.9s 合理范围: (空，全部通过)
```

**运动者1、2的最终结果（问题1的直接答案）**：

（这里的比例尺是用两人各自的真实成绩反标定的，distance 与真实成绩相等是标定方式决定的，不是"预测准确"，

### 3.3 发现完全重复的数据记录（意外情况，如实记录）

在给问题2建训练集之前，我对全部35条位置信息做了两两轨迹签名比对：
```python
sig = tuple(np.round(xy[:20,0,:].flatten(),1))  # 取前20帧鼻子坐标做签名
```
输出：
```
发现的完全重复对: [('3-after-1', '1-baseline-1')]
```
进一步核实：
```
$ md5 附件1/运动者1的跳远位置信息.xlsx 附件3/姿势调整后/运动者3调整后第1次的跳远位置信息.xlsx
MD5 ... = 96db2f0... (不同)
MD5 ... = f7a33ee... (不同)
>>> np.allclose(xy1, xy2)  ->  True   （全部33×301×2个坐标值逐一相等，最大误差0.0）
```
两个 xlsx 文件本身（含格式等元信息）不同，但里面的33个关键点在301帧上的坐标数值**完全相同**。
"运动者3调整后第1次"的真实成绩恰好也是1.58米，与运动者1的成绩完全一致——推测是原始数据打包时的
误复制/占位（真实原因无法从数据本身确认）。无论原因是什么，如果我把这条记录当作"运动者3训练后的真实视频"
纳入回归，会把运动者3的身体数据（身高158.1cm等）错误地配上运动者1的动作特征，属于明显的数据污染。
**处理方式**：在构建问题2/3的训练集时，把 `3-after-1` 这条记录整体剔除，只保留 `3-after-2`
作为运动者3训练后的样本。这是一处必须如实说明的数据缺陷及应对。

### 3.4 起跳水平速度的方向符号bug（意外情况，如实记录）

第一版特征提取里，起跳水平速度 `vx0` 直接取髋点X坐标差分。批量跑完后发现除了运动者11以外，
所有34条训练样本的 vx0 都是正数，唯独运动者11是 `-2.024`——检查后发现原因是**运动者11的视频里，
运动员的助跑/跳跃方向与其余视频相反**（画面里人朝反方向跳），并非模型或数据错误，而是不同视频的
拍摄朝向本身不统一。如果不处理，这会导致"水平速度"这个特征在跨样本比较时符号错乱、无法使用。
**处理方式**：改用"起跳点到落地点的净位移方向"作为该条记录的"前进为正"基准，重新统一了符号，
详见 `extract_features.py` 中 `direction = np.sign(hx[ld]-hx[to])` 一行。

**为什么会有~27%的系统性高估？** 我把这个现象当成一个需要解释的异常，而不是直接套用回归掩盖过去。
排查后认为主因是身高标定方式的系统偏差：标定时用"鼻子到脚跟"的像素竖直跨度对应"真实身高"，
但人体测量学意义上的身高是"头顶到脚底"，鼻子位置比头顶低（约占身高的8%~12%），
这会让像素身高被低估，从而让"米/像素"的比例尺系数被高估，一切基于该比例尺换算出的距离也随之整体偏大；
另外起跳/落地阈值穿越法本身也可能有几帧的系统偏差。由于这个偏差在31个样本上**方向一致、幅度稳定**
（20%~40%区间，均值27%），用线性回归标定是合理且诚实的处理方式（不是"调参凑数"，而是先定性解释偏差来源，
再用真实数据做经验修正），且做了留一交叉验证（LOOCV RMSE=0.091m）确认标定关系不是过拟合：
```
LOOCV RMSE= 0.0909174942939012  LOOCV MAE= 0.0662250470965983
```

但把各运动学特征的训练前后差值（Δfeature）与成绩差值（Δscore）算相关系数：

这是一处我主动选择"如实报告负面/不确定结果"而不是硬凑一个解释的地方。最终问题2的结论建立在

```
  起跳-2帧,落地+0帧 -> 1.114m   起跳+2帧,落地+0帧 -> 1.045m
```

结论：**身高测量误差的影响相对温和**（±5%身高误差 → 预测成绩变化约±0.05m，约±4.5%），
**帧检测误差的影响更大且不对称**（尤其落地帧提前2帧会让预测成绩骤降至0.894m，降幅约19%）——

## 四、自我批判：局限与如果重来会怎么做

1. **起跳/落地检测算法完全是我基于对信号的观察手工调参出来的启发式方法**（阈值15%、窗口±30帧等），
   没有做出统一的、可自动泛化到任意场景的严格判据；虽然在本题35条记录上全部通过了物理合理性检验，
   但换一批新视频未必稳健，理想情况下应该用更系统的方法（如基于加速度二阶导数、或对多个关键点做投票）。
2. **摄影测量的单目视频身高标定本身有明显偏差（约27%），且不同人的"鼻子到脚跟/身高"比例本身也因人而异
   （尤其运动者11是6岁儿童，头身比例与成人差异更大）**，这意味着为成人标定的27%修正系数直接套用到儿童
   身上可能引入额外误差，这是本文没有单独处理、值得进一步改进的地方。
3. **问题2的"纵向配对"分析样本量只有8对，统计功效很弱**，所有Δfeature相关系数都不显著，
   最终对"哪个具体动作要素驱动了成绩提升"这个问题给出的是较弱、定性的结论，而不是强因果结论——
   我认为这是诚实的做法，但也确实是本文分析深度的一个上限。
4. **对运动者11训练效果的"理想成绩"预测本质上是一个类比外推（借用其他8人的平均提升幅度），
   没有对11号本人的技术缺陷程度做定量映射**（比如按摆臂幅度差距的大小去标定一个"专属"提升幅度），
   如果重做，可以尝试建立"技术差距量 → 提升幅度"的简单线性映射，用其余8人的（训练前技术缺陷程度，
   实际提升幅度）这对关系去外推11号，可能比直接套用群体均值更精细，但样本量（n=8）能否支撑这样的模型
   也存疑，需要谨慎权衡是否会过拟合。
5. **发现的两个数据问题（重复记录、坐标缺失）已经处理，但没有对处理方式本身做敏感性分析**
   （例如"如果保留重复记录/不做插值会怎样"），受时间限制未展开，这是可以补充的工作。
6. 本文未使用视频本身的像素/光流信息，也没有用体质报告里的其余细分指标（内脏脂肪、基础代谢等），
   这些理论上也可能与运动表现相关，但考虑到样本量已经很小、盲目增加变量数会直接导致过拟合，主动没有纳入。
