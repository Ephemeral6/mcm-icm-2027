## 一、题目理解与拆解

题目给了三问：
1. 对自建点数据与国控点数据进行**探索性数据分析**（EDA）；
2. 分析导致两者**差异的原因**；
3. 建立数学模型，**用国控点数据校准自建点数据**。

关键信息抓取：
- 国控点（附件1）：每小时一条记录，"两尘四气"六项指标（PM2.5、PM10、CO、NO2、SO2、O3），被视为"较准确"的参考值（ground truth）。
- 自建点（附件2）：低成本电化学/光学传感器，约每 1～2 分钟一条记录，除六项污染物外还有风速、压强、降水量、温度、湿度五个气象量；题目明确"相应于国控点时间且间隔在5分钟内"，即每个国控点整点时刻附近 5 分钟内能找到对应的自建点记录，用于配对比较。
- 题目原文已经把"差异原因"指向三类物理机制：**零点漂移**、**量程漂移**（长期使用累积的仪器老化误差）、**非常规气态污染物之间的交叉干扰**（电化学传感器对气体选择性不完美）、**天气因素**（温湿度会改变气体扩散速率和电化学反应速率，湿度还会让光学法测颗粒物因吸湿增长而虚高）。这是命题人已经暗示的建模线索，不能忽视，必须在模型里体现，而不是套一个黑箱模型了事。

所以这道题本质上是一个"传感器输出 → 真值"的**统计定标（calibration）问题**，属于回归/多元统计范畴，而不是需要发明复杂机理方程的物理题。核心难点不在"选什么算法"，而在于：
(a) 先做对 EDA 把六种污染物、差异结构、致偏因素搞清楚；
(b) 校准模型要不要用气象量和交叉气体量做协变量、要不要考虑随时间变化的漂移，这些都要用真实数据说话，不能想当然。

## 三、配对方法的选择

题目说自建点记录"间隔在5分钟内"对应国控点整点时刻。候选做法：
- 方案A：对每个国控点时刻，取 [T, T+5min] 或 [T-5min, T] 单侧窗口内第一条记录；
- 方案B：用 `pandas.merge_asof(direction='nearest', tolerance=5min)`，取时间上最近的一条自建点记录。

选择方案B——它对整点前后都适用，且是"最近"匹配，最贴近"取当时刻自建点观测值"的物理含义，不会因为自建点这一分钟恰好没数据而人为选到 5 分钟窗口另一端明显更远的点。

代码与结果：
```python
merged = pd.merge_asof(df1, df2, on='时间', direction='nearest',
                        tolerance=pd.Timedelta('5min'), suffixes=('_国控','_自建'))
# merged shape: (4200, 18)
# 未匹配到自建点记录的行数: 148   （占4200的3.5%，说明自建点也有零星缺测/超出5分钟容差的情况）
```
丢弃这 148 条缺失配对后，得到 **4052 组有效配对样本**，作为后续全部分析和建模的基础数据集。

## 四、探索性数据分析（第1问）的真实结果

对 4052 组配对样本逐污染物计算 Pearson 相关系数、RMSE、MAE、平均偏差（自建-国控）：

```
污染物   Pearson r   校准前RMSE   MAE      平均偏差(自建-国控)  国控均值   自建均值
PM2.5    0.914       22.895       18.403   +16.262             55.738     71.999
PM10     0.646       69.214       50.477   +38.800             82.653    121.453
CO       0.316        0.686        0.554   -0.503               1.112      0.609
NO2      0.375       38.392       30.503   +24.689              32.827     57.517
SO2      0.026       31.112       12.761   -5.558               22.072     16.514
O3       0.429       46.346       37.199   +12.227              55.575     67.802
```

第一次看到 SO2 的 r=0.026 时以为数据有问题，专门去查了 SO2_自建 的分布：`min=2, 25%=14, 50%=16, 75%=17, max=1103`——存在极端离群值（自建点SO2最高冲到1103μg/m³，而国控点同期最高只有150）。去掉自建点SO2最高1%的样本后重算相关系数只从0.026升到0.215，说明**不只是离群点的问题，SO2本身两套系统就几乎不呈线性关系**，随后用 Spearman 秩相关做稳健性交叉验证：

```
PM2.5 spearman=0.903   PM10 spearman=0.648   CO spearman=0.243
NO2 spearman=0.234     SO2 spearman=0.068    O3 spearman=0.165
```

结论：颗粒物（PM2.5/PM10）自建点和国控点相关性尚可（这与它们采用光学散射原理、干扰机制相对单一有关）；四种气体（CO/NO2/SO2/O3）自建点与国控点的相关性都偏弱，尤其 SO2 几乎不相关——这符合电化学气体传感器易受温湿度与交叉气体干扰的已知工程事实，也印证了题目原文强调"电化学气体传感器…交叉干扰…天气因素影响"并非虚指，而是这批数据里真实存在的现象。这一发现直接决定了后面建模不能只做"自身读数的一元线性拟合"。

## 五、差异成因分析（第2问）的真实结果

### 5.1 零点/量程漂移的证据：偏差与"运行时长"的相关性

用配对样本第一条记录的时刻作为 t=0，构造 `t_hours`（累计运行小时数），计算偏差(自建-国控)与 t_hours 的相关系数：

```
PM2.5 r(bias,t)=-0.389   PM10 r(bias,t)=-0.538   CO r(bias,t)=0.046
NO2  r(bias,t)=-0.404    SO2 r(bias,t)=+0.315    O3 r(bias,t)=-0.641
```
除 CO 外，五个量的偏差都与运行时间显著相关（|r| 在 0.3~0.64 之间），说明**偏差不是一个恒定常数，而是随仪器使用时间系统性漂移的**——这是支持"零点/量程漂移"假说的直接证据，也提示单纯拟合一个全时段固定的线性校准系数是不够的（后面建模阶段会验证这一点并因此调整方案）。

### 5.2 天气因素的证据：偏差与气象变量的相关性

```
污染物   风速     压强     降水量    温度     湿度
PM2.5   -0.06    +0.34    +0.17    -0.48    +0.57
PM10    -0.11    +0.39    +0.20    -0.51    +0.60
CO      +0.16    +0.02    -0.00    +0.18    -0.22
NO2     -0.04    +0.14    +0.42    -0.13    +0.35
SO2     +0.05    -0.11    -0.22    +0.04    -0.04
O3      -0.24    +0.38    +0.31    -0.46    +0.40
```
颗粒物和 O3 的偏差与**湿度**（+0.57~+0.60）、**温度**（-0.46~-0.51）关联最强——湿度升高会让光学法测颗粒物因吸湿增长而读数虚高，这与自建点PM2.5/PM10均值（72.0/121.5）明显高于国控点（55.7/82.7）的系统性正偏差方向一致，是可以用物理机制解释的真实规律，不是拟合噪声。

### 5.3 交叉干扰的证据：自建点污染物之间的相关性 & 偏差与其它气体读数的相关性

```
自建点读数互相关（节选）：
CO_自建 vs O3_自建  r=0.51（明显交叉）
CO_自建 vs NO2_自建 r=0.26
PM2.5_自建 vs PM10_自建 r=0.96（合理，同属颗粒物）

各污染物"偏差"与"自建点其它气体读数"的相关：
O3 偏差 vs NO2_自建 读数  r=+0.51（最强的交叉干扰信号）
O3 偏差 vs CO_自建  读数  r=+0.31
CO 偏差 vs NO2_自建 读数  r=-0.27
SO2偏差 vs O3_自建  读数  r=-0.25
```
O3 的偏差与 NO2、CO 自建点读数关联最明显，符合电化学传感器工作原理（O3传感器常对NO2有正向或负向交叉响应，这在传感器厂商的技术文档里是公开承认的已知缺陷）。这给了我们把"其它气体的自建点读数"作为校准模型协变量的实证依据，而不是凭空加特征。

## 六、候选建模思路的比较与取舍（第3问）

在看到上面 EDA 结果之后，我列了以下候选方案：

**方案1：一元线性回归**（y_国控 = a + b·x_自建，每个污染物单独拟合）
最简单，也是很多同类论文的"标准答案"。但 EDA 已经显示气体污染物的自身相关性很弱（NO2/SO2/O3 的 Pearson r 都不到 0.45），单靠自身读数做线性拟合天花板很低，而且没有用到题目反复强调的"天气因素""交叉干扰"这两条线索——如果只做这一种模型，等于没有回应第2问分析出的成因。保留作为**基线对照**，不作为最终模型。

**方案2：多元线性回归（自身读数 + 五个气象变量 + 运行时间漂移项 + 其它气体交叉项）**
直接把第2问里验证过的三类致偏因素（漂移、气象、交叉干扰）都写成协变量，物理意义清楚、可解释性强、参数量适中（4052个样本、约9个特征，不会欠定）。这是与本题EDA发现最吻合的方案，作为**主力模型**。

**方案3：非线性机器学习模型（随机森林）**
灵活、免于设定函数形式，作为**能力上限的参照**，用来检验线性模型是否明显欠拟合。同样使用方案2的特征集训练。

**方案4（在实测中被"逼出来"的方案）：滚动窗口周期性再标定**
这不是一个新的"函数形式"，而是**模型使用方式**上的调整——在做完静态训练/测试划分后（见下一节"遇到的问题"），发现漂移是非平稳的，一次性拟合的系数会在几个月后失效，因此提出"每隔一段时间用最近一个窗口的国控点数据重新拟合方案2的系数"，用滚动窗口回测检验这个策略是否真的能挽救静态模型的失效。事后看，这其实是解决第3问"如何校准"最贴近工程落地的答案：不是"用一套公式管全程"，而是"建立一套可定期重新标定的公式"。

## 七、关键假设及其论证

1. **国控点数据视为真值（ground truth）**——题目原文明确国控点"较为准确"，且是校准的参照基准，这是题目给定的建模前提，不是我们自行假设。
2. **国控点整点时刻与自建点最近5分钟内记录可视为同一时刻的观测**——题目原文直接给出这个配对规则（"相应于国控点时间且间隔在5分钟内"），5分钟内污染物浓度变化通常远小于观测噪声（一小时尺度的污染物浓度变化本身是缓变过程），配对误差可忽略。
3. **偏差主要由零点/量程漂移、气象、交叉气体干扰三类因素解释，可用线性叠加近似**——由 EDA 中偏差与运行时长、气象变量、交叉气体读数分别存在中等强度线性相关（|r| 普遍在0.2~0.6）的实证支持；线性叠加是一阶近似，不排除更高阶交互项，但样本量和可解释性的平衡下先用线性形式，用随机森林做上限对照检验欠拟合程度。
4. **漂移在几十天的时间尺度内可视为近似平稳（分段线性/慢时变）**——由静态整段拟合失效、而30~45天滚动窗口拟合明显奏效这一实测结果反推支持（见下一节），不是先验假设，是根据实验结果修正后确认的假设。

### 8.2 遇到的问题：静态划分下气体污染物校准"越校越差"

PM2.5/PM10 在方案2（模型C）下明显优于校准前（PM2.5 RMSE从14.65降到8.81，R²从0.47升到0.81），符合预期。但 **CO、SO2 在所有模型下 R² 都是负的、比不校准还差**；NO2、O3 在方案2/3下也比不校准（baseline）或比一元线性（模型A）更差。这不是代码 bug（多次核对特征列和拟合流程无误），而是一个真实的建模问题：**训练集覆盖2018年11月～2019年4月，测试集是2019年4月～6月，模型学到的是训练期的漂移趋势/气象-偏差关系，外推到测试期时这些关系已经变了**，尤其是漂移项 `t_hours` 做线性外推时，测试期越往后系统性误差越大，多元模型反而比只有截距和斜率的一元线性更容易被"带偏"。

如果止步于此，会得出"多元回归模型不如一元回归"的错误结论，与EDA阶段的证据（气象、交叉干扰确实与偏差强相关）矛盾。于是做了一个诊断实验来搞清楚问题出在"模型形式"还是"外推方式"：

```python
# 随机5折交叉验证（打乱顺序，训练/测试都来自同一时间跨度，属于"内插"而非"外推"）
PM2.5  校准前RMSE=22.883  校准后RMSE=11.318  改善=50.5%
PM10   校准前RMSE=68.968  校准后RMSE=29.693  改善=56.9%
CO     校准前RMSE= 0.686  校准后RMSE= 0.421  改善=38.5%
NO2    校准前RMSE=38.382  校准后RMSE=17.543  改善=54.3%
SO2    校准前RMSE=28.172  校准后RMSE=13.710  改善=51.3%
O3     校准前RMSE=46.336  校准后RMSE=24.544  改善=47.0%
```
**在同分布内插场景下，六种污染物全部获得38%~57%的RMSE改善**——证明"线性+气象+交叉气体项"这个模型形式本身是有效的，方案2的失败纯粹是"训练期与测试期分布/漂移状态不同"造成的外推问题，不是模型选错了特征。

### 8.3 调整方案：滚动窗口周期性再标定

既然问题是"漂移导致模型系数过期"，对策就是"定期用最近数据重新拟合系数"，而不是从全部历史数据里学一套一劳永逸的公式：

```python
train_window = 30天,  predict_window = 7天  # 用过去30天拟合，预测未来7天，然后向前滚动
PM2.5  n_test=3378  校准前RMSE=22.265  滚动校准后RMSE=11.807  改善=47.0%
PM10   n_test=3378  校准前RMSE=64.497  滚动校准后RMSE=34.167  改善=47.0%
CO     n_test=3378  校准前RMSE= 0.697  滚动校准后RMSE= 0.439  改善=37.0%
NO2    n_test=3378  校准前RMSE=33.103  滚动校准后RMSE=28.654  改善=13.4%
SO2    n_test=3378  校准前RMSE=29.650  滚动校准后RMSE=23.104  改善=22.1%
O3     n_test=3378  校准前RMSE=42.762  滚动校准后RMSE=26.054  改善=39.1%
```
六种污染物全部获得正向改善（13%~47%），CO和SO2这两个静态划分下"校准后更差"的量，在滚动策略下也都变成了"校准后更好"（CO改善37.0%，SO2改善22.1%）——问题被解决了。NO2/SO2 改善幅度相对偏小（13%~22%），说明气体交叉干扰的非线性/时变成分比颗粒物更复杂，线性滚动模型对它们的拟合能力仍有上限，这一点在论文的"模型评价"里如实写明，不夸大。

```python
   60      10.957    34.104   0.718    24.177    18.891    26.526
   90      10.358    35.065   0.442    17.758     8.825    28.085
```
窗口=10天时RMSE直接爆炸（PM2.5从预期的十几飙升到935）。查了一下原因：10天窗口内训练样本约195条，特征却包含5个气象量+若干交叉气体项，这些特征彼此相关性较高（如温度和湿度、CO自建和O3自建），短窗口内气象条件变化范围窄，线性回归系数在这种近共线的小样本上方差很大，一旦下一个7天测试窗口的气象条件略微超出训练窗口覆盖的范围，线性外推就会被放大。这是**偏差-方差权衡**的直接体现：窗口太短→方差爆炸（拟合不稳定）；窗口太长→拟合的是过时的漂移状态，偏差增大。30～45天是相对稳健的折中，NO2/SO2 在90天窗口下反而更好，说明它们的"有效信息"需要更长历史才能稳定估计，这与它们本身信噪比低（EDA阶段Pearson r本来就只有0.03~0.38）是一致的。

## 九、结果合理性检验

- 国控点六项污染物均值量级（PM2.5≈56μg/m³、PM10≈83μg/m³、CO≈1.1mg/m³、NO2≈33μg/m³、SO2≈22μg/m³、O3≈56μg/m³）与中国北方城市冬春季（2018年11月至2019年6月）典型的地面监测站数据量级一致，未见异常，说明数据本身可信、我们的读取和单位处理没有出错。
- 自建点PM2.5/PM10均值系统性高于国控点（72.0 vs 55.7；121.5 vs 82.7），方向与"光学法测颗粒物受湿度影响虚高"的已知机理一致，不是随机噪声。
- 校准模型（模型C）在同分布内插和滚动再标定两种更贴近实际部署的检验方式下都取得正向且量级合理的RMSE改善（13%~57%），且改善幅度与EDA阶段观测到的原始相关强弱基本对应（PM2.5/PM10原始相关性最强、改善也最明显；SO2原始相关性最弱、改善幅度也最小），逻辑自洽。

## 十、自我批判：局限与如果重来会怎么改

1. **配对方法的粗糙**：用最近邻5分钟匹配而非窗口内多点平均，单点噪声会被直接带入训练标签；更细致的做法是对5分钟窗口内的自建点多条记录取中位数或线性插值到整点。
2. **漂移建模仍是"分段线性"的工程近似，不是真正的状态空间模型**：更严谨的做法是用卡尔曼滤波或高斯过程回归对漂移系数做连续时变建模，本次因时间限制用"滚动窗口重新最小二乘拟合"来近似，量化上已经证明有效，但理论上不如显式状态空间模型精细。
3. **交叉气体项只用了自建点原始读数的线性项，没有做二阶/交互项筛选**：EDA显示O3偏差与NO2、CO读数都相关，可能存在NO2×温度这类交互效应，本次为控制模型复杂度和过拟合风险未展开，如果有更多时间应该用Lasso/逐步回归做特征选择后再决定是否引入交互项。
4. **随机森林只作为线性模型能力上限的参照，没有认真调参**（`n_estimators=300, max_depth=8`是一次性选定的经验值，没有做网格搜索），如果要把它当正式候选模型，需要更严谨的调参和防过拟合验证。
5. **只处理了一个国控点/一个自建点的配对数据**，模型的普适性（换一个自建点设备、换一个城市）没有验证，题目本身也只给了一组数据，这是数据条件的固有限制，论文里会明确写清楚"本模型的参数是针对这一对站点标定的，迁移到其它站点前需要重新标定"。
