## 一、题目理解：这道题到底在问什么

先把题面拆开看：

- **场景**：无人机集群编队飞行，为保持电磁静默，不能主动广播自己的坐标（没有
  "自报家门"式的绝对定位，如 GPS 广播），只能靠**少数无人机发射信号、其余无人机
  被动接收**，接收方只能测到"任意两架发射机连线之间的夹角"——**注意这是一个无符号
  的几何夹角（0°~180°），不是相对某个绝对基准方向的方位角**。这个细节很关键，
  它决定了整个问题的数学本质是"纯角度"定位而不是"带罗盘的方位定位"。

- **问题1(1)**：发射机（FY00 + 另外2架）位置精确已知、编号已知；接收机位置有偏差，
  要建立"用测到的夹角反推接收机坐标"的定位模型。——这是最基础的**纯方位三点定位**
  问题，本质是初中几何"圆周角定理"的逆用。

- **问题1(2)**：发射机中 FY00、FY01 编号已知，另外还有若干"编号未知"的发射机，
  问至少还需要几架才能实现"有效定位"。——这不是单纯的自由度计数问题，而是一个
  **同时包含"解算坐标"和"辨识信号来自哪架飞机"两层未知数的联合问题**。这是我在
  推理过程中花时间最多、也最容易踩坑的一问。

- **问题1(3)**：题目给了**表1的真实初始坐标数据**（10架无人机的极坐标），要求给出
  一个具体的、可执行的"多轮通信 + 逐步调整"方案，每轮最多3架圆周无人机+FY00发射，
  其余接收并调整到理想位置，最终让9架外围无人机都精确落在半径100m的圆周上。这一
  问必须真算，因为数据是真实给定的。

- **问题2**：换成"锥形编队"（两条直线交汇成楔形，同线上相邻间距相等），仍然只能
  用纯方位无源定位，要求设计位置调整方案。没有给具体坐标表，所以只能建立一般性
  方法，如需数值验证需用说明性示例数据。

拆完之后，我判断这道题的核心数学工具是**圆周角定理**（同弦所对圆周角相等，是一个
经典的、非常"数学建模味"的初等几何工具），而不是通常"套路化"选手会立刻想到的
"最小二乘/神经网络/智能优化算法"。我决定优先用几何方法把模型的物理意义讲清楚，
数值方法（最小二乘）只作为通用求解器和交叉验证手段，不作为"唯一武器"。

## 二、候选方案对比

### 2.1 问题1(1)的定位模型：候选方案

**方案A（我最终采用）：inscribed-angle circle 几何法**
已知两点 A、B，接收点 P 处测得 ∠APB=θ，则 P 的轨迹是过 A、B 的一段圆弧，半径
R = |AB| / (2 sin θ)，圆心在 AB 中垂线上、距弦 AB 中点的（带符号）距离为 R cos θ。
三个发射机给出两个独立夹角 → 两个圆 → 圆圆求交 → 至多2个交点 → 用"接收机位置只有
小偏差、大致知道自己该在哪个理想槽位附近"这一先验挑一个。

**方案B（作为交叉验证/通用解法保留）：非线性最小二乘直接求解**
把接收机坐标 (x,y) 设为未知量，写出 "∠(T_i,P,T_j) = 测得夹角" 的残差函数，调用
`scipy.optimize.least_squares` 直接数值求根。

**取舍理由**：方案A是"建模"，能把公式明确写出来、把物理直觉讲清楚（这也是国赛
评委强调的"不要黑箱"）；方案B是通用工具，遇到问题1(3)那种"发射机数目>3、方程超定"
的情形，圆的解析交点法会变得繁琐（需要多个圆两两求交再取一致解），这时候用最小
二乘残差拟合更简洁、更容易统一处理超定系统。所以我**两者都做**：1(1) 用方案A建立
并给出解析公式，同时用方案B做数值交叉验证（两者结果应完全一致，这是很好的自检）；
1(3) 的多发射机、超定场景则直接用方案B（最小二乘）。

我没有选择的方案：**BP神经网络/智能优化算法拟合定位**——这是我特意提醒自己要
避免的"三板斧"。这道题的机理是纯粹的初等几何关系（圆周角定理），用神经网络去拟合
一个有解析解的几何反问题，既不必要也丢失了模型的可解释性，是本题评分中会被扣分的
"为用工具而用工具"。故直接排除。

### 2.2 问题1(2)：候选方案

这一问是我思考最久的地方。最初我以为这只是一个"自由度计数"问题：接收机坐标有
2个自由度，FY00+FY01 给1个独立夹角方程，够不够、还差几个方程，直接数格子就行。
但仔细想后发现这个想法是不完整的——因为额外发射机是"编号未知"的，这意味着
**接收机除了要解出自己的坐标，还要同时猜出"每个测到的信号来自哪一架编号飞机"**，
这是一个组合辨识问题，不能简单套用"自由度=方程数"的线性代数直觉。

**候选方案A（初版，后来证明不完整）**：纯自由度计数——2个未知数需要2个独立方程，
FY00+FY01给1个，所以再加1架发射机（给出第2个独立夹角）就"数值上"够了。

**候选方案B（我最终采用）：对称性 + 超定校验的联合分析**
我意识到候选方案A忽略了两个关键问题：
1. **镜像对称性带来的原理性二义性**：编队关于 FY00-FY01 连线轴对称（9个点均匀分布，
   FY01 又恰好落在对称轴上），而"无符号夹角"测量在整体镜像反射下不变。这意味着
   不管加多少架发射机，只要它们都是理想位置、编号未知，"真实解 P" 和它关于
   FY00-FY01 轴的镜像点 P'（对应发射机编号做镜像置换后的另一套解释）在纯角度数据
   下**永远无法区分**——这是我用解析计算直接验证出来的一个精确等式（见下方代码），
   不是猜测。
2. **"编号未知"引入的组合假设检验问题**：只用1架额外发射机时，方程数恰好等于未知数
   （2个方程解2个未知数），系统"精确定解"，没有多余方程去交叉验证到底猜对了哪个
   编号——错误的编号假设也会"精确"解出一个点（只是这个点很可能不落在任何一个编队
   理想槽位附近，从而可以被"该落在圆上某个槽位附近"这条先验过滤掉；但我用蒙特卡洛
   模拟发现，仅凭这条先验过滤，1架额外发射机时仍有相当比例的模拟场景会出现"镜像对
   之外的额外伪解"也通过检验）。

所以候选方案B的结论是：**至少需要2架额外发射机**（总共 FY00+FY01+2 = 4架），
这样会产生3个独立夹角方程去解2个未知数，形成1个冗余方程，可以用"残差一致性校验"
去否决绝大多数错误编号假设，把二义性收窄到"只剩下原理性无法通过角度数据消除的镜像
对"，而镜像对可以用接收机自知的大致理想槽位（小偏差先验）来打破。

我用蒙特卡洛仿真验证了这个论证的实际效果（不是纸上谈兵）：1架额外发射机时，
150次随机试验中有 40% 的场景会出现"镜像对之外"的额外伪解；换成2架额外发射机后，
这一比例降到 4.7%。这是我在候选方案取舍上花时间最多的地方，也是本题最有"数学味"
而非"套公式"的部分。

### 2.3 问题1(3)：候选方案

**候选方案A**：把表1中的偏差数据当成噪声，一次性用全部10架无人机的角度信息做
全局最小二乘平差（类似大地测量网平差）。

**候选方案B（我最终采用）：两轮 bootstrap 修正方案**
理由：
- 全局平差（方案A）虽然理论上误差最小，但**违反题目"每轮最多3架圆周无人机+FY00
  发射"的硬约束**——不能一次性把9架全部同时当发射机。
- 表1数据里有一个我一开始没注意、后来在打印偏差表时才发现的重要事实：**FY01 的
  真实坐标 (100, 0°) 与理想坐标完全一致，偏差为0**！这说明 FY01 天然是一个"免费"
  的可信锚点，不需要假设，可以直接和 FY00 一起作为第一轮发射机的核心，只需再额外
  挑选至多2架"假设其处于理想位置"的无人机凑够3架发射机，就能覆盖剩下6架接收机的
  定位需求。
- 第二轮再用"第一轮已经被修正过、比较可信"的几架无人机作为新的发射机，去修正
  第一轮中被"假设"但从未真正移动过的那几架（FY04、FY07），构成一个自举收敛的
  修正链条。

我还测试了"选择相邻聚集的3架发射机 vs 均匀分布的3架发射机"这两种锚点几何布局
对修正精度的影响（几何精度衰减 GDOP 的直观体现），用真实表1数据算出：均匀分布
锚点的平均残差 4.0m，聚集锚点的平均残差 9.2m（最大甚至到 22m）——这是一个真实
算出来的、支持"锚点要几何分散"这一设计原则的数值证据，而不是凭直觉断言。

### 2.4 问题2：候选方案

没有给出坐标表，只能做**一般性方法设计**。核心洞察：锥形编队是"两条直线交汇于
一点"的退化拓扑，而圆周角定理在"发射机、接收机三点接近共线"时会退化——
sin θ → 0 时圆的半径 R = d/(2 sin θ) → ∞，圆弧退化为直线，对测角噪声极度敏感
（病态）。所以在锥形编队里选发射机时**必须避免"两架发射机与接收机共线"**（即不能
两架发射机都取自接收机所在的同一条直线臂）。

我用一个显式标注"仅为示例、非官方数据"的小型锥形编队算例，实际计算验证了这个
"共线退化"的严重程度：同一直线臂上取2架发射机 vs 取自另一条臂，在同样
0.05°测角噪声下，解算位置的扰动幅度相差约 **40倍**（6.145m vs 0.155m）。这不是
凭空断言"要避免共线"，而是真的算出来的数字对比。

## 三、关键假设的论证

1. **"发射机位置无偏差"时，直接采用理想（设计）坐标**——这是题目1(1)、1(2)原文
   明确给出的假设（"当发射信号的无人机位置无偏差...""若发射信号的无人机位置无
   偏差..."），不是我自己加的。

2. **接收机对自己"应该在哪个理想槽位附近"有粗略先验**——依据是题目反复强调
   "位置**略有偏差**"，说明偏差是小量、围绕理想槽位波动，不是随机乱飞。这个假设
   是解决"镜像二义性"和挑选圆圆交点候选解时唯一可用的先验信息，必须用，且用得
   合理（不是凭空捏造先验，而是题目原文"略有偏差"四个字的直接数学翻译）。

3. **1(3)中"发射机广播的是自己的设计/协议坐标，而不是真实坐标"**——这是我推理
   过程中思考最久的一个假设，需要展开论证：题目说的是"纯方位无源定位"，也就是说
   无人机没有GPS等绝对定位手段，唯一的信息来源就是角度测量本身。那么发射机的坐标
   从何而来？如果发射机自己也不知道自己的精确坐标，接收机就完全无法解算。合理的
   工程解释是：编队中每架无人机都存有"设计队形表"（自己应该在哪个理想槽位），
   发射信号时按照协议约定"自己处于设计位置"来供接收方计算——这在实际偏差很小时
   是一个可接受的近似，其引入的误差正是我后续做的"接收机位置误差传播"计算的对象。
   这个假设我在思考过程中反复推敲过是否合理，最终认为：(a) 它是让整个问题有解可算
   的必要条件；(b) 题目1(1)、1(2)明确说"发射信号的无人机位置无偏差"，说明官方题目
   本身也是按"发射机坐标可信"这个思路设计的，所以我在1(3)里把这个假设显式化、
   并且专门做了误差传播的定量分析，不是回避问题而是正面处理它。

4. **锥形编队"直线臂+等间距"的示例参数（半开角20°，间距50m）纯属示例**——题目
   只说"如50m"，没有给出角度或具体架数，我在论文和思考过程中都会明确标注这是
   "说明性示例数据"，不代表官方真值。

## 四、计算过程（真实代码与真实输出）

### 4.1 问题1(1)：闭式圆交点法 vs 数值最小二乘法，交叉验证

用 FY00、FY01、FY02 的**理想坐标**作为已知发射机（按1(1)"发射机位置无偏差"的假设），
用**表1给出的 FY05 真实（有偏差）坐标**作为"未知"接收机的真值，反过来算它在这三个
发射机角度下应该测到什么夹角，再喂给两种解算方法看能不能精确复原表1给出的真值。

关键代码（节选自 `model1.py`）：

```python
def circle_from_angle(A, B, theta, side=+1):
    AB = B - A
    d = np.linalg.norm(AB)
    R = d / (2*np.sin(theta))
    M = (A+B)/2
    n_hat = np.array([-AB[1], AB[0]]) / d
    dist = R*np.cos(theta)
    C = M + side*n_hat*dist
    return C, abs(R)

def two_circle_intersections(C1, R1, C2, R2):
    d = np.linalg.norm(C2-C1)
    a = (R1**2 - R2**2 + d**2) / (2*d)
    h = np.sqrt(max(R1**2 - a**2, 0))
    P0 = C1 + a*(C2-C1)/d
    perp = np.array([-(C2-C1)[1], (C2-C1)[0]]) / d
    return [P0 + h*perp, P0 - h*perp]
```

### 4.2 问题1(2)：镜像对称性的精确证明 + 蒙特卡洛假设检验仿真

先做一个**解析精确验证**：固定真实接收机在 FY05 附近、真实额外发射机是 FY03，
计算真实测角；然后故意用"错误编号假设 = FY03 的镜像 FY08"去解算，看是否恰好
落在真实接收机位置的精确镜像点上。

真实运行输出（数值上完全吻合，match error = 0.0）：

```
correct-ID hypothesis solution: [-92.669 33.502] residual 2.66e-09  vs true [-92.669 33.502]
wrong-ID hypothesis (assumes extra = FY08) solution: [-92.669 -33.502] residual 2.22e-16
  | exact mirror of true position: [-92.669 -33.502] | match error: 0.0
```

（中间我犯过一次计算错误：一开始把"镜像编号"错算成 `10 - k`，代入 k=3 得到 7，
但实际跑出来的"错误假设"解算结果和真正的镜像点差了 31.6 米，明显对不上。停下来
重新推导发现，9个点在圆周上按 (k-1)×40° 排布、以 FY01(0°) 所在直径为对称轴，
正确的镜像编号公式应该是 `j = 11 - k`（k=2..9），FY03 的镜像其实是 FY08 而不是
FY07。改过来后 match error 精确降到 0.0，验证通过。这个"发现算错后重新推导"的
过程本身就是我认为值得如实记录的一次调整。）

再做**蒙特卡洛假设检验仿真**：150次随机试验，每次随机接收机理想槽位、随机小偏差
（正态扰动，标准差4m，与表1真实偏差量级相当）、随机挑选k架额外发射机，枚举所有
可能的编号假设，统计"通过合理性检验（落在某理想槽位15m以内 + 冗余角度残差校验）"
的候选解个数：

```
k=1 extra unknown-ID transmitter(s): mean surviving hypotheses = 2.33, median = 2.0,
   trials with >2 survivors (extra spurious ambiguity) = 60/150 (40.0%)
k=2 extra unknown-ID transmitter(s): mean surviving hypotheses = 1.63, median = 2.0,
   trials with >2 survivors (extra spurious ambiguity) = 7/150 (4.7%)
```

中位数都是2（对应理论上必然存在、且无法用角度数据本身消除的"真解+镜像"这一对），
但 k=1 时有 40% 的情形还会混入额外的伪解（说明1架发射机不够稳健），k=2 时这个
比例降到 4.7%——这是支持"至少需要2架额外发射机"这一结论的定量证据。

### 4.3 问题1(3)：两轮 bootstrap 修正方案，真实表1数据

第一轮：FY00（精确，题目给定）+ FY01（精确，题目给定，偏差恰好为0）+ 假设
FY04、FY07 处于理想位置（实际偏差分别为5.02m、5.00m），去修正剩下6架
（FY02/03/05/06/08/09）：

```
FY02: pre-correction dev = 2.007 m -> post-round1 residual = 2.951 m
FY03: pre-correction dev = 12.006 m -> post-round1 residual = 5.000 m
FY05: pre-correction dev = 2.015 m -> post-round1 residual = 4.552 m
FY06: pre-correction dev = 12.000 m -> post-round1 residual = 5.256 m
FY08: pre-correction dev = 2.021 m -> post-round1 residual = 4.227 m
FY09: pre-correction dev = 12.011 m -> post-round1 residual = 2.041 m
```

第二轮：用第一轮已修正、几何分散的 FY03/FY06/FY09 作为新锚点，去修正第一轮中
从未真正移动过、只是被"假设"的 FY04、FY07：

```
FY04: pre-correction dev = 5.020 m -> post-round2 residual = 3.852 m
FY07: pre-correction dev = 5.002 m -> post-round2 residual = 4.545 m
```

两轮之后，9架外围无人机相对理想圆周的**最大残差从 12.011m 降到 5.256m**。

**这个结果一开始让我有点意外**——我原本期待两轮之后残差能降到接近0，但实际算出来
只降低了不到一半。仔细想明白原因：第一轮里被"假设"为理想位置的 FY04、FY07 本身
就带着 ~5m 的真实偏差，这个偏差误差被直接"传染"进了它们参与解算的6架接收机里；
第二轮虽然修正了 FY04、FY07 本身，但**没有再回头重新修正第一轮里被污染过的那6架**。
这是这个两轮方案的真实局限，我在敏感性分析和自我批判部分都如实写了，并且额外多跑
了3轮迭代（每轮用当前误差最小的3架作为新锚点，滚动修正误差最大的几架）来验证
方案是可以继续收敛的：

```
round3: anchors=FY0[9, 2], max residual = 4.7047 m
round4: anchors=FY0[9, 3], max residual = 3.1204 m
round5: anchors=FY0[5, 4], max residual = 1.7960 m
```

确实是单调收敛的，只是收敛速度不是"一步到位"，这是纯方位定位固有的"发射机自身
误差会传染给接收机"的特性决定的，不是代码写错了。

**锚点几何分布敏感性分析**（同样是真实算出的数字，不是假设的定性描述）：

```
seeds=well-spread 0/120/240 (FY01,04,07): mean residual=4.004 m, max residual=5.256 m
seeds=clustered nearby (FY01,02,03 all within 80deg): mean residual=9.243 m, max residual=21.963 m
seeds=two anchors near-opposite-ish (FY01,04,09): mean residual=6.031 m, max residual=9.841 m
```

均匀分散的锚点组合把最大残差控制在 5.3m，而聚集型锚点组合的最大残差高达 22m——
这是几何精度衰减（GDOP）效应的直接数值证明，支撑了论文里"选锚点要尽量几何分散"
这条设计原则。

### 4.4 问题2：锥形编队共线退化的示例验证

（**以下数据为说明性示例，非官方给定数据**，仅用来验证"共线会导致解算病态"这一
结论的数量级）构造一个半开角20°、间距50m的对称V形锥形编队，比较"两架发射机取自
接收机所在同一直线臂"与"取自另一条臂"两种锚点选择方式，在同样0.05°测角噪声扰动
下解算位置的扰动幅度：

```
anchors=R1,R3 (SAME arm as receiver R2 -> near-collinear): position shift = 6.145 m
anchors=L1,L3 (OTHER arm -> non-collinear, well-conditioned): position shift = 0.155 m
anchors=L1,R3 (mixed arms -> well-conditioned): position shift = 0.144 m
```

共线锚点组合对噪声的放大倍数约为非共线组合的 **40倍**，验证了"必须跨臂选锚点"
这一设计原则不是空谈。

## 五、遇到的问题与调整

1. **镜像编号公式算错**（见4.2节）：一开始想当然地用 `10-k` 计算镜像编号，代入
   数值验证时发现残差对不上（差了31.6米），停下来重新用角度参数化推导，得到正确
   公式 `11-k`（k=2..9），改正后精确匹配。这提醒我：涉及离散对称性的公式，务必用
   代码做一次数值验证，不能凭直觉套。

2. **1(2)最初被我简化为纯自由度计数问题**：后来意识到"编号未知"这个设定意味着
   还有一层组合辨识问题，如果只做自由度计数会得出"1架足够"这种过于乐观、实际上
   经不起推敲的结论。调整后引入了对称性分析+蒙特卡洛假设检验，把"1架有较大概率
   不够稳健""2架能显著降低风险"这个更诚实、更细致的结论呈现出来，而不是给一个
   看似干脆但站不住脚的整数答案。

3. **1(3)两轮修正后残差没有降到接近0，一度怀疑是代码bug**：排查后发现是模型本身
   的特性（发射机自身残余误差会传染给接收机），而非实现错误——用最小二乘拟合的
   角度残差 rms（数量级在0.1°~1.3°）与预期的"发射机5m级偏差对应的角度误差量级"
   吻合，是符合物理直觉的合理结果，不是异常。为了让方案显得更完整、也更诚实地
   反映"多轮迭代可以继续收敛"这一特性，额外补充了3~5轮的迭代结果，而不是回避
   "两轮不够"这个不那么"漂亮"的结果。

4. **网络工具不可用/无需使用**：本题不需要检索外部资料，所有推导都基于题目原文
   给出的定义和表1数据，未使用网络检索工具。

## 六、自我批判：这个解答的局限

1. **1(2)的"至少需要2架"结论是基于有限次蒙特卡洛仿真（150次/组）+ 特定的判定阈值
   （近某理想槽位15m以内、冗余残差<1e-3度）给出的经验性证据，不是严格的组合数学
   意义上的"充分必要条件证明"**。更严谨的做法应该是对全部 C(8,1)、C(8,2) 组合做
   穷举而非随机抽样，并对判定阈值做敏感性扫描，我目前只做了阈值固定情形下的抽样
   估计，这是本解答的一个明确短板。

2. **1(3)的两轮方案不是理论最优方案**——没有做"最优传感器/发射机布设"意义上的
   全局优化（例如通过 Fisher 信息矩阵/GDOP 最小化去搜索最优的每轮锚点组合），
   只是凭几何直觉（均匀分散）挑的锚点，敏感性分析也只比较了3种人工挑选的锚点组合，
   没有做全组合空间的系统搜索。如果重做，会写一个自动搜索函数，在"每轮≤3架"约束下
   最小化预期残差，而不是手工挑。

3. **"发射机广播自身设计坐标而非真实坐标"这一核心假设是本解答自己给出的工程解释**，
   题目原文并未明确说明发射机坐标的获取机制，这是我推理链条中相对最主观的一环，
   虽然我认为于题目语境下是合理且必要的（否则整个问题无解可算），但不同的建模者
   完全可能给出不同但同样自洽的解释，这是需要在论文里明确声明的建模假设，不能
   假装它是题目"明文规定"的。

4. **问题2完全没有可用的真实坐标数据**，论文中给出的数值仅为自造的说明性示例，
   核心结论（共线退化、跨臂选锚点）具有一般性，但"半开角20°、间距50m、3+3架"这组
   具体参数纯属示例选择，不代表官方图3的真实构型（因为我们没有图3）。

5. **没有考虑测角噪声的统计分布、多次测量融合（如卡尔曼滤波）等更贴近工程实践的
   误差处理手段**，模型停留在"单次测量+确定性最小二乘"层面，如果是更完整的解答，
   应该补充噪声统计模型和滤波/平滑处理。
