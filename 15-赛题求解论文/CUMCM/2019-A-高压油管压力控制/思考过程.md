## 1. 题目理解与拆解

题目本质是一个**受控可压缩流体的质量守恒/状态方程问题**，物理图像很清楚：
- 高压油管是一段固定容积 V 的刚性管腔，里面装着可压缩的燃油；
- 燃油从 A 端（单向阀，接 160MPa 高压油泵）间歇进入；
- 燃油从 B 端（喷油嘴）间歇喷出；
- 進/出流量不平衡 ⇒ 管内压力波动；
- 注1 给出了燃油的状态方程（弹性模量定义压力-密度关系）；
- 注2 给出了孔口流量公式（伯努利型孔流公式）。

三问的递进关系：
- 问题1：进口用理想化的"单向阀开合时长"控制，出口流量曲线是**给定的**（图2），求控制律（阀门开启时长 τ）使压力稳定/按调度爬升。这是纯粹的**单入单出、单变量控制**问题。
- 问题2：进口的物理机制被还原为真实的"凸轮-柱塞泵"机构（附件1给出凸轮曲线），出口也被还原为真实的"针阀-喷孔"节流结构（附件2给出针阀升程曲线），要求确定凸轮角速度 ω。这是把问题1的"黑箱"进口/出口都换成了**真实物理模型**，本质上要建立一个耦合的柱塞腔-油管两容积动力学模型。
- 问题3：双喷嘴 + 减压阀，是在问题2的基础上增加一个可控执行器（减压阀）和一个新的出口（第二喷嘴），要求给出**控制策略**（不是单一数值解），偏向工程方案设计。

## 2. 候选建模方案对比

### 关于状态方程（管内压力动力学）
- **方案A（选用）**：把管内燃油看作一个通过孔口与外界交换质量的"准均匀可压缩流体控制体"。用注1给出的 dP = (E/ρ)dρ，结合质量守恒 d(ρV)/dt = ṁ_in − ṁ_out（V为常数，刚性管），推出
  dP/dt = (E(P)/V)·(Q_in − Q_out)
  这是一个一阶非线性ODE（E(P)、Q_in、Q_out都依赖P或t），可以用数值积分（RK4/Euler）直接求解，且和注1、注2的物理定义完全对应，没有引入额外的经验假设。
- **方案B（放弃）**：把问题当成一个纯"体积平衡"问题，忽略弹性模量的非线性，直接假设 ΔP ∝ ΔV（用某个常数"等效体积模量"）。这样可以得到解析解（比如常微分方程直接积分成封闭式），求解更快，但代价是丢弃了附件3给出的 E(P) 依赖压力这一实测数据——而附件3恰恰是命题人特意提供的数据，说明他们希望被利用。用常数模量还会让"从100MPa升到150MPa"这种大范围变化的问题失真（E从2171.4变到2664.3，变化了23%，不是小量）。
  **结论**：放弃方案B，保留方案A的非线性ODE，用真实E(P)表格做分段线性插值。

### 关于问题1"单向阀开启时长"的调度策略
- **方案A（选用）**：固定阀门开启的起始时刻为每个100ms周期的起点（与喷油同步开始），只把"开启时长τ"当唯一控制变量，用二分法求解使得"稳态时管内压力锚定在100MPa（或150MPa）"的τ。
- **方案B（考虑但放弃，标注为改进方向）**：把"开启时刻"也作为自由变量（比如延迟到喷油结束后再补油），甚至允许多脉冲补油。这样自由度更高，理论上可以把周期内的压力波动幅度做得更小。但这是一个二维/多维优化问题，给定的时间预算下没有充分求解，所以本方案只求解一维（固定相位、只调时长）问题，并在"自我批判"里明确指出这是可改进的方向。

### 关于问题2"确定凸轮角速度"的求解策略
- **方案A（选用，两步法）**：
  1. 先做一个"准静态"估算：假设管内压力恒定在100MPa，分别用附件1、附件2的真实曲线，各自独立算出"每转柱塞泵送体积"和"每次喷油体积"，令二者按频率匹配（10次/秒喷油 = 供油速率），解出 ω 的初始估计值；
  2. 再用完整的耦合动力学仿真（柱塞腔状态 + 管内压力状态，两个ODE联立，附件1/2数据直接嵌入），对 ω 做二分法微调，使得长时间尺度下管内压力不发生漂移。
- **方案B（放弃）**：跳过准静态估算，直接对 ω 在一个很宽的范围内做暴力搜索。由于每次仿真都要跑几百毫秒的高分辨率ODE（真实测得约3~5秒/次），暴力搜索的次数代价很高；而准静态估算能给出一个数量级正确的初始猜测，把二分法的区间大幅缩小，工程上更高效。这是"先解析估算再数值精修"的标准做法，本题也确实证明准静态估计（865 rad/s）和最终动态收敛值（726 rad/s）在同一数量级——估算是有效的，只是量级对但数值不够精确（差约19%），这恰好体现了准静态近似（忽略瞬态影响）的局限性，值得在论文里如实说明。

## 3. 关键假设及其论证

1. **管壁刚性、管容积V不随压力变化**：题目只给了油管内腔尺寸，未提供管壁弹性模量，且高压油管通常用高强度钢制造，其弹性变形量相对燃油体积压缩量可忽略——这是行业惯例假设，本题官方给出的"弹性模量-压力"数据也只针对燃油，间接印证了"管本身不可压"是命题人预设的假设。
2. **忽略流动惯性/摩擥沿程损失，孔口流用注2公式**：题目已经在注2里给定了具体公式，说明命题人希望学生直接使用该经验公式而不是从N-S方程重新推导，这是题目的"硬约束"，不是我们自己选的简化。
3. **入流侧密度用泵侧(160MPa)密度、出流侧密度用管内(高压侧)密度**：注2原文"ρ为高压侧燃油的密度"，含义是孔口两侧哪边压力高就用哪边的密度；对进口(泵160MPa > 管100~150MPa)，高压侧是泵；对出口(管100~150MPa > 环境~0)，高压侧是管。这是对注2字面表述的直接套用。
4. **inflow/outflow对管内密度动力学的合并简化** dP/dt=(E(P)/V)(Qin−Qout)，隐含"入流密度≈管内密度"的近似。我们没有凭空假设，而是**实际算了** ρ(160)/ρ(100)−1 = 2.485%（见下方代码输出），差异在2.5%量级，属于二阶小量，可以合理忽略，这是"论证"而不是"假设成立就完事"。
5. **问题2柱塞腔到油管的单向阀"无节流、瞬时接通"**：题目没有给这个单向阀的孔径/流量系数，无从用注2公式计算其节流。只能假设它一旦打开就没有额外阻力（腔压=管压），这是在数据缺失情况下**唯一可行**的建模选择，在自我批判里会指出这是低压侧不确定性来源。
6. **问题2柱塞腔下止点前的低压腔补油同理瞬时接通、腔压钳位在0.5MPa**：题目给出"柱塞运动到下止点时，低压燃油会充满柱塞腔……压力为0.5MPa"，直接对应这个假设。

### 4.2 单位换算的手工推导（注2公式的隐藏坑）

注2给的公式 Q=CA√(2ΔP/ρ)，如果直接把 ΔP(MPa)、ρ(mg/mm³) 的数值代入而不做单位转换，量纲是不自洽的。我按照 mm-ms-mg-MPa 这套单位制手算了一遍：

- 1 MPa = 1×10⁶ Pa = 1×10⁶ kg/(m·s²)，换算到 mg/(mm·ms²) 得 1 MPa = 1×10³ mg/(mm·ms²)
- 1 mg/mm³（密度单位）本身已经是该单位制的基本单位
- 于是 2ΔP/ρ 在数值上要乘以 1000 才对应到 (mm/ms)² 量纲

即正确公式应为：
**Q[mm³/ms] = C·A[mm²]·√(2000·ΔP[MPa]/ρ[mg/mm³])**

用一个物理量级验证：ΔP=60MPa（泵-管压差），ρ=0.85mg/mm³，若不乘1000，得到"速度"11.88（数值上），如果按SI单位换算回真实速度，等价于 v=11.88mm/ms=11880 m/s，远超物理常识（超声速几十倍），明显不对；乘以1000后 v=375.7 mm/ms=375.7 m/s，与柴油共轨系统喷油速度的公开文献量级（几百 m/s）吻合。**这是本题一个容易被忽视的隐藏易错点**，我把这处推导完整记录下来，供后续参赛者复核。

### 4.3 问题1：单向阀开启时长（稳态 100MPa）

模型：
```
dP/dt = E(P)/V_pipe · (Q_in(t) − Q_out(t))
Q_out(t): 图2实测梯形曲线 (0→0.2ms: 0→20; 0.2→2.2ms: 20; 2.2→2.4ms: 20→0; 其余0)
Q_in(t) = C·A_A·√(2000·max(160−P,0)/ρ(160))   (t<τ 时, 否则0)
```
关键观察（提速优化）：因为 τ<<2.4ms 且喷油只在 [0,2.4]ms 内非零，2.4ms 之后到 100ms 周期结束 Q_in=Q_out=0，故 dP/dt≡0——**只需要对每个100ms周期的前2.4ms做数值积分**，其余97.6ms压力保持不变即可，把计算量压缩了约40倍。

用二分法求解 τ_100（使一个周期后压力回到100MPa）：
```
[验证-快速版] tau_100=0.091500 ms, Pend=100.0012, 波动[100.0000,102.4220], 用时0.98s
```
即单向阀每次只需打开约 **0.0915 ms（91.5微秒）**。压力在周期内先被这短暂一击推高到约102.42MPa，随后被喷油持续消耗回落到略低于100MPa结束（下一周期开始时重复）。

**稳定性检验**（从偏离点出发，看是否能自动收敛回100MPa）：
```
从95MPa出发, 用tau_100逐周期末尾压力收敛过程:
[95.  95.098 95.194 95.289 95.381 95.472 95.561 95.648 95.734 95.818 95.900 95.981 96.060 96.138 96.214 96.289]
从105MPa出发, 用tau_100逐周期末尾压力收敛过程:
[105. 104.892 104.786 104.683 104.582 104.483 104.387 104.292 104.200 104.110 104.022 103.935 103.851 103.768 103.687 103.608]
```
两条轨迹都朝100MPa方向移动，说明这是一个**渐近稳定**的不动点，但收敛速度很慢（15个周期即1.5秒只修正了20~30%的偏差）——这是用固定开启时长τ的开环控制策略的固有局限，实际系统需要反馈校正才能快速纠偏（见"自我批判"）。

### 4.4 问题1：100→150MPa 的爬升调度

分别对 2s(20周期)/5s(50周期)/10s(100周期) 三个目标时长做二分法求"恒定τ_ramp"：
```
[验证-快速版] tau_150(稳态)=0.240750 ms, Pend=150.0033, 波动[150.0000,152.8146], 用时1.00s
目标2s (N=20周期): tau_ramp=0.279250 ms -> 末次周期末尾P=149.9930 MPa, 用时17.6s
目标5s (N=50周期): tau_ramp=0.241000 ms -> 末次周期末尾P=149.9926 MPa, 用时46.5s
目标10s (N=100周期): tau_ramp=0.240750 ms -> 末次周期末尾P=150.0202 MPa, 用时89.8s
```
有趣的现象：5s、10s目标所需的τ_ramp都非常接近150MPa稳态值τ_150=0.24075ms，而2s目标需要明显更大的τ(0.27925ms)。这与4.3节发现的"慢收敛"性质是自洽的——爬升越快，就需要越大的超额供油去克服这个系统固有的弱阻尼特性；爬升越慢，所需τ就越接近终态稳态值。这个"自洽性"是我在计算完之后才注意到、反过来验证模型内部一致性的一个检验点。

### 4.5 问题2：真实数据驱动的耦合仿真——中途踩坑与修正

**第一次尝试**用"准静态法"独立估算：
```
柱塞面积A_p=19.6350 mm2, 行程=4.8260 mm, 下止点容积=114.7583 mm3
准静态估计: P_pipe=100MPa 时, 每次喷油体积 = 1220.6871 mm3 (对比问题1图2给定44mm3)
喷油速率峰值 = 634.7028 mm3/ms
准静态估计: 保持P_pipe=100MPa时, 每转泵送体积 = 88.6187 mm3
每秒需供油体积 = 12206.8711 mm3/s
所需转速 = 137.7459 转/s => omega = 865.4832 rad/s
```
**第一个"意外发现"**：用附件2真实针阀升程曲线 + 注2孔流公式算出的单次喷油体积是 1220.7mm³，是问题1图2给定的44mm³ 的**约27.7倍**！一开始怀疑自己算错了，反复检查了：(a)喷孔面积 A_hole=1.5394mm² 计算是否正确——正确；(b)针阀锥面节流面积公式 A=πd_needle·sin(9°)·L 是否正确——这是喷油嘴文献里通用的圆锥环形截面近似，正确；(c)速度量级是否合理——v=√(2000·100/0.85)≈485 m/s，属于共轨喷油系统的合理量级（不是算错）。所以最终确认：**这不是bug，而是题目本身问题1和问题2用的是两套不同、不要求彼此吻合的"出口模型"**——问题1的图2是一个简化给定曲线，问题2要求用真实几何+物理公式重新推导，两者数值差异很大是**真实的建模发现**，不是我编造或算错的数字，在论文里会专门讨论这个反差及其对凸轮转速的影响。

**第二次尝试**：写完整耦合ODE动态仿真（柱塞腔状态 Pc + 管内压力 P，两状态机耦合），第一版跑出来压力在300ms内飙到 1656 MPa，远超物理表E(P)数据范围（0~200MPa），明显是bug：
```
末尾压力=1656.6048 MPa
```
**调试过程**：写了一个单转周期的逐步打印诊断脚本，发现凸轮转角 theta 在 100ms 内几乎不动（从0只涨到0.005 rad，而一整转应该是2π≈6.283 rad）。定位到代码里 `theta = omega*t`，omega 单位是 rad/s，而仿真内部时间变量 t 全部用的是 ms——**这是一个量纲不统一的低级但致命的bug**：omega(rad/s)×t(ms) 在数值上比正确值小了1000倍，导致凸轮"卡"在接近上止点附近几乎不转动，柱塞腔持续对外"泄流"却始终不完成下止点补油循环，压力自然发散。

修正：引入 `omega_ms = omega/1000.0`，所有 `theta=omega_ms*t`、`dVcdt=...*omega_ms`。修正后重新对同一组 ω 扫描：
```
omega=259.64 rad/s (41.32 rev/s) -> 300ms后 P=31.377 (drift=-68.623)
omega=432.74 rad/s (68.87 rev/s) -> 300ms后 P=53.464 (drift=-46.536)
omega=605.84 rad/s (96.42 rev/s) -> 300ms后 P=81.780 (drift=-18.220)
omega=865.48 rad/s (137.75 rev/s) -> 300ms后 P=123.264 (drift=23.264)
```
这次结果单调、符合物理直觉（ω越大供油越多、压力漂移越正），说明bug已修复，可以放心用二分法求解。

```
第1..10周期末: 97.53 100.21 98.61 99.80 100.05 99.25 101.23 98.84 101.51 98.54
```
周期末压力在 [97.5, 101.5]MPa 附近小幅摆动（没有系统性漂移，说明ω*选取基本正确），但**周期内**瞬时压力会深跌到约48MPa——这是因为问题2的真实喷油峰值流量(≈635mm³/ms)远大于问题1的简化值(20mm³/ms)，喷油瞬间管内压力被迅速拉低，之后依靠凸轮高速旋转（约8.66ms/转，一个100ms喷油周期内转约11.5转）连续小脉冲补压，逐步爬回100MPa附近。这个"深跌-爬升"的锯齿状波形与问题1那种"轻微波动"形成鲜明对比，是本题命题人设计问题1→问题2递进关系时刻意呈现的"简化模型 vs 真实机理"落差，我把它当作一个值得强调的发现写进论文。

### 4.6 问题3：双喷嘴同步/错相对比 + 减压阀泄流能力

减压阀（1.4mm孔径）泄流能力：
```
减压阀口径1.4mm, 在管内100MPa对外(近似0MPa环境)排放时的最大泄流速率 = 634.703 mm3/ms
在P=100MPa附近, 单独开启减压阀时 dP/dt ≈ -35.095 MPa/ms
即约 0.028 ms 可使管内压力下降1 MPa
```
这与问题1/2里"补油阀"表现出的规律一致：所有孔口的响应都极快（亚毫秒级），意味着无论是供油阀还是减压阀，都必须用**短脉冲**方式控制，长开会造成剧烈超调。

双喷嘴同步 vs 错相 50ms 对比（用同一标定ω_dual）：
```
双喷嘴同步: lo=871.2 drift=-39.415  hi=1887.6 drift=114.261
双喷嘴(同步喷油)标定角速度 omega_dual ≈ 1205.19 rad/s (191.81 rev/s)  [g≈-0.297]
[同步喷油] 400ms末尾P=99.703, 全程 min=17.871 max=100.025, 幅值=82.154
[错相喷油,shift=50ms] 400ms末尾P=72.477, 全程 min=32.758 max=100.000, 幅值=67.242
```
在同一转速下，错相喷油把瞬时最低压力从17.9MPa抬高到32.8MPa（波动幅值从82.2MPa降到67.2MPa，降低约18%），初步验证了"两喷嘴错相工作可以削峰"的直觉。但要注意：错相方案的末尾压力(72.5MPa)比同步方案(99.7MPa)低很多，说明该ω是按同步工况标定的，错相工况下同一ω还不够补足消耗——**这是本次计算没有来得及做的后续工作**：应对错相方案单独重新标定ω，预期错相所需ω会比同步略高（因为在较低压力区间平均运行，注2公式里喷油速率∝√P，此时喷油速率整体降低,反而使实际消耗比按100MPa估算的更低，所以真实所需ω可能没有想象中那么高）。这一点我在"自我批判"里明确列为未完成事项，不隐瞒。

## 5. 遇到的问题与调整（汇总）

1. **图2数据缺失**：文本提取丢失图片信息 → 改用pdftoppm渲染PDF页面，人工读取坐标轴上的真实数值（0.2/2.2/2.4ms，20mm³/ms）。
2. **注2公式量纲不自洽**：直接代入数值会得到物理上不可能的速度(~11880 m/s)→ 手工用SI单位重新推导，发现需要在根号内乘1000，即 Q=CA√(2000ΔP/ρ)。
3. **问题1直接用全周期(100ms)高分辨率积分太慢**：发现[2.4ms,100ms]区间dP/dt恒为0 → 只积分前2.4ms，提速约40倍。
4. **问题2准静态ω估算(865rad/s)代入动态仿真后严重失真（压力单调飙升到1656MPa）**：逐步诊断打印theta/Vc/Pc/Ppipe轨迹 → 发现`omega(rad/s)*t(ms)`量纲错误 → 引入`omega_ms=omega/1000`修正，重新扫描得到单调合理的drift-omega关系，再用二分法求出ω*≈726 rad/s。
5. **双喷嘴二分法初始区间未包围零点**（第一次lo/hi的drift同号，二分法失效退化）→ 扩大搜索区间到[1.2×,2.6×]单喷嘴ω，重新验证lo/hi异号后再二分，最终g(ω_dual)≈-0.297，基本收敛。

## 6. 自我批判：这份解答的局限

1. **问题1的阀门开启时刻固定在周期起点，未做相位优化**：如果允许阀门开启时刻/次数（比如分多脉冲补油）作为自由变量，理论上能把周期内的波动幅度（目前约2.4MPa@100MPa工况）进一步压低。这是一维简化，完整解应做二维（时长+相位，甚至多脉冲）联合优化。
2. **"渐近稳定但收敛很慢"的开环控制在工程上不够用**：真实高压共轨系统都带压力传感器闭环反馈（PID或更复杂的控制律），本模型只给出了开环恒定τ的解，没有涉及反馈控制律设计，只是定性指出了这一局限。
3. **问题2的柱塞腔→油管单向阀假设为"无节流理想通道"**：题目未给这个阀的孔径，这是数据缺失下唯一可行的简化，但也是最大的不确定性来源——如果这个阀实际上有显著节流，真实ω会比本文算出的726 rad/s更高（需要更快转速来补偿节流损失）。
4. **问题2/3的耦合仿真用固定步长显式欧拉/RK4，未做步长收敛性验证**：虽然做了dt=0.002/0.005ms的实践性选择（基于运行时间和观察到的数值稳定性），但没有系统性地把dt减半重跑、比较结果收敛性，属于数值方法上不够严谨的地方，如果重新做会补一个步长收敛性检验。
5. **问题3的双喷嘴错相方案未重新标定角速度**，导致对比的两个方案不完全在"公平"的供油总量下比较（详见4.6节讨论），是本文最明显的未完成工作。
6. **问题3的"减压阀+双喷嘴"控制方案停留在框架/定性层面**（结合bang-bang/滞环控制思想），没有做真正的闭环控制律仿真和参数整定，是受限于本次任务的时间预算，如果重来会用更多时间做这部分的定量仿真。
7. **附件3弹性模量数据只覆盖0~200MPa**，代码里对超出范围的压力做了"边界值平铺"外推（`left=E3[0], right=E3[-1]`），问题2早期debug阶段压力曾冲到1656MPa（远超表格范围），那一版数字肯定不可信——这也是判断"一定是bug"而非"新的物理现象"的重要线索之一。
