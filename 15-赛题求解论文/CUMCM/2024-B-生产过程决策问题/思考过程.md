## 一、题目理解：这道题到底在问什么

先把题目拆成"信息流"看一遍：

- 背景：企业买零配件1、零配件2 → 装配成成品。零配件坏 ⇒ 成品必坏；零配件都好 ⇒ 成品仍有自己的
  "装配工艺次品率"（附录已明确：半成品/成品次品率是**在零配件/半成品均合格前提下**的次品率，
  这是一条非常关键、能直接消解建模歧义的题目原文信息，必须用好）。
- 问题1：纯粹的统计推断问题——不知道真实次品率，只能抽样，要设计"检测次数尽量少"的抽样方案，
  并且要在两个不同的置信水平/结论方向上给出具体判据。这是一个**验收抽样方案设计**问题。
- 问题2：次品率已知（当作确定性参数），要在"零配件测不测、成品测不测、次品拆不拆、退货拆不拆"
  四个二元决策上做优化——这是一个**决策/运筹优化**问题，核心是把各种决策组合的期望成本/收益
  用公式表达出来，再枚举比较。
- 问题3：把问题2的"一级装配"推广成"多级装配树"（图1给出了2道工序、8个零配件、3个半成品的
  具体拓扑），本质是**同一套决策逻辑在树结构上的递归**。
- 问题4：把问题2、3里"已知的次品率"改成"问题1方法抽样估计出来的次品率"，考察**统计不确定性
  如何传导进下游决策**——这是把问题1和问题2/3缝合起来的问题，检验建模的系统性。

四问的关系其实是一条线：统计推断（1）→ 单级决策优化（2）→ 多级决策优化（3）→ 把不确定性接回决策（4）。
这提示我不能把四问当作互相独立的四道小题分别"套公式"，而要建一套**可复用、可递归的统一框架**。

## 二、问题1：候选方案比较

### 候选A：单点边界法（"只看最极端的观测结果"）
思路：定义拒收阈值使得"全部抽样都不合格"才能95%信度拒收；定义接收阈值使得"全部抽样都合格"才能
90%信度接收。这样阈值和样本量的关系是纯代数方程，样本量可以做到很小。

实际算了一下（详见下面代码）：
- "只有 x=n 全部不合格才拒收，95%信度" ⇒ 最小 n=2（因为 0.1²=0.01≤0.05）。
- "只有 x=0 全部合格才接收，90%信度" ⇒ 最小 n=22（因为 0.9²²≈0.0985≤0.10）。

看起来 n 很小，很有诱惑力，但我立刻发现这个方案**检出力（功效）极差**：n=2、只有两件都不合格才拒收
的方案，即使真实次品率高达20%，拒收概率也只有4%；哪性次品率到了50%，拒收概率也只有25%。
也就是说，这个方案几乎"永远不拒收"，对企业毫无保护意义——用"最小检测次数"这个单一指标去优化，
掉进了一个退化解。这是我在实际写代码验证后才意识到的问题（而不是一开始就想到的），因此把候选A
降级为"反面案例"，在论文里用来说明"为什么不能只看n最小"。

### 候选B（采用）：两点法（AQL–RQL）验收抽样设计
这是国标 GB/T 2828 系列验收抽样标准背后的经典思想：不能只固定一个次品率去做检验，要同时设定
"可接受质量水平"（AQL，这里就是标称值 p0=10%）和一个"不可接受质量水平"（RQL/LTPD），然后要求：
- 当真实次品率恰好是 AQL 时，方案错误拒收的概率 ≤ α（对应问题(1)的95%信度）；
- 当真实次品率达到 RQL 时，方案错误接收的概率 ≤ β（对应问题(2)的90%信度）。

题目只给了一个标称值10%，没有给 RQL。我的处理是：**把 RQL 设为 2×AQL=20%**，理由不是拍脑袋，
而是题目表1本身把"10%"和"20%"作为一对最常见的对照场景反复使用（情况1↔2，情况3↔4的次品率
分别是10%和20%），说明命题人心目中"次品率翻倍"就是一个有实际意义的"次品率明显变坏"的参照点。
我把这一步作为**显式建模假设**写进论文，并在敏感性分析里检验 RQL 取 1.5×/2×/2.5×/3×/4× AQL
时最小样本量 n 如何变化，让读者看到这个假设对结论的影响有多大。

## 三、问题2、3：决策模型的两次尝试（含一次真实的建模纠错）

### 3.1 第一版思路：显式的"更新-回收"递归模型

我最初想严格地把"拆解回收的零件重新走检测/装配流程"字面地建成一个**更新过程（renewal process）**：
定义 `V_recovered`（手上已有零件对，继续往下走的期望利润）和 `V_fresh`（从头买零件开始的期望利润），
两者满足线性方程组，解出闭式解。

写完代码后，我**没有直接相信解析结果**，而是先用蒙特卡洛模拟同一个过程去对拍——这是本题计算量
不算大、但决策逻辑容易出错的地方，必须做数值验证，不能只信代数推导。

```python
# p2_montecarlo_check.py 核心片段（真实跑过）
for _ in range(n_orders):
    stack = [True]
    while stack:
        fresh = stack.pop()
        if fresh: order_profit -= K
        order_profit -= A
        good = random.random() < Pgood
        if good:
            order_profit += S
        else:
            if t3 == 1:
                if t4 == 1:
                    order_profit -= cr; stack.append(False)
            else:
                order_profit += S      # 卖给顾客
                order_profit -= L      # 调换损失
                stack.append(True)     # 补发一件全新品
                if t4 == 1:
                    order_profit -= cr; stack.append(False)
```

第一次对拍时，解析解和MC吻合得很好（情况2甚至连"利润高达1390元/件"这种反直觉的大数字都被MC
复现了），我一度以为模型是对的，"情况2下不检测成品+拆解不合格品，利润高达1390元"这个结论虽然
反直觉但数学上自洽。

**但我没有就此打住**，而是追问了一句"这个数字为什么这么大，经济上讲得通吗？"——继续深挖后发现：
`t3=0`（成品不检测）时，"未检测流入市场的不合格品，被顾客发现后无条件调换"这一支里，我把"补发给
同一个顾客的换货件"当成了"卖给全新顾客的一次全新交易"，即补发时又计入了一次 `+S` 市场收入。
但现实中，换货是**免费补发**，顾客已经为原来那件付过款了，补发件不应该再收一次钱。这是一个
**收入重复计入**的建模错误。

发现的过程：情况2给出1390元/件这种量级明显不合常理（对比市场售价才56元/件），我意识到需要
换一个角度重新核对模型的经济含义，而不是仅仅满足于"MC和解析解互相吻合"——**两者吻合只能说明
代码实现了我脑子里想的公式，不能说明我脑子里想的公式本身在经济学意义上是对的**。这是这次
建模过程中最重要的一次自我纠错。

### 3.2 第二版思路（采用）：退货余值法（salvage-credit），去递归化

修正方案：
- 换货补发**不产生新收入**，只有换货损失 `L` 与"重新生产一件合格替换品"的期望成本
  `(K+A)/P_good`（这是标准的"重试直至成功"型 renewal-reward 公式，和零配件检测中
  `(c+d)/(1-p)` 是同一个数学结构——买/测/不合格丢弃/再买，直到拿到合格品为止）。
- 拆解回收不再显式模拟"回收件进入哪一件具体产品"的无穷级数，而是把回收价值近似记为
  被拆解零件的采购成本 `K`（一阶近似：拆解省下了下次生产时的采购支出），拆解费 `cr` 作为
  额外支出扣除。这样彻底避免了递归/临界发散问题，且不再重复计收入。

重新对拍（`python3 p2_v2_mc_check.py`，真实输出）：

## 四、关键假设的论证

1. **零配件/半成品被拆解后，等价于"重新走一遍同样的检测/装配流程"，其内部零件的次品率分布
   与原始批次相同**——这是题目原文"重复步骤(1)和步骤(2)"的字面要求，不是我加的假设，只是我把
   它翻译成了"用同一个 p_i 重新参数化"，没有额外引入。
2. **拆解回收的价值近似等于被拆解件的采购成本 K**（退货余值法）——这是一阶近似，最精确的做法应该
   是显式追踪"回收件流向了哪一件具体产品"，但那样会引入无穷级数/临界发散问题（见3.2节的教训），
   在企业成本会计里，"废品回收抵扣未来采购成本"本身就是常见处理方式，我认为这个近似在题目给定的
   参数量级下是合理的，但**当次品率很高、拆解费很低时，这个近似可能低估真实的循环收益**（因为
   真实情况下回收件还能再次被检测筛选，节省的不只是采购成本），这一点在"模型评价"部分会展开讨论。
3. **换货补发的替换品通过"重试直至合格"获得，不重新计入市场收入**——这是2024年的一处真实建模纠错
   （见3.2节），我认为这是对题目"无条件调换"最经济学自洽的理解：调换是履行保修义务，不是新的销售。
4. **零配件检测/成品检测的检测结果是完全准确的（无第一类、第二类检测误差）**——题目没有给出
   检测误差率，也没有要求讨论，这是常见的简化，但在模型评价里我会明确指出：如果检测本身有
   误判概率，整个决策框架需要引入"检测灵敏度/特异度"两个新参数，属于可拓展方向。
5. **抽样服从二项分布（相当于假设批量远大于样本量，可用有放回近似代替实际的不放回抽样）**——
   题目没有给出批量大小，这是标准处理，我在论文里会提示：若批量较小，应改用超几何分布，
   结论（尤其是最小样本量）会有所不同。
6. **问题2/3中，两点法的 RQL=2×AQL 是我的建模选择，非官方给定**——已在第二节说明并做了敏感性分析。

def find_plan(p0, p1, alpha, beta, n_max=500):
    for n in range(1, n_max+1):
        for c in range(0, n+1):
            if binom.cdf(c,n,p0) >= 1-alpha and binom.cdf(c,n,p1) <= beta:

## 六、结果合理性检验

1. **量级检验**：所有情况下的最优期望利润都在个位数到20多元/件的量级，与市场售价56元、200元处于
   同一数量级，没有出现"利润远超售价"的异常（这正是3.2节中被我识别并修正掉的那类错误）。
2. **单调性检验（敏感性分析部分）**：调换损失L越高，越倾向于检测成品(t3从0变为1)；拆解费cr越高，
   越倾向于直接丢弃(t4从1变为0)；零配件次品率p1越高，越倾向于检测该零配件(t1从0变为1)；市场售价S

## 八、自我批判：这版解答的局限

1. **退货余值法（salvage-credit）是一阶近似，不是exact的多期库存/再制造模型**。更精确的做法应该
   显式建立一个"在制品池"的马尔可夫链/排队论模型，跟踪回收件的真实流向和二次筛选后的质量分布，
   但那样会大幅增加复杂度，且在企业实际决策语境下，一阶近似已经能刻画"检测-拆解-调换"三个决策
   杠杆的主要方向性效应，我认为在训练示范的范围内这个取舍是合理的，但如果重来，我会用更细致的
   离散事件仿真（SimPy一类工具）去交叉验证一阶近似在极端参数下的偏差量级。
2. **问题1的RQL=2×AQL是我的建模选择，不是题目给定**，虽然做了敏感性分析，但这仍然是一个需要
   读者/评委认可的假设，如果重来，我会同时给出"不依赖RQL假设"的简单替代方案（如序贯概率比检验
   SPRT，可以不预设RQL、按贝叶斯/似然比动态决定何时停止抽样），作为并列方案而不是候选后被舍弃。
3. **忽略了检测误差（假阳性/假阴性）**——真实质检设备/流程不可能100%准确，这是本模型最大的简化，
   会系统性地高估"检测"这个决策选项的价值。
4. **问题3的56536种组合是穷举出来的，没有做剪枝/动态规划优化**——对这道题的规模（16个二元变量）
   穷举完全可行（0.3秒跑完），但如果 m、n 规模显著变大（比如10级工序、上百个零配件），穷举法会
   指数爆炸，需要改成真正的树形动态规划（自底向上、对每个子树独立求最优，而不是全局联合枚举），
   这是本模型在"任意m道工序"意义下并未完全兑现的一个理论承诺——我在问题3代码里用的是"暴力枚举
   一个具体3层小树"，没有实现一个真正通用的、适用于任意深度/任意拓扑的递归DP代码。
5. **问题4只做了"示例性"的抽样估计（次品数=标称值×样本量的四舍五入），不是真实抽样过程的随机
   模拟**——更严谨的做法应该是对每个情景做多次随机抽样模拟，给出决策"翻转概率"的分布，而不是
   只算一个确定性的置信上限点。这是时间预算内的取舍，我在论文里已经明确标注这些数值是说明性的。
6. **没有引入折现/资金时间价值**——所有的"重试直至成功"型期望成本公式，都隐含假设资金没有时间
   成本、生产周期长短不影响决策，这在实际供应链管理里通常是需要考虑的因素。
