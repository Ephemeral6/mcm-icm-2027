## 1. 题目理解：这道题到底在问什么

拆解后，题目本质是一个**带资源约束、多方式补给、时间窗口的路径规划 + 采购/挖矿决策问题**，四个
子问题构成一条难度递增的链条：

- **Q1（单人 · 全知）**：天气全程已知，一次性规划出发点购买量 + 每日行动（走/停/挖矿）+ 途中在
  村庄的补给量，使得到达终点时"现金+剩余物资退款"最大。这是一个**确定性最优化**问题。
- **Q2（单人 · 逐日已知）**：只知道当天天气，此后天气不确定。要给出一条**策略（状态到动作的映射）**
  而不是一条具体路径，需要对未来天气的不确定性做出应对。这是一个**随机最优化/序贯决策**问题。
- **Q3(1)（n人 · 全知 · 一次性承诺）**：n人有相同初始资金，一起出发，第0天就要把**全程**方案定
  死，此后不能改。规则里"同一天同一行为的人数k"会改变消耗/收益/价格，构成人与人之间的**外部性
  （拥挤效应）**。这是一个**联合/协作优化**问题（不是零和博弈——后面会论证为什么不用纳什均衡）。
- **Q3(2)（n人 · 逐日已知 · 每轮公开信息后重新决策）**：天气逐日揭晓，且每天行动结束后大家能互相
  看到对方的行动与剩余资源，据此决定次日方案。这是一个**带完全监控的重复随机博弈/滚动重规划**问题。

四个子问题共享同一套"地图+资源+价格+挖矿"物理规则，只是**信息结构**（全知/逐日已知）和**主体
数量**（1人/n人）不同，所以我从一开始就决定：先把单人确定性模型的状态空间和转移方程搭对，后面
三问都是在这个骨架上加信息约束层或加人数维度，而不是四套互不相关的模型。

## 2. 候选建模思路比较

### 2.1 关于"总体求解范式"的比较（适用于全部四问）

| 候选方案 | 优点 | 缺点 | 是否采用 |
|---|---|---|---|
| **(a) 整数规划/MILP**（把路径、购买量、挖矿标记都设为决策变量，交给求解器） | 建模直观，约束表达灵活，容易加新规则 | 需要商业/开源 MILP 求解器；条件逻辑（"到达矿山当天不能挖矿""沙暴日强制停留"）要写大量 big-M 约束，多人问题变量数随人数、天数、地图规模组合爆炸 | 单人问题作为**备选验证**手段之一提及，不作为主方法 |
| **(b) 启发式/贪心**（每天走最短路，缺资源就买，能挖矿就挖） | 实现简单、速度快 | 无法证明最优，遇到"要不要绕路去挖矿""要不要现在买够用还是留着以后在村庄买"这类真正的权衡问题时容易系统性次优 | 不采用（这正是题目要求"给出一般情况下的最优策略"而不是"给出一种可行方案"，贪心不满足"最优"要求） |
| **(c) 动态规划/状态空间搜索**（状态=(天,位置,水,食,现金)，用状态转移方程逐层求解） | 精确、可证明最优（在给定离散化下）；天然可以从"全知"扩展到"逐日已知"的逆向归纳；状态空间大小可控 | 状态维度较高（尤其加上"现金"这一维），朴素实现会很慢 | **采用为主方法**，并配合下面 2.2 的"资金坍缩"技巧压缩状态空间 |

选择 (c) 的核心原因：这道题的"最优策略"本质上就是一个有限阶段决策过程的值函数问题，DP 是最贴合
问题结构的方法，而且能够**同时**服务于 Q1（确定性DP）和 Q2（随机DP/MDP，只是把"取最大"换成
"取期望最大"），思路一以贯之，不需要为不同子问题另起炉灶——这也是题目反复要求"给出一般情况下的
策略"（而不是只答一个具体关卡数字）所暗示的建模期望。

### 2.2 一个关键的建模简化：要不要把"现金"放进状态里？

这是我在写 Q1 代码之前认真比较过的一个关键点，因为它直接决定状态空间大小。

- **朴素做法**：状态 = (天, 位置, 水, 食, 现金)，现金也离散化。状态空间是天数×位置数×水量×
  食物量×现金档位的乘积，五维，计算量明显更大。
- **我采用的做法（状态占优/"资金坍缩"）**：注意到，对于**同一个** (天, 位置, 水, 食) 组合，手里
  现金越多**绝不会更差**——因为现金只有两个用途（购买资源、计入最终得分），多余的现金总可以选择
  不花。于是可以证明：只需要在每个 (天, 位置, 水, 食) 上记录"历史上能达到的最大现金"这一个标量，
  就足以保留所有对未来决策有意义的信息，不需要为每个现金档位单独展开状态。这把状态维度从5维降到
  4维（现金变成每个状态上的一个值而不是一个维度），是本模型中一个真正的、需要论证而不是拍脑袋的
  建模决策。

**这个"占优"论证严格吗？** 我在写 Q1（确定性）代码时用的是这个技巧的最直接版本：**正向**逐日
展开状态，每个 (位置,水,食) 只保留能达到的最大现金，因为确定性情形下"未来"就是固定的天气序列，
正向枚举天然可行。

**到了 Q2（随机情形），同样的"现金坍缩"是否还成立？** 我专门想清楚了这一点：占优论证本身
（"同一物理状态下现金越多永远不会更差"）并不依赖未来是确定的还是随机的——对**任意**一条未来天气
实现路径，现金多的分支都能重演现金少的分支的所有动作，所以占优在随机情形下依然成立。但**随机情形
下我们要的是"策略"而不是一条路径**，需要逆向归纳来处理"当前决策要预判未来所有可能性"这件事，而
逆向归纳本身要求知道"从某状态出发、按最优策略走到底的**期望**总收益"，这个量本质上是"未来净现金
流的期望"，如果构造成"从此处起（不含已经拿到手的现金）未来净流入-净流出的期望值"，就可以**不用
把现金当状态**，直接做逆向归纳（详见第4节模型二）。这是一个只有在证明"现金对未来策略的边际影响
是可分离的"之后才能放心使用的简化——我在论文里会说明它依赖"资金不构成购买行为的硬约束（不差
钱）"这一假设，并在示例参数下做了自检（起点一次性买满负重上限16箱的花费为160元，仍低于初始资金
200元）。这不是一个可以无条件套用的技巧，是需要论证成立条件的。

### 2.3 Q2 的候选方案比较：概率模型 or 鲁棒优化？

Q2 要求"给出一般情况下玩家的最佳策略"，但只知道当天天气，要对未来天气不确定性做决策，这就必须先
回答一个前置问题：**未来天气的不确定性用什么数学对象刻画？** 官方附件（关卡天气统计规律表，如果
存在的话）我们没有拿到，所以这里必须做出建模选择，而不是想当然套用某个"标准配置"：

| 候选 | 假设 | 适用场景 | 是否采用 |
|---|---|---|---|
| **(a) 一阶马尔可夫链**（明天天气只依赖今天天气，转移概率矩阵已知/可估计） | 天气有一定持续性但不需要长记忆 | 若附件给出历史天气频率统计，可以直接估计转移矩阵；若没给，需要自己假设一组合理的转移概率 | **采用为主方法**（转移矩阵本身在无附件数据时用示例数值） |
| **(b) 鲁棒/极小化极大（对最坏天气序列做优化）** | 不需要任何概率假设，只需知道天气的取值集合 | 数据严重缺失、或希望给出"无论天气怎么变都不会失败"的保守方案时更合适 | **作为候选备选方案在论文中讨论**，不作为主计算方法（会显著牺牲期望收益换取最坏情形下的安全边际） |
| **(c) 完全不建模不确定性，纯反应式规则**（例如"资源不够2天缓冲就不再往前走"） | 简单直觉规则 | 容易实现，但无法回答"一般情况下的最优策略"这个题目原话要求的问题 | 不采用为主方法（不满足"最优"要求），但其思想被吸收进(a)的策略结果里作为对比解释 |

选 (a) 而不是无脑上贝叶斯或者无脑上鲁棒优化的理由：题目原文说"玩家仅知道当天的天气状况，可据此
决定当天的行动方案"，这暗示了一种"边观察边决策"的序贯结构，天然对应 MDP/随机动态规划的框架；而
如果真实关卡数据里给出了历史天气统计规律（大概率会给，因为不给统计信息，"最佳策略"在数学上就是
不适定的——极小化极大虽然不需要概率也能定义"最优"，但通常会比期望最优保守很多），那么(a)能更充分
利用这些信息。我在论文里把(b)作为"若无法获得可靠天气统计规律时的备选"明确写出，不隐藏这个方法论
选择背后的前提假设。

### 2.4 Q3(1) 的候选方案比较：博弈论 or 联合规划？

题目给出的n人耦合规则（同行/同挖矿/同购买会改变消耗与价格）容易让人第一反应是"多智能体博弈论/
纳什均衡"。但我仔细看了规则的**数学结构**之后放弃了这个方向，理由：

- 规则(3)中三种"扎堆"效应（移动2k倍、挖矿人均收益1/k、购买4倍）**全部是负外部性**——即扎堆永远
  不会给任何一方带来额外好处，只会拉高成本或摊薄收益。也就是说，这不是一个"利益冲突、需要猜测对手
  策略"的博弈，而是一个所有人利益方向完全一致（都想避免扎堆）的**协调问题**。
- 题目条件里 n 名玩家"第0天统一确定全程方案、此后不能更改"——这是一次性、同时的集体承诺，不存在
  "看到对手行动后背叛"的机会，纳什均衡里典型的"单边偏离"分析在这里没有用武之地。
- 因此更合适的框架是**社会最优/联合规划**：把 n 人的问题当作一个**社会规划者**在同时安排 n 条路径
  （或者说是一个"多商品流+时间窗调度"问题），目标是让扎堆造成的负外部性最小化。

于是我把Q3(1)分解为：**(i) 对每人独立求解单人最优DP（第3节模型一）得到"理想路径集合"；(ii) 检测
这些理想路径是否存在"同天同边/同矿/同村"的碰撞；(iii) 若无碰撞，n人各自执行自己的最优路径即可（因
为没有任何合作收益，独立最优=联合最优）；(iv) 若地图结构导致碰撞不可避免（比如只有一座矿山、多人
都想挖），则退化为一个小规模的**错峰调度/指派问题**，用组合搜索或简单整数规划求解。** 我用一个
3节点小地图的例子（第4节模型三）具体验证了"错峰严格优于扎堆"这个论断的数量级效果。

### 2.5 Q3(2) 的候选方案比较

Q3(2) 与 Q3(1) 的核心区别是"逐日揭晓+逐日公开对手信息"，我考虑过三种处理方式：

- **(a) 各自独立、每天都只优化自己**：完全不利用"看到对手行动"这条信息，容易反复撞车（比如两人都
  觉得"今天该我去挖矿"，同时行动，双双吃亏，然后第二天可能又同时"礼让"导致矿山空置一天）。
- **(b) 滚动时域联合重规划**：每一轮结束、公开信息之后，把"从此刻状态出发的联合问题"当成一个新的
  Q3(1)（全知区间内做一次性承诺，但只承诺到下一次天气/信息更新之前的若干天，或者简化为只承诺下一
  天），重复执行。
- **(c) 完全中心化、每天都重新求解整个剩余阶段的联合MDP**：最严谨但计算量最大，在真实关卡（人数、
  地图规模更大时）可能不现实。

我在论文中把 (b) 作为推荐框架（复用 2.4 节证明的"错峰优于扎堆"结论，把它变成一条每轮都执行的
"隐性约定"：默认谁先到、谁资源更紧张谁优先使用稀缺资源点，其余人错峰），(c) 作为"计算资源充裕时的
金标准"提及，(a) 明确指出其在本题的外部性结构下是被支配的。**受时间限制，Q3(2) 只给出算法框架和
定性论证，没有像 Q1/Q2/Q3(1) 那样写代码做完整数值实验**——这是本解答一个明确的局限，第6节会再次
说明。

## 3. 关键假设及其论证

1. **天气只通过"沙暴日强制停留"影响可行动作集合，不改变资源消耗速率。**
   理由：题目原文规则(3)(4)(5)——只有规则(4)提到沙暴日必须停留，规则(5)给出的"基础消耗量"与"行走
   2倍/挖矿3倍"的定义中**没有任何一处提到消耗量随天气类型变化**。因此按规则字面意思，"高温"与
   "晴朗"在数学上是等价的（只有"是否允许移动"这一件事因天气而不同，沙暴=不允许，其余两种=允许）。
   我在两个脚本里都严格只用这一条规则，没有额外杜撰"高温多耗水"之类的机制。如果官方关卡数据附带
   了"基础消耗量随天气变化"的表格（这是可能的，只是我们拿不到），模型中只需把 `R_W,R_F` 换成
   `R_W(q_t), R_F(q_t)` 的函数即可，不影响整体框架——这一点在论文里也会写清楚，避免读者误以为"高温"
   在本模型里完全没有意义。

2. **"到达矿山当天不能挖矿"不需要额外的状态变量。**
   理由：规则(4)已经把"移动"和"停留"列为互斥的两种当日行为，而规则(7)把"挖矿"定义为"停留"的一个
   子选项。也就是说，"移动到矿山"当天的行为类型是"移动"，根本不存在"移动+挖矿"的组合动作，所以
   "到达当天不能挖矿"在移动/停留互斥的框架下是**自动满足**的，不需要专门再记一个"是否刚到达"的
   标志位。到达矿山后的**下一天**，玩家的位置已经是矿山（属于"停留在矿山"的状态），可以正常选择
   挖矿——这是我认为最贴合规则字面表述、也是最简洁的解释；我在论文里会明确指出这是一处需要解释
   的规则条款，并说明我采用的解释及理由，供后续对照真实关卡数据校验。

3. **购买顺序：起点购买发生在第0天行动之前；村庄购买发生在当天消耗结算之后。**
   理由：注2原文——"Result.xlsx中剩余资金数……指当日所需资源全部消耗完毕后的资金数……若当日还有
   购买行为，则指完成购买后的资金数"，这明确说明购买是在"当日消耗结算完毕之后"发生的。但起点购买
   是特例：玩家第0天初始水、食物均为0，若不先购买就无法满足哪怕"停留"这个最基础动作的消耗，所以
   起点购买必须发生在第0天行动**之前**（否则第0天任何动作都不可行，游戏在数学上无法开始）。

4. **游戏失败（超时未到/资源耗尽）的记分约定为一个"劣于任何真实完成路径"的有限惩罚值，而不是
   0或负无穷。**
   这不是官方规则给出的数值（规则只说"视为游戏失败"，没有给失败的量化后果），是我们为了把"到达/
   失败"这个离散判定转成一个可优化的数值目标而**必须**做出的建模选择。第5节会详细记录我是如何
   通过实际调试发现"0"和"-1e9"两个自然的候选都有问题，最终选定一个有限的负值（-500，示例参数下
   严格劣于任何可行完成方案的得分区间[0,190]）的完整过程——这一条假设的论证过程本身就是一次真实
   的建模纠错，不是凭空写的。

5. **Q2、Q3(2) 中"资金不构成硬约束"（购买只受负重上限约束，不受现金余额约束）。**
   理由：这是为了让状态空间从5维（含现金）降到4维、把逆向归纳的计算量控制在可在几秒到十几秒内跑完
   的范围内所做的**计算可行性简化**，不是普适真理。我用示例参数做了一个自检：一次性买满负重上限
   （16箱×10元=160元）仍低于初始资金（200元），说明在本例参数下这个假设基本成立；论文里会明确

6. **地图上"矿山/村庄"数量、位置、负重上限、基准价格、初始资金、天气序列均为自建的说明性示例。**

### 4.3 模型二（Q2）：随机动态规划（有限时域MDP，逆向归纳）

状态 `(t, v, w, f, q)`，`q`为当天已实现的天气。值函数 `G(t,v,w,f,q)` = 从该状态起、按最优策略

即第2.2节论证的"现金可分离"简化）。天气按一阶马尔可夫链演化，转移矩阵为示例数据：

```python
TRANS = {

#### 4.3.1 第一次调试失败：失败惩罚用 -1e9 导致期望值失真

第一版把"未在截止日期前到达终点"记为 `-1e9`（一个直觉上"足够大"的负数）。运行后：

```
若第0天天气=S: 起点最优购买(水,食)=(8, 8), 期望最终得分=-865348.39
若第0天天气=H: 起点最优购买(水,食)=(8, 8), 期望最终得分=-1706697.78
若第0天天气=D: 起点最优购买(水,食)=(8, 8), 期望最终得分=-19299458.13
```

这明显不合理——真正的失败概率在这套参数下应该很小（负重充裕、时间窗有10天而最短路径只需3天）。
问题根源：**期望值对失败惩罚的量级是线性放大的**，哪怕失败概率只有万分之几，乘上 `-1e9` 依然会
把期望值拖到几十万甚至上千万的量级，完全掩盖了正常情形下的真实收益差异，这是把"最坏情形下的
不可行"（适合鲁棒优化里的"大M约束"写法）错误地搬进了"期望值目标"里——两种目标函数对惩罚量级的
敏感性完全不同，必须区分对待。

#### 4.3.2 第二次调试失败：失败惩罚用0又制造了"摆烂"最优解

把惩罚改成 `0.0`（"不惩罚，只是拿不到后续收益"）后：

```
若第0天天气=S: 起点最优购买(水,食)=(0, 0), 期望最终得分=200.00
```

`(0, 0)`——最优解变成了"起点什么都不买"！原因是：本模型把"最终得分"拆成"初始资金-起点购买支出
+未来净流"，如果起点不买任何东西，当天连"停留"这个最基础动作需要的1箱水1箱食物都凑不出来，游戏
第0天就直接失败，但由于失败记0分、且没买东西所以也没花钱，"什么都不买"反而**保留了全部200元本金**，
数值上比任何认真尝试穿越（必然要花钱买至少上百元资源）都"更优"。这暴露了一个真实的目标函数设计
漏洞：**失败必须严格劣于任何认真尝试的完成路径，不能是一个中性的0分**，否则最优化会去钻这个空子，
选择"根本不参与游戏"这种在数学上合法、但完全不是题目本意的策略。

#### 4.3.3 最终修正：失败记为 -500（有限、且严格劣于任何可行成功得分）

结合两次失败的教训，最终选定：失败记 `-500`（本例中任何认真完成的路径得分都落在 `[0, 190]`
区间内，`-500` 保证了"失败"在数值上永远不划算，同时量级远小于 `1e9`，不会让期望值被极小概率的
尾部事件不成比例地支配）。这是**我们的建模约定，不是官方规则给出的数值**，论文里会明确标注。

修正后的真实输出：

```
=== 检验 1：期望意义下的最优得分（对天气分布取期望） ===
若第0天天气=S: 起点最优购买(水,食)=(6, 6), 期望最终得分=63.40
若第0天天气=H: 起点最优购买(水,食)=(7, 7), 期望最终得分=57.46
若第0天天气=D: 起点最优购买(水,食)=(8, 8), 期望最终得分=24.22
```

三行结果本身就很有说服力：第0天天气越"危险"（沙暴>高温>晴朗，因为沙暴当天强制停留、浪费一天的
移动机会），策略给出的起点购买量就越大（6→7→8箱），期望得分越低——符合直觉，且不是我手工设定的，
是逆向归纳自己算出来的。

#### 4.3.4 检验：信息价值非负（Q2 ≤ Q1）

把 Q2（仅知当天天气）算出的策略，放到 Q1 用过的那条**具体已实现**天气路径上跑一遍：

```

这条路径恰好是"好运气"的情形（Q2策略的应对没有被沙暴拖后腿），所以正好打平 Q1 的全知最优解，
不等式取等号也是合理的边界情形（在信息论意义下，"信息价值≥0"允许取等号，不代表模型有问题）。

#### 4.3.5 蒙特卡洛交叉验证（并且发现并修正了一个统计口径错误）

用同一套逆向归纳得到的策略，按转移矩阵随机抽样2万条天气路径做正向模拟，统计平均得分，与逆向归纳
算出的理论期望值 63.40 比对。第一次跑出来的结果是：

```
蒙特卡洛 20000 次抽样（初始天气=S）：成功 19820 次，失败 180 次
样本均值=67.69，与逆向归纳的理论期望值 63.40 比较
```

67.69 与 63.40 有约4点的持续偏差，且样本量已经到了2万条，不像是纯统计噪声。排查后发现是**统计口径
不一致**：打印的"样本均值"只对**成功**的路径取了平均，把180条失败路径直接丢弃没有计入，而逆向归纳
算的理论期望值是把失败（记-500分）也按概率加权在内的**全样本**期望。加上失败样本重新计算（把失败
也计为 -500 纳入平均）：

```
仅成功样本均值=67.69（不含失败惩罚，口径与理论值不同，仅供参考）
全样本均值（含失败记 FAIL_VALUE=-500）=62.58，与逆向归纳理论期望值 63.40 比较，相对误差 1.30%
```

口径对齐之后，蒙特卡洛全样本均值62.58与理论值63.40的相对误差只有1.30%，在2万次抽样、且存在
0.9%左右小概率大幅度惩罚（-500）事件的情况下，这个误差完全落在合理的抽样噪声范围内，**逆向归纳
的实现得到了独立验证**。

这个"先被误导、再排查口径、最后对齐验证"的过程，我认为比"一次性给出正确数字"更值得如实记录——它
真实反映了随机模拟验证里最容易踩的一个坑：**平均值的分母到底该不该包含失败样本**，这个问题在很多
仿真类建模论文里其实经常被悄悄做错。

### 4.5 模型三（Q3）：多人拥挤外部性引理 + 小规模联合调度演示

#### 4.5.1 三条引理的直接数值验证（用题目给出的真实倍数公式）

```
[引理1] 同一天同一条边上共同行走的消耗倍数 = 2k； 单独行走 = 2
  k=1: 消耗倍数=2  <- 单独行走（基准）
  k=2: 消耗倍数=4  是单独行走的 2.0 倍
  k=3: 消耗倍数=6  是单独行走的 3.0 倍
  ...
  结论：k>=2 时 2k > 2，移动应尽量错峰。

[引理2] 同一天同一矿山共同挖矿：消耗恒为3倍（与k无关），人均收益 = 基础收益/k
  k=1: 消耗倍数=3, 人均收益=80.0  <- 单独挖矿（基准）
  k=2: 消耗倍数=3, 人均收益=40.0
  k=3: 消耗倍数=3, 人均收益=26.7
  ...
  结论：消耗不变但收益被稀释，k>=2 时人均净收益严格低于单独挖矿。

[引理3] 同一天同一村庄共同购买：单独=2倍基准价；k>=2 时统一为4倍基准价
  结论：只要有>=1名同伴同日同村购买，价格立即翻倍，应尽量错峰采购。
```

这三条引理**直接来自题目给出的真实公式**（2k倍、3倍不变+1/k收益、4倍跳变），不是我编的博弈论
套话——这也是我在2.4节里放弃"纳什均衡"框架、改用"联合规划+避免碰撞"框架的数学依据：三条效应
全部是负外部性，没有任何"扎堆能占到便宜"的情形，所以协调（而不是博弈对抗）才是正确的建模方向。

#### 4.5.2 3节点小地图上的联合调度演示（S-矿山M-终点E）

用一组独立于Q1/Q2示例参数的新示例数值（基础收益调高到80，使"单人挖矿"净收益为正、便于展示"扎堆
把净收益打没"的效果），比较"两人扎堆"与"两人错峰1天"两种联合方案：

```

  两人合计最终资金 = 260.0

结论：错峰方案比扎堆方案，两人合计多保留资金 220.0 元
```
