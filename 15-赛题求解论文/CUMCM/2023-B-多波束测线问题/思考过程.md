## 2. 候选建模方案比较

### 2.1 问题1：覆盖宽度与重叠率模型

**方案A（选用）：截面内几何光线追踪 + 有符号局部坡度的解析解**
把与测线垂直的截面看成一个二维平面（水平轴 x，垂直深度轴 z），海底是一条过(x0, D(x0))、斜率为 tanα 的直线。换能器在水面 (x0,0) 处以半张角 θ/2 向两侧发射光线，分别求光线与斜线的交点，得到覆盖宽度的两条边界 x_left(x0)、x_right(x0)，从而 W = x_right - x_left，且不要求左右对称。

**方案B（弃用）：忽略坡度对称近似**
直接用平地公式 W = 2·D·tan(θ/2)，D 取该测线处的"局部水深"，坡度只用来算 D(x)。这是很多入门解法会犯的简化——它假定覆盖宽度左右对称，但实际上因为坡度存在，斜坡下坡方向的波束要走更长的光程才能到达海底，覆盖宽度天然是不对称的（上坡边界近、下坡边界远）。若只用对称近似，重叠率的计算会系统性偏离真实值，尤其在 α 相对 θ 不算太小、或者用于问题3/4的大跨度设计时误差会累积放大。

**决策**：选方案A，因为题目原文明确要求"建立...覆盖宽度及相邻条带之间重叠率的数学模型"，说明官方期待的是能反映坡度导致的不对称性的精确几何模型，而不是平地近似再打补丁。方案A的解析式还能在 α→0 时自然退化为方案B（平地公式），这是一个很好的自检手段（见4.1节）。

**推导要点**（详见论文正文）：设 x 方向为深度增大的方向，局部有符号坡率 k=tanα，测线在 x0，局部水深 D(x0)=D0+k·x0。光线方向角 φ∈[-θ/2, θ/2]（以垂线为0），光线 X(φ)=x0+t·sinφ, z(φ)=t·cosφ，与海底面 z=D0+k(X-x_ref) 联立解出：

X_hit(φ) = x0 + D(x0)·sinφ / (cosφ - k·sinφ)

取 φ=±θ/2 得到两条边界，W = X_hit(θ/2) - X_hit(-θ/2)（取绝对值排序）。

这个"通用 k 值直接代入"的写法，比教材上常见的 "cos(θ/2±α)" 形式更不容易在编程时出符号错误——因为 k 可正可负、可以直接对应"哪一侧更深"，不用再额外判断 sign(α)。用它我在写代码时几乎没有再犯符号错误（唯一犯错的地方在问题3做敏感性分析时，见第5节）。

### 2.2 问题2：测线方向偏离坡向的模型

**方案A（选用）：三维坐标分解，把β折算成"等效横向坡度"再复用问题1模型**
设水平面内 X轴=坡面法向水平投影方向（即最大坡降方向），Y轴=走向（等深线方向），海底 D(X,Y)=D0+X·tanα（与Y无关）。测量船沿与X轴夹角β的测线、距中心ρ处：
- 该点局部水深：D_local = D0 + ρcosβ·tanα
- 垂直于测线的水平方向为 (-sinβ, cosβ)，沿该方向移动距离 s，水深变化 ΔD=-s·sinβ·tanα，即截面内的等效有符号坡率 k_eff = -sinβ·tanα
- 直接把 (D_local, k_eff) 代入问题1的 W 公式即可

**方案B（弃用）：重新单独推导三维空间中光线与倾斜平面的交点**
即不做"投影降维"，直接在三维中对任意方向的测线重新建立光线-平面求交方程。这条路数学上完全可行，但推导量大很多、极易出符号错误，而且本质上和方案A的结果应该完全一致（因为问题2的物理情形就是"任意方向截面下的问题1"）。

**决策**：选方案A，因为它可以直接复用问题1已经验证过的公式和代码，只需把有效局部坡率 k_eff 换成方向相关的表达式。而且方案A自带一个内部一致性检验：当 β=90°或270°（测线垂直于坡向）时，k_eff = ∓tanα，应该退化到问题1在对应水深下的结果——代码算出来确实如此（见4.2节），这是对推导正确性的强力验证。

### 2.3 问题3：变间距测线设计

**方案A（选用）：从最深/最浅一侧边界出发，用"目标重叠率10%"逐条推进（marching）的贪心算法**
先把第一条测线放在能让其覆盖宽度左边界恰好贴合海域边界的位置（不做浪费），然后逐条向另一侧推进：给定上一条测线的覆盖右边界，用方程"重叠长度 = 10%×当前测线覆盖宽度"解出下一条测线的位置（该方程是关于下一条线位置的隐式方程，因为覆盖宽度本身依赖测线所在的局部水深，用 `scipy.optimize.brentq` 数值求根）。重复直到覆盖到另一侧边界为止。

**方案B（弃用）：等间距设计，间距按最浅处（或平均）水深统一取值**
这正是题目背景材料图5、图6批评的两种做法——用平均水深会在浅处漏测，用最浅水深会在深处冗余。既然题目已经在引言里明确指出了这两种方法的缺陷，出这道题显然是希望我们给出"变间距"的解，如果用等间距方案，等于没有真正建模坡度的影响。

**方案C（弃用）：把10%~20%重叠率区间的中点15%作为固定目标**
这样设计更保守（每两条线间距更小），能提供更大的安全余量，但由于问题3所依赖的地形是**严格线性的理想斜面**（没有噪声），用10%下界完全可以精确满足约束（见后面验证：34条线，重叠率精确等于10.00%，没有一条超出[10%,20%]区间），没有必要牺牲总长度去换取不需要的安全余量。故对问题3选择10%（区间下界，最大化间距从而最小化测线条数与总长度），但在问题4（真实地形，有噪声）中，由于1D近似无法保证处处精确，改用更保守的目标值并做灵敏度分析（见2.4节）。

**决策**：选方案A + 10%目标，因为它同时满足"总长度最短"（用允许的最大间距）和"完全覆盖+重叠率10%~20%"（用问题1模型精确保证每一步的重叠率恰好等于下界，不会超出上界，因为地形是理想线性平面，重叠率不会因为局部涨落而失控）。

### 2.4 问题4：真实地形上的测线设计

这是最开放的一问。可选方案：

**方案A：把问题3的方法整体照搬，只是把"理想斜面"换成"数据的列（东西向）均值剖面"**
先用列均值 D̄(x)=mean_y Z(y,x) 和其数值梯度 dD̄/dx 当作"局部坡度"，跑跟问题3一样的marching算法确定测线的东西向位置；测线取南北满长（覆盖整个5海里）。得到设计后，再回到**真实二维网格**（不是均值曲线）逐行验证实际的覆盖/重叠情况，如实统计漏测比例、超20%重叠长度。

**方案B：全二维整数规划/贪心覆盖优化**
把海域离散成网格，把候选测线集合和覆盖关系建成集合覆盖问题，用整数规划或贪心算法直接求"最小测线条数下满足覆盖率与重叠率约束"的最优解，允许测线间距任意、甚至允许非平行测线。这是理论上更优的方案，但求解规模大（201×251网格），且需要专门的地形自适应最优化建模能力，超出本次训练示例的时间预算，实践中海洋测量软件也常用启发式/分区方法而非全局整数规划。

**方案C：先做全局线性趋势面拟合（最小二乘平面），当成理想斜面直接套用问题3公式**
更简单，但拟合出来的平面 R²只有0.56（见4.4节实测），说明地形有相当大的局部起伏无法用一个全局平面刻画，用它设计出来的测线在局部起伏区域会有明显失配。

**决策**：选方案A。相比方案C，用列均值剖面（保留了东西向的整体趋势，同时不强行假设线性），能更好地跟随地形的实际变化（比如问题里数据其实并非单调西深东浅，而是在x=0.4~0.6NM附近有一个浅坑，见4.4节）；相比方案B，方案A的计算量和实现复杂度可控，且给出的是"可以现场执行、便于讲解"的经典平行测线方案，符合真实海洋调查的作业习惯（联系条方向固定的测量船实际作业方式）。

同时诚实说明：这是一种**降维近似**——用"东西向1D趋势 + 南北向按行代入实际局部值验证"的方法，本质上假设"每条测线的横向（东西）位置由整体趋势决定，但每一行的实际覆盖用该行的真实数据算"。这不是全局最优解，是一个可解释、可复现、计算量可控的工程近似解，其性能指标（漏测率、超限重叠长度）在论文里如实报告，并做了目标重叠率的灵敏度扫描以说明这是一个可调的权衡曲面而非唯一答案。

## 3. 关键假设的论证

1. **海底局部可视为平面/线性坡（问题1-3）**：题目本身就是这样设定的（"坡度为α的斜线"），这不是我加的假设，是题目给定的理想化条件，直接采用。
2. **换能器波束在垂直于航迹的截面内呈以正下方为中心、半张角θ/2对称展开的扇形**：这是多波束声呐的标准工作原理（题目图2描述），是行业公认的建模基础。
3. **覆盖宽度取"水平投影宽度"而非"沿斜坡表面的宽度"**：因为题目定义重叠率 η=1-d/W 中的 d 是"相邻两条测线的间距"，而测线本身是画在水平面（海图）上的，因此 W 必须是水平投影量纲才能跟 d 相除，这是从重叠率定义反推出的必然要求，不是随意选择。
4. **重叠率的一般定义**：η_i = (R_{i-1}-L_i)/W_i，其中 [L_i,R_i] 是第i条测线覆盖宽度的水平区间。我验证了此式在坡度α→0时精确退化为题目给出的 η=1-d/W（见4.1节自检），说明这个推广定义与题目原始定义完全兼容，是题目定义在非平坦地形下唯一自然的推广。
5. **问题2中 X 轴（法向水平投影方向）与坡向重合**：坡面法向的水平投影必然指向坡度下降最快的方向，这是解析几何的直接结论（斜面法向量在水平面的投影就是梯度方向），不需要额外假设。
6. **问题3测线方向选南北向（垂直于坡向东西向）**：因为若测线走向偏离等深线方向，则单条测线内部水深也会连续变化，覆盖宽度会沿线变化，需要处理二维协变问题；若测线严格顺着等深线（走向方向），则一条测线内水深近似恒定，问题退化为纯粹"如何在东西方向排布测线间距"的一维问题，模型最简洁、也最符合工程直觉（这也是问题背景材料图4-6一直用的框架）。
7. **问题4选东西向作为主变化方向、测线取南北走向**：不是假设，是从数据直接验证的（见4.4节：按列（东西）方向的深度标准差均值16.6m，小于按行（南北）方向的标准差均值24.2m，说明水深沿东西方向变化更剧烈、更适合作为"跨线"方向）。

### 4.1 问题1：覆盖宽度与重叠率

```python
theta = np.deg2rad(120.0); theta2 = theta/2
alpha = np.deg2rad(1.5); D0 = 70.0
k = np.tan(alpha)

def swath_edges(x0, D0_center, k, x_ref=0.0):
    D_local = D0_center + k*(x0 - x_ref)
    def X_hit(phi):
        return x0 + D_local*np.sin(phi)/(np.cos(phi) - k*np.sin(phi))
    x_a, x_b = X_hit(theta2), X_hit(-theta2)
    x_left, x_right = min(x_a,x_b), max(x_a,x_b)
    return D_local, x_left, x_right, x_right-x_left
```

### 4.2 问题2：任意方向偏角β下的覆盖宽度

## 5. 遇到的问题与调整

1. **第一次尝试**：直接用列均值剖面+15%目标重叠率设计问题4的测线，验证发现36%的（行,相邻线对）出现漏测（负重叠）、33%超过20%重叠，比预期差很多。

2. **诊断**：这是因为真实地形在南北方向（沿单条测线走向）上也有不小的起伏（前面测得的"按X列的标准差均值16.6m"），而设计阶段只用了列均值这个"一维简化"，导致某些行的真实局部水深明显偏离列均值，天然会在部分行产生漏测、部分行产生超额重叠——这是1D近似方法固有的局限，不是代码bug。

3. **尝试修正方案①**：把设计剖面从"列均值"换成"列的15百分位数（偏保守/偏浅）"，配合更紧的10%目标间距，重新设计。结果：条数从46增加到65，总长度从230海里增加到325海里，漏测率从6.1%大幅降到0.075%（几乎消除），但超20%重叠的长度反而从75.3海里恶化到190.8海里（59%的行对超限）。这证明了"消除漏测"和"控制超额重叠"是相互竞争的目标，不存在能同时把两者都压到很低的免费午餐。

4. **调整**：没有继续沿着"单一方案"死磕，而是做了一次目标重叠率的**灵敏度扫描**（10%/12%/15%/18%/20%，仍用列均值剖面），把"总长度 vs 漏测率 vs 超限长度"的权衡关系用数据摆出来（结果见论文表格），最终选择15%（区间中点）作为汇报的主设计方案，因为它在三个指标之间处于相对均衡的位置，同时诚实报告了这条权衡曲线，而不是挑一个看起来数字最好看的极端方案掩盖trade-off的存在。

5. **问题3做灵敏度分析时踩的坑**：测试 α=1.7°、2.0°时，`brentq`求根直接报错 `f(a) and f(b) must have different signs`。一开始以为是数值求解的bug，排查后发现是**物理上不合理的参数组合**——当 α=1.7°时，东边界处的理论水深 `110 - 3704×tan(1.7°) ≈ 0.068m`，几乎为零；α更大时东边界水深会变成负数，也就是这一斜面模型在东边界"露出水面"了，这已经超出了模型的物理适用范围（不是船只能不能测的问题，而是海底比海平面还高，物理上不成立）。于是我算出了这道题参数组合下的临界坡度 `α_crit = arctan(D0/halfwidth) ≈ 1.701°`，把它作为问题3模型适用范围的一个边界条件写进了论文的"模型评价"部分，而不是简单跳过这个报错。

## 6. 自我批判：本解答的局限

1. **问题4的"1D趋势+逐行验证"方法本质是降维近似，不是全局最优解**：真实的最优测线设计应当是一个二维（甚至允许非平行线、局部转向的）覆盖优化问题，理论上可以用集合覆盖/整数规划或更精细的地形自适应算法做得更好（漏测率更低、超限重叠更少、总长度更短）。这次为了在有限时间内给出可解释、可复现的方案，选择了计算量可控的一维近似，性能指标（6.1%漏测、75.3海里超限重叠）应被理解为"一种朴素但诚实的基线"，而非该题的最优解。
2. **测线方向没有做真正的多方向搜索**：我依据"东西方向标准差大于南北方向"这一个统计量就直接选定了测线走南北向，没有对其他任意角度方向做量化比较（比如测线走30°方向是否总长度更短），严格来说这是一个经过论证但未经过完整搜索验证的选择。
3. **总长度计算假设每条测线都取满整个南北5海里**：实际上边缘测线的有效覆盖需求可能不需要跑满全长（比如最东/最西几条测线如果只覆盖局部区域，可以裁短），这会进一步压缩总长度，我的方案在这一点上偏保守（总长度可能被高估）。
4. **相邻重叠率的计算只考虑了"紧邻的两条线"**：没有检查非相邻测线之间是否也产生了意外重叠或者遗漏交叉验证，在测线顺序单调排列、且没有大幅折返的前提下这是合理简化，但严格意义上不是穷尽验证。
5. **问题4的目标重叠率15%是一个经过灵敏度扫描后的主观权衡选择，不是通过显式优化目标函数得到的"最优点"**：如果要做得更严谨，应该定义一个显式的多目标损失函数（比如总长度+惩罚项×漏测率+惩罚项×超限重叠率），再去优化目标重叠率或分段变化的目标重叠率，而不是从5个候选值里挑一个"看起来均衡"的。
6. **附件数据只用于问题4，没有反过来验证问题1-3的理想化假设（比如附件数据的整体坡度、深度量级是否与问题1-3给定的教学参数吻合）**——这两组任务本来就是独立的（问题1-3用给定参数，问题4用真实数据），我没有强行关联，这是符合题意的，但如果要写一篇更完整的论文，可以补充讨论"真实数据揭示的坡度与理想化假设的差距有多大"作为呼应。
