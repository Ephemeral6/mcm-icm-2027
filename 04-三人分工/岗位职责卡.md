# 岗位职责卡（定岗后各自保存一份）

> 配套 [README](README.md) 的分工方案。原则：职责到人、能力交叉——人人都要懂建模，写作从第二天开始。

## 🧑‍💻 岗位一：AI 工程师 + 编程/可视化手（你）

**备赛期职责**
- 搭建并维护全队 AI 工作流：提示词库（读题/思路/建模/代码/润色五段式）、AI 交互台账（格式见 [02/AI-Use-Report模板](../02-AI使用政策/AI-Use-Report模板.md)）
- Python 数据栈熟练：pandas / scikit-learn / xgboost / shap / matplotlib+seaborn
- 建常用模型的模板代码库（每个模型：输入格式/输出格式/适用场景/调用示例）
- 定死全队绘图样式表（.mplstyle）

**赛中职责**
- 数据清洗、EDA、模型实现、调参、出所有统计图
- 每次 AI 交互当场记台账；赛末整理成 Report on Use of AI Tools 章节
- 对 AI 生成的代码和结论做第一道验证

**赛中红线**：AI 输出未经验证不得直接交给写作手进论文。

## 📐 岗位二：建模手 + 队长（数学最好的队友）

**备赛期职责**
- 吃透必学模型清单（见 [05/必学模型清单](../05-备赛路线/必学模型清单.md)），每个模型能讲清"什么时候用、假设是什么、怎么解释结果"
- 精读 O 奖论文的建模章节，积累"假设→模型→验证"的论证套路
- 学敏感性分析与不确定性量化的常用做法（C 题连续三年点名要求）

**赛中职责**
- 主导选题决策（流程见 [03/选题决策清单](../03-题型与选题策略/选题决策清单.md)）
- 把每一问翻译成数学问题：变量定义、假设清单（每条配 justification）、模型选择与论证
- 把关 AI 给出的建模建议——采纳/拒绝的理由要能写进论文
- 队长职责：控进度（每晚对齐会）、拍板争议、盯截止时间

**赛中红线**：模型再好，编程手说实现不了就立即降级方案——不恋战。

## ✍️ 岗位三：写作手 + 排版（英文最好的队友）

**备赛期职责**
- LaTeX + mcmthesis 模板熟练（模板已内置：[11/mcmthesis](../11-LaTeX与工具/mcmthesis/)），赛前改造出队伍论文骨架
- 精读 O 奖论文的行文与图表 caption 写法，整理摘要写作套路（见 [06/摘要写作指南](../06-获奖策略与评审/摘要写作指南.md)）
- 练"给非技术受众写 memo/信"文体（近三年 C 题每年都考）

**赛中职责**
- 第 1 天：搭论文骨架（章节、图表环境、参考文献就位）
- 第 2 天起：与建模求解**并行写作**，当天内容当天成稿
- 第 4 天：摘要精写（预留至少半天）、全文合体、图表编号与交叉引用检查
- 终校：25 页限制、盲评检查（无姓名/校名）、PDF 命名=控制号、<25MB

**赛中红线**：摘要绝不留到最后一小时；正文任何地方不出现队员姓名和学校。

## 全队共同纪律

- 每晚固定 30 分钟对齐会：今日产出、明日计划、卡壳点
- 所有中间产物进共享文档库（版本可追溯）
- 赛期内不与队外任何人讨论题目（含指导教师）——官方规则，违者 DQ
- 睡眠轮换表提前排好，保证任何时刻至少一人清醒推进
