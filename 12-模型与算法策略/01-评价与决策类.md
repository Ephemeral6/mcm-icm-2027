# 01 评价与决策类模型策略

> 本类是三轮调研中验证最充分的类别。适用信号总纲：题目在问"**排序/打分/选优/公平吗**"。
> C 题相关度：★★★★（2026 年 C 题第 2–5 问本质全是评价与机制设计）

## 0. 快速决策表

| 你的处境 | 选它 |
|---|---|
| 有专家判断/主观偏好要转成权重（无客观数据） | AHP |
| 已有各指标客观数据，想让数据自己定权重 | 熵权法 |
| 有了权重，要对多个方案排序 | TOPSIS |
| 评价标准边界模糊（"好/较好/一般"这类语言等级） | 模糊综合评价 |
| 小样本下看两组序列走势像不像/谁影响谁 | 灰色关联分析 |
| 多投入多产出比相对效率（不想人为定权） | DEA |
| 既想要数据客观性又想保留专家经验 | AHP+熵权组合赋权 |

## 1. 逐模型策略卡

### AHP 层次分析法【已验证·学术印证】
- **信号**：多层次、多准则决策（方案选择/资源分配/风险评估）；有专家打分需要转化为权重。
- **弱点**：判断矩阵受专家主观偏差影响；准则增多时两两比较数按 n(n-1)/2 爆炸；**一致性检验（CR<0.1）必须做并写进论文**。
- **易错**：拿 AHP 处理纯客观数据问题（评委会问"权重哪来的"）；不报告一致性检验。
- **搭配**：AHP（主观）+ 熵权（客观）组合赋权——学界推荐的折中。
- **库**：numpy 手写（20 行内）。

### 熵权法【已验证·含被证伪点】
- **信号**：已有指标数据矩阵，希望按指标差异程度客观定权。
- **⚠️ 关键澄清**（流传说法被 3 票否决）：熵权**不是**"完全客观、避免一切主观干扰"——它只度量数据离散度，忽视指标实际含义与决策者经验，权重可能与真实重要性相悖；且**对极端值敏感**（先处理离群点再算权）。
- **易错**：把离散度大 = 重要 当成不证自明；论文里应说明"熵权反映信息量差异"并讨论局限。
- **搭配**：熵权 + TOPSIS（标准组合，见下）；因子分析/PCA + 熵权构建两层权重体系。
- **库**：numpy 手写。

### TOPSIS 优劣解距离法【已验证】
- **信号**：多方案多指标排序；判据 = 到理想最优/最劣解的相对距离。
- **优点**：计算简单、结果直观（0–1 接近度）。
- **弱点**：**结果对权重敏感**——这正是"熵权+TOPSIS"成为标准搭配的原因；学界另指出 rank reversal（增删方案可能颠倒排序）问题。
- **O 奖实例**：2024 C 题 O 奖 2401919 用 **PCA-TOPSIS**（PCA 取 6 主成分、按特征值占比定权）——评委点名认可其规避了主观权重。
- **标准流程**：熵权定权 → 指标加权 → 正/负理想解 → 距离 → 接近度 C 排序。

### 模糊综合评价【已验证】
- **信号**：评价因素边界不清、只能用语言等级表达（"很欢迎/欢迎/一般/不欢迎"）。
- **门槛**：隶属度函数要靠专家经验构建——论文必须交代构建依据，否则就是把主观性藏进函数里。
- **弱点**：指标集过大时分辨率变差（结果都挤在中间等级）。

### 灰色关联分析【经验】
- **信号**：小样本下比较序列间关联程度/走势相似度；常当"穷人版相关分析"用于因素筛选。
- **易错**：分辨系数取值（惯例 0.5）影响结果却常不做敏感性检验。

### DEA 数据包络分析【已验证】
- **信号**：多投入多产出的**相对效率**比较（学校/医院/银行），且不想人为设定权重。
- **弱点**：对数据变化和异常值敏感；只给相对效率（全体都差时也会有"有效"单元）。

## 2. 本类的论文写法要点

1. 权重来源必须可辩护：主观（AHP+一致性检验）、客观（熵权+局限讨论）或组合，三选一并说明理由；
2. 排序结果必配**权重敏感性分析**（权重扰动 ±10–20%，排序变不变？）——TOPSIS 权重敏感是评委已知弱点，主动回应它；
3. 评价类结论天然主观，用第二种方法交叉验证排序（如熵权 TOPSIS vs 灰色关联排序一致性）是加分套路。
