# 03 优化类模型策略

> 适用信号总纲：题目在问"**怎么安排/分配/设计使某目标最优**"。
> C 题相关度：★★（C 题偶尔在最后一问要求"设计更优方案"时用到；A/B/D 题的主战场）
> 可信度说明：本类未进入第三轮的三票验证名单，内容为抓取素材【经验】+ O 奖队指南【已验证来源】，逐条标注。

## 0. 快速决策表

| 问题特征 | 选它 |
|---|---|
| 目标和约束都是线性 | 线性规划 LP（单纯形，秒解） |
| 决策变量必须取整（选/不选、数量） | 整数规划 IP / 混合整数 MIP |
| 目标或约束非线性且**凸** | 梯度类方法（收敛快）【经验规则】 |
| 非凸/不可导/组合爆炸（如 TSP） | 启发式（GA/SA/PSO）【经验规则】 |
| 多阶段序贯决策 | 动态规划 DP【经验·直接映射】 |
| 多个冲突目标 | 多目标：ε-约束 / 加权 / NSGA-II 求 Pareto 前沿 |

**建模三步**【经验】：先定决策变量 → 再定目标函数 → 最后写约束。写论文时按这个顺序陈述模型。

## 1. 逐模型策略卡

### 线性/整数/非线性规划【经验】
- LP/IP 是"资源分配/生产计划/选址"类小问的默认工具；标准形式写清楚（决策变量表、目标、约束逐条编号）本身就是得分点。
- 求解：scipy.optimize.linprog / PuLP / OR-Tools（IP 用后两者）。
- 易错：忘记非负约束；整数问题松弛成连续后不做说明。

### 动态规划【经验】
- **信号**：多阶段决策 + 最优子结构（每阶段决策影响后续状态）。
- 易错：状态定义不清导致维度爆炸；论文要写清"阶段—状态—决策—转移方程"四件套。

### 多目标优化【含被证伪点】
- ⚠️ **流传说法被 3 票否决**："标准解法=加权转单目标+遗传算法"——加权法只是多种标量化方法之一。
- 正确姿势：说明目标间冲突 → 选择策略并论证（加权需论证权重来源；ε-约束把次要目标变约束；NSGA-II 直接求 Pareto 前沿再从前沿中按决策偏好选点）。
- 展示 Pareto 前沿图是这类问题的标配图表。
- 库：pymoo（NSGA-II）。

### 启发式算法（遗传 GA / 模拟退火 SA / 粒子群 PSO）【经验】
- **共同本质**：并行、随机、有方向的搜索；**共同短板**：全局收敛性无保证 + 超参数敏感（GA 的种群/交叉/变异率；SA 的初温/终温/衰减因子/内循环次数）。
- **GA 适用信号**：问题可表达为显式目标函数 + 决策参数可编码。
- **SA 适用信号**：易陷局部最优的问题（靠概率性接受劣解逃出局部）。
- **前置分析策略**【已验证来源·O 奖队指南】：盲目对整个高维参数空间做全局优化是坏主意——先利用对称性/物理约束降维再搜索（指南作者在喷灌题中把 SA 耗时从数小时降到几分钟）。
- **论文纪律**：报告超参数取值与调参过程；跑多次取统计（随机算法单次结果无说服力）；给收敛曲线图。

### 直接搜索 vs 梯度法【已验证来源·O 奖队指南】
- 无导数/非光滑/高维 → 直接搜索（爬山、SA、Nelder-Mead、GA、PSO）；
- 可导且光滑 → 梯度类收敛快得多，能用就用。

## 2. 本类论文写法要点

1. 一定先写数学模型（变量/目标/约束），再谈算法——"用了遗传算法"不是模型，是求解器；
2. 启发式结果要和某个基线对比（随机解/贪心解/小规模精确解）；
3. TSP 等 NP 完全问题要点明组合爆炸（这是选启发式的正当理由）【经验】。
