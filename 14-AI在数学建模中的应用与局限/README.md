# 14 AI 在数学建模中的应用与局限

> 专项调研（2026-07-15，110 代理，23 条论断三票确认、2 条否决）。这是你们队伍最核心的一份文件——你们的优势是 AI 工具使用能力，这份调研回答"这个优势具体能兑现在哪、边界在哪"。
> 可信度标注：**【已验证】** = arXiv 一手论文/官方文件 + 三票确认；**【经验】** = 综合前述证据后的推导建议，未经同等强度验证。

## 0. 一句话结论

AI 在数学建模中最大的风险**不是算不出数字，而是"过程不可靠却看起来很像回事"**。顶尖模型在有标准答案的数学题（AIME）上已经能拿满分，但面对真正开放式的建模论证，仍然会跳步骤、藏假设、循环论证；面对无解或条件不全的问题，AI 几乎不会说"做不出来"，而是编个条件硬凑一个像样的答案。COMAP 和 CUMCM 官方都已经看到这个风险，各自立了强制披露规则。你们的任务不是"少用 AI"，而是"知道 AI 在哪个环节最容易出这种错，把人工验证精确卡在那几个环节"。

**2026 年 7 月更新**：已经有专门为 MCM/ICM 设计的多智能体系统（MM-Agent），并且**在 2025 年真实帮两支本科生队伍拿到了 MCM/ICM Finalist 奖**（27,456 支队伍中前 2%）——AI agent 自动化建模不再是纯理论，见 [05-Agent自动化建模的最新进展](05-Agent自动化建模的最新进展.md)。但同一时期的研究也用数字证明了"多智能体协作"不是幻觉的自动解药，架构设计（尤其是让 agent 互相对抗挑刺而非互相点头）比堆数量更重要。

## 2. 本轮调研最重要的四个发现（已验证）

1. **答案对不代表过程对**：多篇 arXiv 基准论文证明，即使最终数值答案正确率很高，推理步骤中仍常见"跳步骤、循环论证、隐藏假设、论证含糊"，某测试集里约 5% 的题目是"答案蒙对了但推理过程是错的"。**含义：审论文时必须看 AI 输出的完整推导链条，不能只看结论数字对不对。**
2. **AI 自查自纠的能力比直接解题弱得多**：GPT-4 在识别别人代码/解题中"是否有错、错在哪、错误类型是什么"的任务上平均正确率只有 76%，且下降主要发生在"判断错误类型"这一步——**含义：不能指望"让 AI 检查一下这个结果对不对"就万事大吉，AI 自己也经常判断错。**
3. **AI 不会对无解问题说"不行"**：面对条件不全或本身无解的问题，前沿模型（DeepSeek-R1、o3-mini 等）在标准提示下几乎不会主动指出问题无解，而是编造缺失条件强行给出答案；专门设计的"要求诚实"提示词也只能部分缓解。**含义：题目条件模糊或数据缺失时，AI 给出的"顺畅答案"反而更值得怀疑。**
4. **官方规则不是防君子的摆设**：COMAP 政策原文直接写"AI 会产生语言通顺但科学上不成立的幻觉内容，包括编造不存在的引用文献"，并据此把"模型选型、代码生成、结果解读、得出结论"列为官方点名的高风险环节，要求这几处必须人工把关。CUMCM 的规定思路完全一致（核心建模分析须独立完成）。**这两份官方文件的措辞，和上面三条技术证据是同一个问题的两面。**
