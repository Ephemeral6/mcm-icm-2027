# 05 Agent 自动化建模的最新进展（2025.10–2026.7）

> 专项调研（2026-07-19，107 代理）。⚠️ 本轮调研的自动汇总环节发生故障（返回了无效占位内容），本文件是从底层搜索/抓取/验证记录中直接提炼整理的，未经过完整的三票对抗验证流程——已在每条后标注来源与发布时间，可信度参考来源质量（【primary】=一手论文/官方文件，【secondary】=聚合网站，【blog】=中文社区转述）。这是对 [01-技术局限性](01-技术局限性.md) 的更新延伸：那份讲"AI 单独使用的局限"，这份讲"2025–2026 年学界怎么用多智能体系统去补这些局限"。

## 0. 一句话结论

**已经有专门为 MCM/ICM 这类开放式建模设计的多智能体系统，并且真的帮真实队伍拿过奖**——但同一时期的研究也用数字证明了"多智能体协作"本身不是幻觉的自动解药：协作机制设计不当，反而会让本来答对的意见被投票投掉。对你们这种用现成 ChatGPT/Claude、不自建 agent 系统的普通队伍，最大的启示不是"该不该自建 agent"，而是这些系统验证有效的**架构原则**——分角色、对抗式互相挑战、把验证和写作分开——你们完全可以用"人工扮演不同 agent 角色"的方式低成本复现。

## 1. 最重要的发现：MM-Agent，已有真实获奖案例

**MM-Agent**（论文《MM-Agent: LLMs as Agents for Real-world Mathematical Modeling Problems》，arXiv 2505.14148，2025-05-20 提交，被 **NeurIPS 2025** 接收）是目前查到的唯一一个**专门针对开放式数学建模（而非 AIME/IMO 式封闭数学证明）设计**的多智能体框架【primary，来源：arXiv 论文 + GitHub 仓库 usail-hkust/LLM-MM-Agent】。

**架构**：把建模拆成四个阶段——问题分析 → 结构化模型构建 → 计算求解 → 报告生成，正好对应"读题→建模→写代码→算结果→写论文"的完整链条，核心组件是一个含 **98 种建模方法**的分层建模知识库（HMML），用于指导模型选型环节。

**配套基准 MM-Bench**：作者同时发布了首个开放式建模基准，收录 **2000–2025 年 MCM/ICM 历年真题共 111 道**，覆盖物理、生物、经济等十大领域——这是目前唯一一个直接拿 MCM/ICM 真题当评测集的公开基准。

**评测结果**：MM-Agent（GPT-4o 骨干）在 MM-Bench 上综合得分**比人类专家方案平均高 11.88%**（人类团队 2021–2024 年综合分 7.91、2025 年 8.02；MM-Agent 对应 8.85、8.38），单题耗时仅 15 分钟、成本 0.88 美元；而**不带 agent 编排、直接裸调 GPT-4o/DeepSeek-R1** 综合得分只有 6.28–6.50，明显更低——这是"多智能体编排 vs 裸模型直接解题"在开放式建模场景下少见的量化对比。

**⭐ 最关键的一条：真实案例**——MM-Agent 已经**在 2025 年 MCM/ICM 比赛中实际帮助两支本科生队伍获得 Finalist 奖（27,456 支参赛队伍中前 2.0%）**。这不是论文里的模拟实验，是官方比赛协议下真实发生的事。

## 2. Sci-Mind：更新的继任框架（2026-04），核心设计原则是"对抗式辩论"

**Sci-Mind**（2026-04-02 发布）在同一 MM-Bench 基准上超过 MM-Agent，核心创新是把"验证"环节做成**两个目标互相冲突的 agent 互相辩论**——一个 Theorist agent 追求理论严谨性，一个 Pragmatist agent 强制检查数据可行性，二者对抗后再决定是否通过【primary】。

**最值得记住的一条量化结论**：作者做了消融实验对比"目标一致的多个 agent 互相审查"（Homogeneous Debate）vs "目标冲突的对抗式审查"（ACD）——在 MM-Bench（GPT-4o）上，同目标多智能体审查只把"建模严谨性"从单模型自省的 7.68 提到 7.92（几乎没提升），而**目标对抗式辩论提到 8.92**（大幅提升）。作者原话：**"agent 数量不重要，目标是否互相冲突才重要"**——单纯堆更多 agent 互相点头没用，得让它们站在对立立场互相挑刺。

另一条有意思的发现：Sci-Mind 用较弱的 GPT-4o 骨干，在工程建模基准 EngiBench 上已经全面超过 MM-Agent 用更强的 DeepSeek-R1 骨干——说明**架构设计比底层模型强弱更重要**。

**局限声明**（作者自己承认）：Sci-Mind 只在竞赛式建模任务上验证过，能否泛化到真实工业/实验室科研场景尚未验证。

## 3. 反直觉的警示：多智能体协作不能可靠抑制幻觉

一项 2025-10 发布的研究（arXiv 2509.23537）专门量化测试了"多个顶尖模型（GPT-5、Gemini 2.5 Pro、Grok 4、Claude Sonnet 4）投票共识"这种协作方式，用的是封闭式基准（GPQA-Diamond 等），结果值得警惕【primary】：

- 在 GPQA-Diamond 上，**至少一个 agent 给出正确答案的比例高达 95.5%**，但最终共识准确率只有 87.4%——协作过程中"丢失"了本来存在的正确信息；
- 在编排出错的案例中，**64% 的情况下其实至少有一个 agent 是对的**，其中 **31% 是两个以上 agent 都答对，却仍然投票投成了错的共识**；
- 案例分析显示，投票机制有时会被"看起来很详尽但其实错误"的分析压过"正确但简短"的答案——幻觉没被消除，只是从"单模型层面"转移到了"共识层面"；
- 协作机制的设计细节本身会引入新偏差：当投票时能看到彼此身份（而非匿名）时，模型的自我投票率会上升（GPT-5 从 81.0% 升到 88.4%），平局比例从 14.1% 骤升到 23.2%。

**结论**：多智能体系统不是"人多力量大"就自动可靠，Sci-Mind 那种**目标对抗式**设计能起效，是因为它刻意制造分歧、逼真验证；如果只是简单多数投票，反而可能被"说得好听但错的"意见带偏。

## 4. 结构化领域的强证据：99% vs 0%

一篇 2026-03 发布的论文针对结构工程建模（用 OpenSeesPy 自动生成有限元建模脚本）做了严格对比【primary】：

- 专门设计的多智能体架构（问题分析/规划、几何装配、荷载施加、代码翻译各一个 agent，分别用不同规模骨干模型分工）在 20 个代表性框架结构基准题上（每题 10 次试验）达到 **99% 平均准确率**；
- 对比：简单顺序式多智能体架构 91%，**Gemini-2.5 Pro 直接解题 37%，GPT-4o 直接解题 0%**；
- 效率也更优：单题运行时间 75–194 秒（顺序基线 269–949 秒），成本仅 0.013–0.023 美元/题。

这不是开放式建模场景（是结构工程细分领域），但和 MM-Agent 的结论互相印证：**专门设计的多智能体分工，在特定建模任务上能把"完全不会做"（0%）拉到接近满分（99%），差距极大**。

## 5. 幻觉检测本身也要专门做：OptArgus 与"答案对≠过程对"的再次印证

**OptArgus**（2026-05-12）是一个专门检测"LLM 生成的优化建模方案里有没有幻觉"的多智能体系统，用协调器+目标/变量/约束/实现四个专家 agent 分工审计【primary】。

**最重要的数字**：作者发现，即使 LLM 生成的优化模型**最优值数字和参考答案完全对得上**（表面看"算对了"），仍有**高达 31.1%** 的模型存在结构性幻觉（比如目标函数方向反了、该是整数变量的被错误松弛成连续变量、约束条件被悄悄弱化）——**这条和我们 [01-技术局限性](01-技术局限性.md) 第 1 条"答案对不代表推理过程对"是同一件事在优化建模这个具体场景下的实锤**。

OptArgus 用专才分工审计架构对比单智能体检测基线：干净样本上误报更少、单一错误定位更准、对真实 LLM 产出的幻觉检测能力更强——再次印证"分角色审查"比"一个模型自己检查自己"可靠。

## 6. 通用数据科学 Agent 的现状：能力边界很清楚

多个 2025 年末–2026 年的基准测试（DSAEval、DSGym、DataSciBench、DeepAnalyze-8B 等）勾勒出当前 agent 在"开放式数据分析/建模"任务上的真实水平【primary，多来源交叉】：

- **"简单性偏见"**：在 DSGym 的 DSPredict-Hard（基于 Kaggle 高难度竞赛构建）上，前沿 agent 有 **85.7% 能产出可运行的有效提交**，但达到"奖牌级别"（真正做得好）的比例只有约 **4.8%**——agent 倾向于"能跑通就停"，不会主动迭代优化到真正优秀的水平；
- **科学领域差距持续存在**：在 DSBio（源自同行评审论文的生物信息学任务）上，前沿模型准确率只有 **4–43%**，85–96% 的失败源于"领域知识/数据关联"错误而非代码错误——说明纯编程能力强不代表懂科学背景；
- **基准测试本身有漏洞**：研究发现把数据文件整个删掉、只给提示词，模型表现只下降 40–44%——意味着不少所谓"数据分析基准"其实很大程度上可以"蒙"，不是真的依赖对数据的理解；
- 2025-10 的一篇综述指出，现有 45 个数据科学 agent 系统中 **超过 90% 缺乏明确的信任/安全/治理机制**，且普遍集中在"探索性分析、建模"这类技术中段，**对"理解问题在问什么"（对应读题）这类前端环节反而投入最少**——这恰好是 MCM/ICM 这类题目最考验的地方。

## 7. 竞赛官方立场：政策没变，Agent 不享受特殊豁免

已核实截至 2026 年 7 月，**COMAP（现行版本 v102025）和 CUMCM（2025 年 9 月试行版）的 AI 政策条文均未修改，也都没有专门针对"自主 AI Agent"（区别于聊天式 AI）新增条款**【primary，直接核对官方 PDF】：

- COMAP 政策定义 AI 工具时列举 LLM、生成式 AI、数学软件、翻译软件、代码协作工具等，全文没有出现 "agent" 或 "autonomous" 字样；
- CUMCM 规定同样只列出"大语言模型、生成式 AI、代码辅助工具、智能数学软件"，未区分聊天式 AI 与自主执行任务的 Agent；
- 2026 年 ICM F 题本身是以"Gen-AI 对未来工作的影响"为主题，但这是命题内容，不是新的 AI 使用规则；
- **含义**：即使你们真的用上了 MM-Agent 这类工具，仍然适用和用 ChatGPT 完全相同的披露规则——正文标注 + AI Use Report / "AI 工具使用详情"，没有因为工具"更自动化"就有特殊要求或豁免，也没有因为更自动化就被禁止。核心的"核心建模与分析必须由参赛队独立完成"这条红线同样适用于 agent 系统的产出。

## 8. 对普通团队（用现成 ChatGPT/Claude，不自建 agent）的可落地启示

1. **把验证角色做成"目标对立"的，不要做成"互相点头"的**：Sci-Mind 的核心发现——真正有效的不是多问几次 AI"这样对吗"，而是明确让一次提问扮演"挑刺者"角色，目标就是找茬（"这个假设哪里站不住脚"），而不是笼统地问"帮我检查一下"。
2. **"能跑通"和"做得好"是两回事**：DSGym 的"简单性偏见"发现直接适用——AI（和你们自己）都容易在代码跑通、结果长得像样后就停手，敏感性分析、多方案对比这些"锦上添花"的步骤最容易被跳过，恰恰是评委最看重的部分（见 [12-模型与算法策略](../12-模型与算法策略/README.md)）。
3. **"答案数字对"不能作为验证通过的标准**：OptArgus 的 31.1% 结构性幻觉率说明，哪怕最终数字和某个参考值对上了，也可能是"蒙对的"——必须检查推导过程和中间假设，不能只对答案。
4. **AI 最弱的环节是"理解问题在问什么"**：综述发现现有 agent 普遍把力气花在建模、可视化这些技术中段，读题理解反而投入最少——这提示你们真人队伍应该把"审题、拆解问题"这一步留给人工主导，AI 更适合在模型已经想清楚之后接手代码和计算。
5. **MM-Agent 的真实获奖案例说明"专门设计的建模工作流"确实管用**，但它的核心不是"用了更牛的模型"，而是"把建模拆成四个阶段、每阶段配一个懂行的知识库"——**你们三人的岗位分工（见 [04-三人分工](../04-三人分工/README.md)）本质上就是这套架构的人工版本**，人工分工 + AI 辅助单个阶段，比让 AI 一次性通吃全流程更可靠。
6. **披露规则没有捷径**：不管用的是聊天式 AI 还是更花哨的 agent 工具，官方对"要不要披露"的要求完全一样，别指望工具越先进就能绕开合规要求。
